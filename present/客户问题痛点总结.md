## 二、核心问题详述

### 2.1 评审滞后 ⭐ 核心问题

| 维度         | 现状描述                                                           |
| ------------ | ------------------------------------------------------------------ |
| **问题表现** | 质检数据需要次日下午才能录入系统，质量评审严重滞后                 |
| **产生原因** | 当班质检填写手抄报表 → 次日下午才能录入局域网 → 质量工程师才能评审 |
| **影响范围** | 评审滞后 → 计划滞后 → 排产滞后 → 整体效率低下                      |
| **滞后时间** | 约 **24小时以上**（当班填写 → 次日下午录入 → 评审）                |

**问题链条：**

```
当班质检 → 手抄报表 → 次日下午录入系统 → 质量工程师评审 → 计划员匹配订单
         ↑                                                    ↓
     滞后起点                                            至少24小时后
```

---

### 2.2 数据采集分散

| 维度         | 现状描述                                                 |
| ------------ | -------------------------------------------------------- |
| **问题表现** | 针检、表检、离线板型、下料信息等数据分散在不同报表和位置 |
| **数据载体** | 手抄报表、局域网电子表格、各设备独立记录                 |
| **影响结果** | 无法快速获取完整的料卷质量信息，数据不集中               |

**数据分散现状：**

| 数据类型 | 记录方式 | 存储位置           |
| -------- | -------- | ------------------ |
| 针检数据 | 手抄     | 纸质报表           |
| 表检数据 | 手抄     | 纸质报表           |
| 离线板型 | 设备记录 | 设备本地/纸质      |
| 下料信息 | 手抄     | 纸质报表           |
| 评审信息 | 手工录入 | 局域网表格（次日） |

---

### 2.3 物料堆积严重

| 维度         | 现状描述                                   |
| ------------ | ------------------------------------------ |
| **问题表现** | 分切后大量料卷堆积在现场，形成"地堆式"存放 |
| **堆积数量** | 约 **1000个料卷** 堆放现场                 |
| **产生原因** | 分切产能 > 精切产能，料卷无法及时消化      |
| **存放方式** | 多层堆垛，无序堆放                         |

**堆积场景：**

- 分切后待评审料卷 → 暂放现场
- 评审合格待排产料卷 → 暂放现场
- 评审不合格料卷 → 暂放现场
- 精切成品待包装 → 暂放现场（包装无夜班，存在滞后）

---

### 2.4 找料/挑料困难

| 维度           | 现状描述                                   |
| -------------- | ------------------------------------------ |
| **问题表现**   | 从约1000个料卷中人工挑选目标料卷，费时费力 |
| **操作方式**   | 人工在料堆中逐个查找标签，识别目标料卷     |
| **挑料后操作** | 挑选后需合并料框，工作重复性高             |
| **效率影响**   | 严重影响精切排产效率                       |

**调研原文：**

> "现场约1000个卷，挑选暂放料费时费力，找料困难，需要解决这一大难题。"

---

### 2.5 计划排产滞后 ⭐ 核心问题

| 维度         | 现状描述                                         |
| ------------ | ------------------------------------------------ |
| **问题表现** | 计划员需等待评审完成后才能匹配订单、制定精切计划 |
| **依赖条件** | 质检评审信息 + 客户订单信息                      |
| **操作方式** | 在局域网"订单池表格"中进行人工匹配               |
| **滞后程度** | 受评审滞后影响，计划同样滞后至少24小时           |

**⭐ 关键痛点：精切工序只能生产前一天的料卷！**

因为质量评审和排产无法当天完成，精切工序永远都是在生产**前一天**分切的料卷。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              当前流程时间线                                   │
└─────────────────────────────────────────────────────────────────────────────┘

        第一天                              第二天                    第三天
          │                                   │                         │
    ┌─────┴─────┐                     ┌───────┴───────┐           ┌─────┴─────┐
    │ 分切生产  │                     │ 下午录入系统 │           │ 精切生产  │
    │ 质检填表  │                     │ 质量评审     │           │ (前天料卷)│
    │ (手抄报表)│                     │ 计划排产     │           │           │
    └───────────┘                     └───────────────┘           └───────────┘
          │                                   │                         │
          └───────── 滞后约24小时 ───────────┘                         │
                                              └────── 再滞后1天 ────────┘

★ 结果：今天分切的料卷，最快也要后天才能精切！
```

**这意味着：**

- 📦 料卷必须在现场堆放至少1-2天
- ⏰ 订单响应速度慢，无法快速交付
- 📊 库存周转效率低，资金占用多
- ⚠️ 堆积越多，找料越难，安全隐患越大

---

### 2.6 物料转运效率低

| 维度         | 现状描述                                     |
| ------------ | -------------------------------------------- |
| **问题表现** | 物料在各工序间转运效率低，存在碰伤风险       |
| **转运环节** | 分切→暂存、暂存→电晕、精切→包装、包装→成品库 |
| **操作方式** | 行车吊运 + AGV转运 + 人工辅助                |
| **碰伤风险** | 转运过程中存在碰伤、擦划伤风险               |

**精切成品转运流程（复杂）：**

```
精切下料 → AGV转运 → 人工取穿料棒 → 插入管芯 → 行车吊运 → 暂放料框
                        ↑
                   人工操作多，效率低
```

---

### 2.7 安全隐患多

| 风险类型         | 具体描述                             |
| ---------------- | ------------------------------------ |
| **高空作业风险** | 料卷多层堆放，人员需爬高取料、挂吊带 |
| **掉落风险**     | 发生过掉轴、砸脚等事故               |
| **交叉作业风险** | 包装区叉车与成品库叉车交叉作业       |
| **现场混乱**     | 物料堆放造成通道不畅、现场不规整     |

**调研原文：**

> "人员吊卷爬上爬下、挂吊带存在诸多安全风险"  
> "前期出现掉轴、砸脚等事故"

---

### 2.8 追溯困难

| 维度         | 现状描述                             |
| ------------ | ------------------------------------ |
| **问题表现** | 出现质量问题时，难以快速追溯完整信息 |
| **数据来源** | 分散在多个纸质报表和电子表格中       |
| **追溯耗时** | 需要手工翻查多份报表，耗时长         |

---

### 2.9 成品库管理低效

| 维度         | 现状描述                                                               |
| ------------ | ---------------------------------------------------------------------- |
| **库存规模** | 每日包装约300吨成品，需入库管理                                        |
| **存储方式** | 叉车堆放，最多4层                                                      |
| **占地面积** | 一期3300㎡ + 二期2500㎡ = 5800㎡                                       |
| **配货问题** | 需要的货在里面时，需先将外侧货物叉出，再取出目标货物，再将外侧货物放回 |
| **配货效率** | 低效、耗时，影响发运时间                                               |

## 六、总结

### 核心问题一句话

> **"手抄报表导致数据滞后和分散，评审计划无法及时进行，物料大量堆积，找料困难，安全隐患突出。"**

### 第一阶段改造聚焦

**智能数据采集与管理系统**重点解决：

1. ✅ 消除手抄报表 → 数据实时采集
2. ✅ 消除评审滞后 → 自动判定 + 及时通知
3. ✅ 消除计划滞后 → 智能排产 + 自动匹配
4. ✅ 消除追溯困难 → 全流程追溯查询
