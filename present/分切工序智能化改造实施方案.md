# 分切工序智能化改造实施方案

## 一、改造背景

客户现状存在的核心问题：

- ❌ 评审滞后、计划滞后
- ❌ 数据采集不集中（针检、表检、离线板型、下料查料）
- ❌ 物料堆积严重（约1000个料卷地堆式存放）
- ❌ 找料和挑料困难，效率低下
- ❌ 物料转运、上下料辅助时间长
- ❌ 安全隐患多（掉轴、砸脚、爬高作业等）

---

## 二、智能化改造的六大核心方面

### 🎯 方面一：智能数据采集与管理系统

**改造目标：** 建立集中式智能管理系统，实现数据实时采集与自动评审

**系统定位：** 作为分切工序智能化改造的核心基础系统，为后续高架库、AGV等硬件智能化提供数据支撑

#### 1.0 功能模块总览

系统采用 **"4+4"架构**，包含4个核心业务模块和4个基础支撑模块：

**核心业务模块：**

1. **数据自动采集模块** - 替代手抄报表，实现生产数据实时采集
2. **智能评审模块** - 自动质量判定，加速评审流程
3. **智能排产模块** - 订单与料卷自动匹配，生成精切计划
4. **追溯查询模块** - 全流程追溯，支持质量问题分析

**基础支撑模块：** 5. **基础数据管理模块** - 管理产品规格、质量标准、客户信息等基础数据 6. **料卷全生命周期管理模块** - 管理料卷从产生到发货的全状态流转 7. **用户权限管理模块** - 管理系统用户、角色和权限 8. **报表与看板模块** - 提供生产看板、质量报表等数据展示

#### 1.2 核心业务模块详述

##### 1.2.1 数据自动采集模块

**模块目标：** 替代现有的手抄报表，消除次日下午才能录入的滞后问题

| 数据类型     | 数据来源    | 采集方式        | 采集频率 | 采集内容                   |
| ------------ | ----------- | --------------- | -------- | -------------------------- |
| 料卷基础信息 | 分切机/人工 | 扫码 + 人工录入 | 下料时   | 卷号、规格、重量、母卷号等 |
| 针检数据     | 针检仪      | 设备接口/人工   | 实时     | 针孔数量、密度、位置分布   |
| 表检数据     | 表检仪      | 设备接口/人工   | 实时     | 缺陷类型、数量、位置、图像 |
| 板型数据     | 离线板型仪  | 设备接口/人工   | 检测后   | 板型偏差值、曲线图         |
| 电晕数据     | 电晕机      | 设备接口/人工   | 生产时   | 电晕功率、速度、宽度       |
| 精切数据     | 精切机/人工 | 设备接口/人工   | 下料时   | 精切规格、米数、宽度、厚度 |

- [ ] 表检仪品牌型号及数据接口类型
- [ ] 离线板型仪品牌型号及数据接口类型
- [ ] 分切机PLC型号及通讯协议
- [ ] 电晕机控制系统及通讯协议
- [ ] 精切机PLC型号及通讯协议

##### 1.2.2 智能评审模块

**模块目标：** 根据采集数据自动进行质量判定，权限范围内自动评审，无需等待次日质量工程师

**自动评审规则框架：**

| 检测项   | 自动判定合格 | 自动判定不合格 | 需人工评审   |
| -------- | ------------ | -------------- | ------------ |
| 针孔数量 | ≤ A个/㎡     | > B个/㎡       | A < 数量 ≤ B |
| 表面缺陷 | 无A类缺陷    | 存在C类缺陷    | 存在B类缺陷  |
| 板型偏差 | < X mm       | > Y mm         | X ≤ 偏差 ≤ Y |

**评审流程：**

```
质检完成 → 系统自动判定 → ┬→ 合格 → 自动入库/可排产
                         ├→ 待评审 → 通知质量工程师 → 人工评审 → 合格/降级/不合格
                         └→ 不合格 → 自动隔离 → 不合格品处理
```

**评审结果类型：**

- 合格（可转包装/可排产精切）
- 降级使用（降低质量等级后可用）
- 不合格-返工（返回上游工序修复）
- 不合格-报废（无法修复，报废处理）

**待调研确认项：**

- [ ] 具体判定阈值（A、B、X、Y等参数）
- [ ] 缺陷分类标准（A/B/C类定义）
- [ ] 当班质检权限范围
- [ ] 复合判定规则（多项指标综合判定）
- [ ] 降级使用规则及审批流程

##### 1.2.3 智能排产模块

**模块目标：** 根据质检数据自动选料，与客户订单自动匹配，生成精切来料计划

**核心功能：**

- **订单管理：** 从ERP同步客户订单，管理订单状态
- **自动选料：** 根据订单规格要求，自动从合格料卷池中匹配
- **排产计划：** 生成精切来料计划，指导生产
- **订单跟踪：** 跟踪订单执行进度，支持订单变更

**物料匹配规则：**

| 匹配参数  | 匹配规则                 | 优先级 |
| --------- | ------------------------ | ------ |
| 厚度      | 必须完全匹配             | 1      |
| 宽度      | 料卷宽度 ≥ 订单要求宽度  | 2      |
| 长度/米数 | 料卷米数 ≥ 订单要求米数  | 3      |
| 质量等级  | 料卷等级 ≥ 订单要求等级  | 4      |
| 库存时间  | 优先选择库存时间长的料卷 | 5      |

**排产优先级：**

1. 订单交期（交期紧的优先）
2. 客户重要性（VIP客户优先）
3. 相同规格集中排产（减少换型）
4. 库存时间（先进先出）

##### 1.2.4 追溯查询模块

**模块目标：** 全流程数据追溯（从轧机到包装），便于质量问题追溯和分析，为包装工序匹配木箱提供准确信息

**追溯方向：**

- **正向追溯：** 轧机母卷 → 分切料卷 → 精切料卷 → 包装成品
- **反向追溯：** 包装成品 → 精切料卷 → 分切料卷 → 轧机母卷
- **横向追溯：** 同一轧机母卷的所有分切料卷、同一批次的料卷等

**追溯信息：**

- 生产信息：各工序的生产时间、设备、操作人员
- 质量信息：质检数据、评审结果、评审人员
- 工艺信息：分切参数、电晕参数、精切参数
- 状态信息：料卷状态流转历史

##### 1.3.2 料卷全生命周期管理模块

**模块目标：** 管理料卷从产生到发货的全状态流转

**料卷状态定义：**

| 状态   | 说明                   | 可执行操作 |
| ------ | ---------------------- | ---------- |
| 待检测 | 分切下料后，等待质检   | 开始质检   |
| 检测中 | 正在进行质检           | 完成质检   |
| 待评审 | 质检完成，需要人工评审 | 提交评审   |
| 合格   | 质检/评审通过          | 排产、转运 |
| 待精切 | 已分配订单，等待精切   | 上料精切   |
| 精切中 | 正在精切生产           | 完成精切   |
| 待包装 | 精切完成，等待包装     | 开始包装   |
| 已包装 | 已装箱，等待发货       | 安排发货   |
| 已发货 | 已出库发货             | 归档       |
| 不合格 | 质检/评审不通过        | 返工、报废 |

**状态流转图：**

```
分切下料 → 待检测 → 检测中 → ┬→ 合格 → 待精切 → 精切中 → 待包装 → 已包装 → 已发货
                             ├→ 待评审 → 评审 → ┬→ 合格/降级
                             │                  └→ 不合格 → 隔离
                             └→ 不合格 → 隔离 → ┬→ 返工 → 待检测
                                               └→ 报废
```

### 🎯 方面二：智能高架仓库系统（核心改造）

**改造目标：** 解决物料堆积、找料困难、安全隐患等核心痛点

#### 2.1 分切后物料高架库

**现状问题：**

- 约1000个料卷地堆式存放
- 找料、挑料困难，费时费力
- 合并料框工作重复性高
- 人员爬上爬下挂吊带，安全风险高

**改造方案：**

- 建立自动化高架仓库，分切下料后自动转运入库
- 配备自动出入库系统
- 根据精切订单指令自动选料出库
- 库位管理系统，精确定位每个料卷

#### 2.2 精切后成品高架库

**现状问题：**

- 包装工序有滞后性（无夜班）
- 精切成品堆积包装现场
- 转运过程存在碰伤、擦划伤风险
- 人工取穿料棒、吊运效率低

**改造方案：**

- 在包装前建立高架库
- 待包装的母卷、精切料自动暂存
- 根据木箱、订单自动从高架库取料
- 配合AGV实现自动转运

#### 2.3 包装后成品高架库

**现状问题：**

- 每日约300吨木箱成品地堆存放
- 现有仓库面积大（一期3300㎡，二期2500㎡）
- 最多堆垛4层，配货需反复搬运
- 叉车交叉作业，安全风险高
- 配货效率低，影响发运时间

**改造方案：**

- 建立成品智能高架库
- 自动出入库功能
- 提高空间利用率，减少占地面积
- 智能配货系统，快速定位发货物料
- 减少人员成本，提高配货效率

---

### 🎯 方面三：AGV智能物流系统

**改造目标：** 实现物料自动转运，减少人工搬运和安全风险

**具体内容：**

1. **分切至高架库**
   - 分切下料后，AGV自动转运至高架库入口
   - 自动完成入库动作

2. **高架库至电晕机**
   - 根据精切订单，AGV从高架库运送至电晕机入口
   - 自动上料系统

3. **精切至包装区域**
   - 精切下料后，AGV运输至包装区域高架库
   - 减少人工取穿料棒、吊运等操作

4. **包装区域内部转运**
   - 替代现有叉车部分工作
   - 减少叉车交叉作业的安全风险

---

### 🎯 方面四：电晕/精切工序自动化

**改造目标：** 减少人工辅助时间及安全风险

**具体内容：**

1. **自动调刀系统**
   - 根据来料宽度、电晕宽度自动调刀
   - 消除人工调整误差

2. **自动上料系统**
   - AGV运送至电晕机入口后自动上料
   - 减少人工操作

3. **自动上卷取轴**
   - 自动完成卷取轴的安装
   - 避免掉轴、砸脚等事故（前期已发生）

4. **自动卸料系统**
   - 精切完成后自动卸料
   - 配合AGV转运

---

### 🎯 方面五：自动包装系统

**改造目标：** 提高包装效率，减少人工错误

**具体内容：**

1. **智能木箱匹配**
   - 根据精切信息自动匹配合适的木箱
   - 替代现有人工手抄匹配方式
   - 降低来料信息错误率

2. **自动包装线**
   - 从高架库自动取料
   - 自动完成包装或辅助人工包装
   - 自动转运至成品高架库

3. **包装信息自动采集**
   - 包装完成后自动记录信息
   - 与成品库系统对接

---

### 🎯 方面六：三期项目整体规划

**改造目标：** 在新建项目中实现全流程智能化

**规划建议：**

1. **整体布局优化**
   - 铝箔轧机 → 分切机 → 高架库 → 电晕机 → 精切机 → 高架库 → 包装区域 → 成品高架库
   - 物流路径最优化设计

2. **三个高架库的协同**
   - 分切后暂放料高架库
   - 精切成品至包装高架库
   - 成品库高架库
   - 每个工段均具备智能自动出入库功能

3. **空间利用优化**
   - 减少暂存空间浪费
   - 提高单位面积存储量
   - 改善物流通畅性

4. **前期准备工作**
   - 设计调研与考察（目前暂无完整案例）
   - 结合现场实际工况
   - 制定最终详细方案

### 4.1 分阶段实施

**第一阶段：** 智能数据采集与管理系统

- 优先级：⭐⭐⭐

- 优先级：⭐⭐⭐

## 七、风险管理

### 7.1 风险识别与应对

| 风险类型 | 风险描述                     | 影响程度 | 发生概率 | 应对措施                                               |
| -------- | ---------------------------- | -------- | -------- | ------------------------------------------------------ |
| 技术风险 | 检测设备无数据接口或接口封闭 | 高       | 中       | 提前调研设备接口；必要时增加数据采集模块或联系设备厂商 |
| 技术风险 | ERP系统对接困难              | 中       | 中       | 提前与客户IT和ERP供应商沟通，明确接口规范和责任        |
| 实施风险 | 系统上线影响正常生产         | 高       | 低       | 分模块上线；设置并行期；利用非生产时间部署             |
| 数据风险 | 历史数据迁移困难             | 中       | 中       | 制定数据清洗和迁移方案；必要时人工补录关键数据         |
| 管理风险 | 一线人员抵触新系统           | 中       | 中       | 加强培训；简化操作界面；设置过渡期奖励机制             |
| 需求风险 | 需求变更频繁                 | 中       | 中       | 明确需求确认流程；变更需书面确认并评估影响             |

### 7.2 关键成功因素

1. **设备接口可行性**：检测设备能够输出数据是系统成功的前提
2. **客户配合度**：需要客户IT、生产、质量等部门的积极配合
3. **评审规则明确**：质量判定规则需要客户提供明确标准
4. **系统集成能力**：与现有ERP系统的顺利对接
