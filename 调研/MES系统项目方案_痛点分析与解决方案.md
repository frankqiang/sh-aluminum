## 痛点分析与解决方案

> **文档版本**：V1.0  
> **编制日期**：2026年2月  
> **文档目的**：基于现场调研，梳理当前生产管理痛点，提出MES系统解决方案

# 第一章：项目背景

## 1.1 企业现状

贵公司是一家专业生产各类铝箔卷（电池箔、双零箔等）的制造企业，拥有：

- 轧机：14台（3台粗轧+3台中轧+1台中精轧+7台精轧）
- 分切机：14台（其中8台配备针检/表检设备）
- 精切机：49台
- 合卷机：2台（双零箔专用）
- 退火炉：60台

目前生产管理主要依赖**Excel表格**和**人工协调**，随着产能扩大和客户质量要求提升，现有管理方式面临较大压力。

## 1.2 调研范围

本次调研重点覆盖以下环节：

```
胚料出库 → 轧机（多道次）→ 分切 → 精切/退火 → 包装
              ↓                ↓          ↓
           轧机质检        分切质检    精切质检  
```

通过与计划员、质检人员的深入沟通，我们识别出当前生产管理的**4大核心痛点**。

# 第二章：现状痛点分析

## 2.1 痛点总览

| 痛点                            | 现状描述                                                               | 影响                                                   |
| ------------------------------- | ---------------------------------------------------------------------- | ------------------------------------------------------ |
| **1. 计划管理分散**             | 32个Excel Sheet分散管理轧机/分切/精切计划，无汇总视图                  | 计划员在Sheet间切换、信息不同步、难以把握整体进度      |
| **2. 质量评审不规范且传递断链** | 评审信息采用自由文本描述不统一，且需手动复制传递，包装靠共享文件夹获取 | 难以统计分析、评审信息滞后、漏看、计划可能先于评审执行 |
| **3. 在制品追踪困难**           | 约1000个分切子卷堆在现场，靠人工找料，位置信息靠记忆/手写              | 精切人员找料困难、在制品积压无法量化、呆滞料不可见     |
| **4. 订单进度不透明**           | 订单无状态跟踪，多卷满足一个订单时靠人工统计、逾期无预警               | 交期管理困难、逾期风险高、客户投诉                     |

---

## 2.2 痛点1：计划管理分散

### 现状描述

当前生产计划管理存在以下问题：

1. **计划表分散**：轧机/分切/精切计划分布在32个Excel Sheet中（每台设备一个Sheet）
2. **无汇总视图**：计划员需要在Sheet之间频繁切换，难以一眼把握整体进度
3. **信息不同步**：轧机下料后，分切计划需要手动更新可用料清单
4. **版本混乱**：计划修改后无历史记录，出问题时难以追溯

### 具体表现

- 计划员每天需要在32个Sheet间切换查看各机台进度
- 轧机完成后，需手动将信息复制到"轧制跨在制料表"供分切参考
- 计划调整后，其他人不知道改了什么、为什么改

### 影响

- 计划员工作量大，容易遗漏
- 信息同步滞后，可能出现"轧机已下料但分切不知道"的情况
- 出问题时无法追溯计划调整记录

---

## 2.3 痛点2：质量评审信息不规范且传递断链

### 现状描述

质量评审信息存在以下问题：

1. **描述不规范**：评审信息采用自由文本填写，不同人员对同一类型缺陷的描述方式不一致，难以统计分析
2. **手动复制**：质量部门将评审结论写在质量报表Excel中，计划员需手动复制到分切计划
3. **可能滞后**：存在"料已到分切工位，但评审还没出"的情况
4. **传递链长**：评审信息从质量部门→计划员→分切→精切→包装，每个环节都靠人工传递
5. **容易漏看**：包装工序通过共享文件夹获取信息，容易遗漏

### 具体表现

**描述不规范的例子**：
| 同一类型缺陷 | 不同人员的描述方式 |
|-------------|-------------------|
| Q侧针孔问题 | "Q侧890-990针孔切除"、"Q890-990针孔密集需切"、"Q侧890mm处有针孔要切掉" |
| 边部辊印 | "C侧辊印避白"、"C边辊印要避开"、"C25辊印切除" |

- 没有统一的缺陷类型编码，全靠文字描述
- 位置表述方式不一致（有的写"Q侧890mm"，有的写"Q890"）
- 处理方式表述不统一（"切除"、"避白"、"避留白"混用）

**传递断链的表现**：

- 质量评审结果写在Excel的"备注"列，计划员需手动复制到分切计划的"说明"列
- 包装需要到共享文件夹查看评审信息，确认是否需要"避留白"

### 影响

- **难以统计分析**：同一类型缺陷描述不一致，无法统计"这个月针孔问题有多少"
- **理解歧义**：新员工或其他工序人员可能误解评审意图
- 评审信息可能滞后或漏传
- 存在"未评审先生产"的风险
- 包装环节可能漏看重要的质量备注

---

## 2.4 痛点3：在制品追踪困难

### 现状描述

在制品管理存在以下问题：

1. **找料困难**：分切完成后约1000个子卷堆在现场，精切人员需要在大量料卷中找到目标料卷
2. **位置信息模糊**：料卷位置（如"3195号料框"）靠手写或口头传递
3. **状态不可见**：无法实时知道"现在有多少料等待精切""哪些料已经等了很久"
4. **呆滞料隐藏**：长期未处理的料卷可能被遗忘

### 具体表现

- 精切人员拿到计划后，需要在现场凭经验寻找对应的料卷
- 有时"料明明在那里，但找了半天才找到"
- 部分料卷可能因为各种原因长期滞留现场，无人处理

### 影响

- 找料时间长，影响生产效率
- 在制品积压情况不透明
- 呆滞料占用场地资源

---

## 2.5 痛点4：订单进度不透明

### 现状描述

订单管理存在以下问题：

1. **无状态跟踪**：订单在MES订单池中没有明确的状态标记（待排产/生产中/已完成）
2. **人工统计**：一个订单需要多卷料满足时，靠人工统计已完成多少
3. **无逾期预警**：交期临近时才发现可能赶不上，无提前预警机制
4. **重排困难**：质量问题导致订单需要重新匹配料卷时，进度更难追踪

### 具体表现

- 订单优先级主要靠计划员记忆和经验判断
- 客户问"我的订单做到哪了"，需要翻多张Excel表统计
- 某个订单差一卷没做完，可能直到发货时才发现

### 影响

- 交期管理被动，逾期风险高
- 客户询问时无法快速响应
- 多卷订单容易遗漏

# 第三章：解决方案

## 3.1 解决方案概述

针对上述4大痛点，我们提出MES系统解决方案，包含以下核心模块：

| 痛点             | 对应模块         | 核心价值                             |
| ---------------- | ---------------- | ------------------------------------ |
| 计划管理分散     | **排产模块**     | 统一平台管理所有计划，提供生产看板   |
| 质量评审传递断链 | **质量模块**     | 评审信息结构化录入，自动同步到各环节 |
| 在制品追踪困难   | **物料管理模块** | 在制品实时可视，位置可查，呆滞预警   |
| 订单进度不透明   | **生产执行模块** | 订单状态实时跟踪，交期预警           |

---

## 3.2 排产模块：解决计划管理分散

### 功能设计

| 功能             | 说明                                          | 价值                        |
| ---------------- | --------------------------------------------- | --------------------------- |
| **统一排产平台** | 轧机/分切/精切三层计划在同一系统中管理        | 取代32个Sheet，一个界面全览 |
| **生产看板**     | 实时显示各机台的计划进度、在制数量            | 一眼看清整体状况            |
| **计划联动**     | 轧机完成自动更新可分切料清单                  | 减少手动复制，信息实时同步  |
| **修改留痕**     | 所有计划调整保留修改记录                      | 可追溯谁改了什么            |
| **订单匹配推荐** | 系统根据规格/交期推荐最优分切方案（可选功能） | 辅助决策，减少计算负担      |

## 3.3 质量模块：解决评审信息不规范和传递断链

### 功能设计

| 功能             | 说明                                                                                                                                                          | 价值                                     |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------- |
| **结构化录入**   | 缺陷类型、位置、处理方式均采用下拉选择+数值输入，统一格式                                                                                                     | 解决"同一缺陷不同描述"问题，支持统计分析 |
| **评审信息录入** | 质检员在系统中录入评审结论，结构化存储                                                                                                                        | 取代Excel自由文本，格式统一              |
| **流程卡点**     | 评审完成后才能进入下道工序报工                                                                                                                                | 杜绝"未评审先生产"的风险                 |
| **实时同步**     | 评审完成后计划员/包装立即可见                                                                                                                                 | 信息无延迟，不再漏看                     |
| **权限分级**     | 轻微问题当班生效，降级/报废需工程师确认                                                                                                                       | 既保效率又控风险                         |
| **修改留痕**     | 评审结论可修改但保留修改记录                                                                                                                                  | 满足追溯要求                             |
| **辅助信息展示** | 录入评审时，系统自动显示该客户的质量阈值要求，以及类似缺陷类型的历史评审案例（例如：上次同样是"Q侧针孔密集"，评审结论是什么、怎么处理的），帮助质检员参考判断 | 辅助决策，减少人工查询和经验依赖         |

### 结构化录入示例

**以前（Excel自由文本）**：

- "Q侧890-990针孔切除"
- "Q890-990针孔密集需切"
- "Q侧890mm处有针孔要切掉"

**现在（系统结构化录入）**：
| 字段 | 录入方式 | 示例值 |
|------|---------|-------|
| 缺陷类型 | 下拉选择 | 针孔 |
| 缺陷位置-侧 | 下拉选择 | Q侧 |
| 缺陷位置-起始 | 数值输入 | 890mm |
| 缺陷位置-终止 | 数值输入 | 990mm |
| 处理方式 | 下拉选择 | 切除 |
| 备注 | 自由文本 | （可选） |

**好处**：

- 格式统一，不同人员录入结果一致
- 支持按缺陷类型统计："这个月针孔问题有多少次"
- 下道工序理解无歧义

### 评审流程示意

```
轧机下料 → 质检 → 【结构化评审录入】 → 评审完成
                         ↓
                系统自动同步到：
                ├── 计划员（排分切计划时可见）
                ├── 分切工位（扫码可见评审备注）
                ├── 精切工位（扫码可见评审备注）
                └── 包装工位（自动显示"避留白"等信息）
```

## 3.4 物料管理模块：解决在制品追踪困难

### 功能设计

| 功能               | 说明                                                 | 价值                       |
| ------------------ | ---------------------------------------------------- | -------------------------- |
| **在制品状态看板** | 实时显示轧机→分切、分切→精切各环节的在制品数量和状态 | 一眼看清现场有多少料       |
| **料卷状态流转**   | 每卷料有状态标记（已下料、质检中、可分切、待精切等） | 知道每卷料在哪个环节       |

| **扫码查询**       | 现场可扫码查看料卷信息、历史轨迹                     | 方便快速验证"这卷料是什么" |

# 第四章：系统价值总结

## 4.1 场景对比

| 场景               | 使用系统前                   | 使用系统后             |
| ------------------ | ---------------------------- | ---------------------- |
| **查看今天的计划** | 在32个Excel Sheet中切换      | 打开生产看板，一屏全览 |
| **了解订单进度**   | 人工统计各卷完成情况         | 系统自动汇总，实时显示 |
| **质量评审传递**   | 手动复制到分切计划，可能漏看 | 评审即时同步，漏不了   |
| **精切找料**       | 在1000个卷里凭经验找         | 扫码查位置，快速定位   |
| **发现逾期风险**   | 临近交期才发现               | 提前预警，留出调整时间 |
| **追溯问题卷**     | 翻多张Excel表                | 输入卷号，一键追溯     |
| **计划调整**       | 修改后无记录                 | 所有修改留痕可查       |

## 4.2 期望达成的效果

### 1. 提高效率

- 减少计划员在Excel间切换、手工复制、人工统计的时间
- 减少精切人员找料时间
- 减少质量评审信息传递的时间

### 2. 减少错误

- 信息自动同步，避免漏看、漏填
- 流程卡点控制，避免"未评审先生产"
- 系统留痕，避免版本混乱

### 3. 增强可控性

- 实时掌握订单进度、在制品状态
- 异常情况（逾期、呆滞）及时预警
- 管理决策有数据支撑

### 4. 支撑追溯

- 完整的数据链条：胚料→轧机→分切→精切→包装
- 问题可追溯、责任可界定
- 满足客户质量追溯要求

| -------- | -------------------------------------------------------- |

| 电晕要求 | 达因值≥33，部分需两遍电晕                                |
