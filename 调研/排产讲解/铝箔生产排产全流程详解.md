## 一、铝箔卷材的物理形态

### 1.1 卷材的基本形态

铝箔是**卷筒状**的薄片材料，类似于保鲜膜或胶带：

```
         侧视图                     正视图（横截面）
    ═════════════            ╔═══════════════════╗
    ║           ║            ║                   ║
    ║  铝箔卷   ║            ║   宽度 (mm)       ║
    ║           ║            ║                   ║
    ═════════════            ╚═══════════════════╝
    ←─ 长度 (米) ─→          ←──── 1350mm ────→
```

**三个关键尺寸：**

- **厚度**：13μm（0.013mm，非常薄）
- **宽度**：1350mm（横向尺寸）
- **长度**：25000米（卷起来的总长度）

## 二、工序物理过程详解

### 2.1 轧机工序：从厚到薄

```
胚料卷（厚220μm）
    ↓
┌──────────────────────────────────┐
│   轧机（5道次，逐步压薄）         │
│                                  │
│   道次1：220μm → 110μm           │
│   道次2：110μm → 55μm            │
│   道次3：55μm → 38μm             │
│   道次4：38μm → 13μm             │
│   道次5：13μm → 13μm (成品道次)   │
└──────────────────────────────────┘
    ↓
轧机成品（薄13μm，宽1350mm，长25000米）
```

**关键点：**

- 轧机只改变**厚度**
- 宽度和长度基本不变（略有延伸）
- 输出：宽幅成品卷

---

### 2.2 分切工序：纵向切宽度

#### 物理过程

分切机采用**纵向切割**，类似于切披萨：

```
输入：母卷（宽1350mm）

正视图（从端面看）：
╔═══════════════════════════════╗
║         1350mm 宽              ║
║                               ║
╚═══════════════════════════════╝

分切机刀片位置（纵向刀，沿长度方向切）：
╔═══════════════════════════════╗
║ │边丝│    │成品│    │边丝│    ║
║ 10mm  1100mm      240mm      ║
╚═╪═══════╪═════════╪═══════════╝
  刀1     刀2       (废料)

输出：多个窄卷（车次）
╔═══════════╗   ╔═══════╗
║ 1100mm宽  ║   ║ 240mm ║ (废料)
╚═══════════╝   ╚═══════╝
  车次1            边丝
```

**关键理解：**

- ✅ 分切是**纵向切**（沿着卷的长度方向，从头到尾切成多条）
- ✅ 切的是**宽度**（把宽卷切成窄卷）
- ✅ 长度**不主动切**（整卷连续走，到设定米数自然停机）
- ✅ 一卷母卷切成多个"车次"（子卷）

#### 车次号的含义

```
母卷：Y2601301（1350mm × 25000米）
    ↓ 分切
车次1：Y2601301/1-1（1100mm × 23000米）← 第1车
车次2：Y2601301/2-1（1100mm × 2000米） ← 第2车（余料）
```

**车次号格式：** `母卷号/X-Y`

- X = 第几车（按时间顺序）
- Y = 待确认（通常为1）

---

### 2.3 精切工序：横向切长度 + 可能再切宽度

#### 物理过程

精切机的两个主要功能：

**功能1：横向切断（切长度）**

```
输入：车次（860mm × 14500米）
      ═══════════════════════════  ← 一卷长料

精切时按长度分段切断：
      ═══════════│══════════════
      6500米     │  8000米
      ↓          ↓
   给订单A      给订单B
```

**功能2：可能再切宽度（如果订单需要更窄）**

```
车次宽度：860mm
订单A要求：654mm

正视图：
╔═══════════════╗
║   860mm宽     ║
╚═══════════════╝
      ↓ 精切
╔═══════════╗  ╔═╗
║  654mm    ║  ║X║ ← 切掉206mm
╚═══════════╝  ╚═╝
```

**关键理解：**

- ✅ 精切的核心是**切长度**（一车切成多段）
- ✅ 可能**再切宽度**（如果订单要求更窄）
- ✅ 有电晕处理（表面处理工艺，提升达因值）
- ✅ 输出：精切成品（最终规格）

## 三、完整排产流程（从订单到成品）

### 3.1 流程总览

```
【客户下单】
    ↓
【订单进入订单池】(Excel/MES)
    ↓
┌────────────────────────────────────────┐
│  阶段1：轧机排产                        │
│  决策：选胚料 + 分配道次 + 排设备        │
└────────────────────────────────────────┘
    ↓
【轧机执行】→【质检】
    ↓
┌────────────────────────────────────────┐
│  阶段2：分切排产 ⚠️核心                 │
│  决策：宽度分配 + 切刀方案 + 订单匹配    │
└────────────────────────────────────────┘
    ↓
【分切执行】→【质检】
    ↓
┌────────────────────────────────────────┐
│  阶段3：精切排产 ⚠️核心                 │
│  决策：长度分配 + 切割顺序 + 宽度调整    │
└────────────────────────────────────────┘
    ↓
【精切执行】→【包装出库】
```

### 3.2 阶段1：轧机排产（从胚料到成品卷）

#### 场景：订单池中的需求

**订单信息：**

- 客户：弘力
- 合金：1235D
- 规格：13μm × 1340mm
- 米数：23000米/车
- 卷数：50卷
- 交期：2月5日

#### 计划员的决策过程

**步骤1：选胚料**

```
查看胚料库存（2026卷号.xlsx）：
Y2601301：合金1235D，厚220μm，重量2.5吨 ✅ 符合

决策：用Y2601301轧制
```

**步骤2：规划道次**

```
目标厚度：13μm
道次规划：
道次1-3：220μm → 38μm   粗轧机（1-3号）
道次4：  38μm → 13μm    中轧机（4-6号）
道次5：  13μm → 13μm    精轧机（8-14号）成品道次
```

**步骤3：分配设备**

```
考虑产能平衡：
- 1号粗轧：1-3道次连轧，1.5小时
- 4号中轧：道次4，1小时（需提前备料）
- 13号精轧：道次5，2小时

在Excel中填写轧机计划：
卷号：Y2601301
道次：1-3-4-5
设备：1号-1号-4号-13号
说明：厚度不超6，针孔≤1000（订单要求）
```

#### 质检环节

**轧机成品质检结果：**

```
卷号：Y2601301
厚度：13.2μm ✅
针孔：A级850个，B级120个
表面：Q侧有轻微辊印 ⚠️
板型：B级

质检员录入Excel：
"针孔略多需注意，Q侧840-990mm有辊印透，板型良好"
```

**此时Y2601301成为"轧机成品"，进入待分切库存。**

---

### 3.3 阶段2：分切排产（宽卷 → 窄卷）

#### 场景：计划员拿到待分切料

**在制品信息（轧制跨在制料.xlsx）：**

```
卷号：Y2601301
规格：1235D合金，13μm厚，1350mm宽，25000米长
质量：Q侧840-990mm有辊印透
```

#### 计划员的决策过程

**步骤1：理解质量信息（关键！）**

```
质量记录："Q侧840-990mm有辊印透"

计划员的人工理解：
- Q侧 = 驱动侧（卷的右边）
- 840-990mm = 从右边缘840mm到990mm的区域
- 辊印透 = 严重缺陷，需要切除

画图理解（正视图）：
┌─────────────────────────────────────┐
│         1350mm 宽                   │
│                                     │
│  0mm ←────────────────────→ 1350mm │
│  C侧                          Q侧   │
│                       ▓▓▓▓▓         │
│                       840-990mm     │
│                       (缺陷区)       │
└─────────────────────────────────────┘
```

**步骤2：计算有效宽度（人工！）**

```
原始宽度：1350mm
缺陷宽度：990 - 840 = 150mm
边丝宽度：左右各5mm = 10mm

有效宽度 = 1350 - 150 - 10 = 1190mm
```

**步骤3：订单匹配（人工！）**

```
查看订单池：
- 弘力订单：要1340mm宽 → 1190mm < 1340mm ❌ 不够！
- 源元订单：要860mm宽，但合金是1070 ❌ 不匹配
- XX客户：要1100mm宽，1235D合金 ✅ 匹配！

决策：这卷料切给XX客户
```

**步骤4：设计切刀方案（人工！）**

```
有效宽度：1190mm
订单要求：1100mm × 1刀

切刀方案：
┌───┬─────────────┬─────┐
│边丝│   1100mm    │余料 │
│10mm│   (成品)    │80mm │
└───┴─────────────┴─────┘

废料率：(10+80)/1350 = 6.7% ✅ 可接受
```

**步骤5：在Excel中填写分切计划**

```
序号：012
卷号：Y2601301
来料规格：13×1350
执行单号：NC240XXXX
说明：
  1、13×1100电池箔×23000-24000
  用152.4管芯
  Q840-990辊印切除，有效宽度1190mm
```

#### 分切执行

**操作工按计划执行：**

```
Y2601301（1350mm × 25000米）
    ↓ 分切机纵向切割
Y2601301/1-1（1100mm × 23000米）← 第1车
Y2601301/2-1（1100mm × 2000米） ← 第2车（余料）
```

#### 分切质检（每一车都要检！）

**质检员检测车次/1-1：**

```
车次号：Y2601301/1-1
针检仪：A级94个，B级13个，C级2个
表检仪：底部5800米处针孔密集，20262米Q584油块
板型仪：B级，下榻量2.5mm

质检员人工总结录入Excel：
"底部5800米针孔切二级品，Q620孔洞避留白，Q100针孔切除，C25针孔切除"
```

**此时车次/1-1进入待精切库存。**

---

### 3.4 阶段3：精切排产（一车 → 多段成品）

#### 场景：计划员拿到待精切料

**待精切车次信息：**

```
车次号：Y2601301/1-1
规格：1100mm宽 × 23000米长
质量：底部5800米针孔切二级品，Q620孔洞避留白，Q100切除
```

#### 计划员的决策过程

**步骤1：理解质量信息（长度方向！）**

```
质量记录："底部5800米针孔切二级品"

计划员的人工理解：
- 底部5800米 = 从卷的底部（起始端）往上5800米
- 针孔 = 缺陷类型
- 切二级品 = 可以用，但只能降级

画图理解（卷展开的侧视图）：
┌────────────────────────────────────────────┐
│  底部                           顶部        │
│  0m ────────────────────────→ 23000m      │
│  ▓▓▓▓▓▓▓│                                 │
│  缺陷区  │      好料区                     │
│  5800m  │                                 │
└─────────┴─────────────────────────────────┘
```

**步骤2：查看订单需求**

```
订单池中需要1100mm左右宽度的订单：
- 弘力：654mm × 6500-7000米
- 源元：860mm × 14000-16000米
```

**步骤3：长度分配决策（关键！）**

```
缺陷在0-5800米区间
订单A（弘力）：6500米
订单B（源元）：14000米

方案对比：
┌───────────────────────────────────────┐
│ 方案1：先切源元（14000米）             │
│  ▓▓▓▓▓▓▓│─────────────────────│       │
│  缺陷   │   源元的料           │ 余料  │
│  ❌ 缺陷会落在源元料里                 │
└───────────────────────────────────────┘

┌───────────────────────────────────────┐
│ 方案2：先切弘力（6500米）              │
│  ▓▓▓▓▓▓▓│────────────────────────────│
│  弘力料 │   源元的料（好料）           │
│  ✅ 缺陷被弘力消化，源元用好料         │
└───────────────────────────────────────┘

决策：先切弘力，后切源元
```

**步骤4：宽度调整**

```
车次宽度：1100mm
弘力要求：654mm → 需要精切
源元要求：860mm → 需要精切

精切后宽度：
- ① 1100mm → 654mm（切掉446mm）
- ② 1100mm → 860mm（切掉240mm）
```

**步骤5：在Excel中填写精切计划**

```
卷号：Y2601301/1-1
精切方案：
  ①精切:13*(654弘力*6500-7000m)
  ②精切:13*(860源元*14000-16000m)
备注：
  底部5800米针孔切二级品，两遍电晕
```

#### 精切执行

**操作工按计划执行：**

```
Y2601301/1-1（1100mm × 23000米）
    ↓
① 先切长度到6500米 + 宽度到654mm
   成品1：654mm × 6500米 → 弘力订单
    ↓
② 再切剩余的 + 宽度到860mm
   成品2：860mm × 16500米 → 源元订单
```

## 四、排产的核心难点分析

### 4.1 分切排产的难点

| 难点             | 说明                                         | 能否自动化        |
| ---------------- | -------------------------------------------- | ----------------- |
| **质量信息理解** | "Q840-990辊印透"这种自由文本，计划员需要理解 | ❌ 除非结构化录入 |
| **有效宽度计算** | 人工计算：1350 - (990-840) - 10 = 1190mm     | ✅ 如果质量结构化 |
| **订单匹配**     | 在100+订单中找宽度≤1190mm且合金匹配的        | ✅ 可以自动筛选   |
| **切刀方案设计** | 1190mm如何切？1刀还是2刀？废料最少？         | ⚠️ 可以推荐方案   |
| **多目标权衡**   | 交期vs废料vs客户优先级                       | ❌ 很难自动化     |

### 4.2 精切排产的难点

| 难点             | 说明                                     | 能否自动化        |
| ---------------- | ---------------------------------------- | ----------------- |
| **质量信息理解** | "底部5800米针孔"这种自由文本             | ❌ 除非结构化录入 |
| **缺陷区间识别** | 0-5800米是缺陷区                         | ✅ 如果质量结构化 |
| **订单匹配**     | 找合适长度的订单                         | ✅ 可以自动筛选   |
| **切割顺序决策** | 判断先切谁后切谁，让缺陷落在能接受的订单 | ⚠️ 可以推荐顺序   |
| **宽度调整计算** | 1100mm切成654mm和860mm                   | ✅ 可以自动计算   |

### 4.3 两者的共性

**共同的决策逻辑：**

```
1. 读取质量信息（自由文本）
   ↓
2. 人工理解缺陷含义
   ↓
3. 人工计算可用区域
   ↓  【分切：有效宽度】  【精切：可用长度】
4. 在订单池中人工查找匹配订单
   ↓
5. 人工设计方案
   ↓  【分切：切刀方案】  【精切：切割顺序】
6. 人工权衡多目标（交期、废料、客户）
```

**结论：分切和精切的复杂度相同，都是"质量约束下的订单匹配问题"**

## 五、智能排产的突破点

### 5.1 质量信息结构化（基础，必须先做）

**当前录入方式（自由文本）：**

```
"Q840-990辊印透，底部5800米针孔切二级品"
```

**结构化录入方案：**

```
缺陷记录1：
- 缺陷类型：辊印
- 位置方向：宽度方向
- 位置坐标：Q侧 840-990mm
- 处理方式：切除

缺陷记录2：
- 缺陷类型：针孔
- 位置方向：长度方向
- 位置坐标：底部0-5800米
- 处理方式：降级（可切二级品）
```

### 5.2 自动计算可用区域

**分切：有效宽度自动计算**

```
输入：
- 来料宽度：1350mm
- 缺陷位置：Q840-990mm
- 边丝宽度：10mm

系统自动计算：
有效宽度 = 1350 - (990-840) - 10 = 1190mm
```

**精切：可用长度区间识别**

```
输入：
- 来料长度：23000米
- 缺陷位置：底部0-5800米

系统自动识别：
缺陷区：0-5800米
好料区：5800-23000米
```

### 5.3 订单自动筛选

**分切订单筛选：**

```
输入：
- 有效宽度：1190mm
- 合金：1235D
- 厚度：13μm

系统自动筛选：
查询条件：宽度≤1190mm AND 合金=1235D AND 厚度=13μm
排序：按交期升序

输出：
订单1：XX客户，1100mm × 23000m，交期2月3日 ✅
订单2：YY客户，1000mm × 20000m，交期2月5日
...
```

**精切订单筛选：**

```
输入：
- 车次宽度：1100mm
- 可用长度：23000米
- 缺陷区：0-5800米

系统自动筛选：
1. 找需要≤1100mm宽度的订单
2. 按长度分组：
   - 短单（≤5800米）：可以消化缺陷
   - 长单（>5800米）：需要好料

输出推荐：
先切短单：弘力，654mm × 6500米 ← 消化缺陷
后切长单：源元，860mm × 14000米 ← 用好料
```

### 5.4 方案自动推荐

**分切：切刀方案推荐**

```
输入：有效宽度1190mm

系统计算多种方案：
方案A：1刀
- 成品：1100mm × 1条
- 废料：90mm
- 废料率：7.6%

方案B：2刀
- 成品：550mm × 2条
- 废料：90mm
- 废料率：7.6%

方案C：3刀
- 成品：370mm × 3条
- 废料：80mm
- 废料率：6.7%

推荐：方案A（匹配到1100mm订单，废料率可接受）
```

**精切：顺序推荐**

```
输入：
- 缺陷区：0-5800米
- 订单A：6500米
- 订单B：14000米

系统分析：
① 如果先切订单B（14000米）
   → 缺陷会落在订单B料里 ❌

② 如果先切订单A（6500米）
   → 缺陷被订单A消化 ✅
   → 剩余好料给订单B ✅

推荐顺序：①订单A → ②订单B
理由：让缺陷落在能接受的订单中
```

## 六、不可自动化的部分

以下决策必须由计划员人工完成：

| 决策类型       | 说明                   | 原因                     |
| -------------- | ---------------------- | ------------------------ |
| **多目标权衡** | 交期vs废料vs客户优先级 | 权重动态变化，无固定公式 |
| **让步放行**   | 与客户沟通降级使用     | 涉及商务谈判             |
| **改切降级**   | 原计划A客户，改给B客户 | 需要判断客户接受度       |
| **异常处理**   | 设备故障、质量突发问题 | 需要灵活应对             |
| **经验判断**   | 特殊案例的特殊处理     | 靠长期经验积累           |

## 七、系统定位建议

**智能排产系统应定位为：**

> "基于质量结构化数据的排产辅助决策系统"

**核心功能：**

1. ✅ 质量信息结构化录入
2. ✅ 自动计算有效宽度/长度
3. ✅ 自动筛选匹配订单
4. ✅ 推荐切刀方案/切割顺序

- 提供决策参考，由人工确认

## 八、总结

### 8.1 关键结论

1. **分切和精切的物理过程不同：**
   - 分切：纵向切宽度（宽卷→窄卷）
   - 精切：横向切长度（一车→多段）+ 可能再切宽度

2. **但决策逻辑完全相同：**
   - 都是"质量约束下的订单匹配问题"
   - 都涉及：理解缺陷 → 计算可用区域 → 匹配订单 → 设计方案

3. **智能排产的核心前提：**
   - **必须先做质量信息结构化**
   - 否则系统无法理解"Q840-990辊印透"的含义

4. **可突破的环节：**
   - ✅ 自动计算（有效宽度、可用长度）
   - ✅ 自动筛选（订单匹配）
   - ✅ 方案推荐（切刀方案、切割顺序）
5. **不可突破的环节：**
   - ❌ 多目标动态权衡
   - ❌ 让步放行、商务沟通
   - ❌ 经验性判断

### 8.2 对开发的启示

**如果要做分切/精切智能排产：**

**第一步（必须）：**

- 质量管理模块的结构化录入
- 包括：缺陷类型、位置（方向+坐标）、处理方式

**第二步（核心价值）：**

- 自动计算功能（有效宽度/长度）
- 订单筛选推荐
- 切刀方案/顺序推荐

**第三步（锦上添花）：**

- 废料率优化
- 历史案例推荐
- 多方案对比
