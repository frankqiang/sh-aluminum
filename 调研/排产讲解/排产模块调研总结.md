## 三、轧机排产逻辑【完整版】

### 3.1 设备能力约束

#### 3.1.1 轧机分布与功能

| 轧机类型   | 编号   | 数量 | 可轧道次 | 厚度范围   | 单道次时间 |
| ---------- | ------ | ---- | -------- | ---------- | ---------- |
| **粗轧机** | 1-3号  | 3台  | 1-3道次  | 220μm→38μm | 30min/道次 |
| **中轧机** | 4-6号  | 3台  | 4道次    | 38μm→13μm  | 1h/道次    |
| **中精轧** | 7号    | 1台  | 4-5道次  | 38μm→4.5μm | 1h或2h     |
| **精轧机** | 8-14号 | 7台  | 5道次    | 13μm→4.5μm | 2h/道次    |

> **特殊说明**：
>
> - 粗轧机1-3道次可连续轧制，无需换机
> - 7号中精轧机具备4-5道次双功能，若轧4道次需1小时，若轧5道次需2小时
> - 精轧机（8-14号）仅能轧成品道次（第5道次）

#### 3.1.2 道次分配规则

```
胚料卷 → [道次1] → [道次2] → [道次3] → [道次4] → [道次5] → 成品
             ↓           ↓           ↓           ↓           ↓
        粗轧机1-3号   粗轧机1-3号   粗轧机1-3号   中轧机4-6号   精轧机8-14号
         (可连轧)      (可连轧)      (可连轧)    或7号中精轧   或7号中精轧
```

**案例：胚料卷Y2601301的道次分配**

| 道次 | 厚度变化    | 分配设备          | 耗时   | 说明                             |
| ---- | ----------- | ----------------- | ------ | -------------------------------- |
| 1    | 220μm→110μm | 1号粗轧机         | 30分钟 | 1-3道次可在同一台粗轧机连续轧制  |
| 2    | 110μm→55μm  | 1号粗轧机（连轧） | 30分钟 | 无需换机，直接轧下一道次         |
| 3    | 55μm→38μm   | 1号粗轧机（连轧） | 30分钟 | 粗轧完成，下料转中轧             |
| 4    | 38μm→13μm   | 4号中轧机         | 1小时  | 中轧道次                         |
| 5    | 13μm→13μm   | 13号精轧机        | 2小时  | 成品道次，保持厚度，优化表面质量 |

### 3.2 产能平衡约束

#### 3.2.1 核心原则

> **目标**：保证每个机台都不停机，让设备最大化生产。

| 约束类型     | 具体要求                               | 说明                       |
| ------------ | -------------------------------------- | -------------------------- |
| **提前备料** | 前道工序提前**1个班（8小时）**完成备料 | 确保后道工序有料可轧       |
| **缓冲库存** | 道次之间保持**2卷以上**的缓冲库存      | 允许一定容差，避免停机等料 |
| **产能梯度** | 前道产能 > 后道产能，自然形成库存积累  | 粗轧5卷/班 > 中轧4卷/班    |

#### 3.2.2 产能计算

| 设备类型 | 单卷耗时               | 班产能（8小时） | 说明                |
| -------- | ---------------------- | --------------- | ------------------- |
| 粗轧机   | 1.5小时（3道次×30min） | 约5卷           | 1-3道次连轧         |
| 中轧机   | 2小时（加换卷时间）    | 约4卷           | 含道次4             |
| 精轧机   | 2.5小时（加换卷时间）  | 约3卷           | 含道次5（成品道次） |

**产能平衡案例**：

- 粗轧班产能5卷 > 中轧班产能4卷 → 每班自然富余1卷
- 时间一长，缓冲库存自然积累
- 计划员需监控缓冲库存，避免过度积压

#### 3.2.3 设备故障处理

```
发现故障 → 评估影响 → 转移到同功能设备 → 调整计划顺序
              ↓
      例：2号粗轧机故障
              ↓
      将计划转移到1号或3号粗轧机
              ↓
      粗轧机只排粗轧机，中轧机只排中轧机
```

### 3.3 轧机计划"说明"列解析

#### 3.3.1 说明列内容类型

轧机计划表的"说明"列包含**多种类型信息**，不仅仅是轧制方案：

| 信息类型         | 示例                                           | 来源     |
| ---------------- | ---------------------------------------------- | -------- |
| **工艺指标要求** | "厚度不超6 针孔≤1000"                          | 订单要求 |
| **厚度变更**     | "5.3改轧7 针孔＜80"                            | 计划调整 |
| **质量控制指令** | "无黑线，导辊及熨平辊擦干净，严格控制带油"     | 订单要求 |
| **操作注意事项** | "轧制中注意板型及表面质量有问题及时反馈赵士畅" | 生产管理 |
| **缺陷关注点**   | "注意暗面亮点 人字纹"                          | 订单要求 |
| **多重参数要求** | "除油A+ 油温≥45° 针孔≤1000 厚度不超5.5"        | 订单要求 |
| **分卷参考**     | "24000M/车"                                    | 订单要求 |

#### 3.3.2 分卷参考说明

"说明"列中的"24000M/车"含义：

- **不是**该卷轧完的总长度
- **而是**分卷的参考米数，方便轧机操作工知道在多少米时可以停机检查
- 来源于订单要求的每车米数
- 轧机停机时自然打断，不需要切割

> **设计建议**：此字段作为参考信息，现场人员根据经验已熟知，可选填。

## 四、分切排产逻辑

### 4.1 设备能力约束

| 设备分类       | 编号           | 数量 | 检测设备  | 可切产品类型   |
| -------------- | -------------- | ---- | --------- | -------------- |
| **有检测设备** | 1#、2#、4#、5# | 8台  | 针检+表检 | 电池箔、双零箔 |
| **无检测设备** | 3#、6#         | 6台  | 无        | 仅双零箔       |

> **说明**：一期6台 + 二期8台 = 14台分切机，设备效率基本一致。

### 4.2 订单匹配逻辑

#### 4.2.1 匹配原则（按优先级排序）

| 优先级 | 匹配原则                | 说明                                       |
| ------ | ----------------------- | ------------------------------------------ |
| 1      | **交期优先**            | 交货日期越近，优先级越高                   |
| 2      | **大客户/重要客户优先** | 重要客户订单优先安排                       |
| 3      | **规格集中**            | 尽量将相近规格的订单集中切割，减少换刀次数 |
| 4      | **减少废料**            | 边丝/余料最小化，边丝标准8-15mm            |
| 5      | **先进先出**            | 同等条件下，先来的订单先排                 |

#### 4.2.2 匹配约束

| 约束类型     | 具体要求                                     |
| ------------ | -------------------------------------------- |
| **宽度约束** | 来料宽度 ≥ 订单要求宽度总和 + 边丝(16-30mm)  |
| **合金匹配** | 来料合金必须与订单要求的合金一致             |
| **规格匹配** | 厚度、状态等必须满足订单要求                 |
| **质量匹配** | 来料质量等级 ≥ 订单要求等级（一级品/二级品） |

#### 4.2.3 订单池核心字段

> 数据来源：MES系统导出的`订单池.xlsx`，共44列字段

| 字段分类     | 字段名              | 说明                                 |
| ------------ | ------------------- | ------------------------------------ |
| **订单标识** | NC执行单号          | 主键，与分切计划的"执行单号"列对应   |
|              | 合同号              | ERP合同编号                          |
| **客户信息** | 客户                | 客户名称                             |
| **产品规格** | 产品                | 产品类型（电池箔成品/食品箔/母卷等） |
|              | 规格型号            | 厚度(mm)\*宽度(mm)                   |
|              | 合金                | 合金代号（决定电池箔/双零箔）        |
|              | 状态                | 退火状态码（O=退火态，H18=硬态）     |
|              | 内径                | 管芯内径（76mm=3寸，152.4mm=6寸）    |
| **数量交期** | 卷数                | 订单要求卷数                         |
|              | 重量                | 订单重量(kg)                         |
|              | **交货日期**        | **订单优先级的核心依据**             |
|              | 外径范围/长度公差   | ⚠️同列语义不同，按产品类型区分       |
| **执行跟踪** | 已分切卷数/未切卷数 | 订单执行进度                         |

### 4.3 分切计划"说明"列解析

#### 4.3.1 说明列语法

分切计划的"说明"列采用**多行结构**，包含不同类型信息：

```
第1行：序号、规格方案（订单要求）
       格式：序号、厚度×宽度产品类型×米数范围
       示例：1、13×1340电池箔×23000-24000

第2行：管芯规格等附加要求（订单要求）
       示例：用152.4管芯/壁厚4管芯  管芯长度1605mm

第3行：计划要求
       示例：合理计算米数尽量不废料  装框单备注管芯壁厚

第4行：质量评审信息
       示例：Q554 C485 来料白条，评审可切下料复查
```

> **注意**：以上顺序不是绝对的，但订单要求一般放在第一行。

#### 4.3.2 规格方案解析

以 `1、13×1340电池箔×23000-24000` 为例：

| 组成部分 | 值          | 说明                     |
| -------- | ----------- | ------------------------ |
| 序号     | 1、         | 第1个分切方案            |
| 厚度     | 13          | 13μm                     |
| 宽度     | 1340        | 1340mm                   |
| 产品类型 | 电池箔      | 产品类型                 |
| 米数范围 | 23000-24000 | 每车米数范围（订单要求） |

### 4.4 切刀方案设计

| 配置项       | 参数值     | 说明                               |
| ------------ | ---------- | ---------------------------------- |
| **最大刀数** | 7-8刀      | 精切可能8刀，分切一般不超过7刀     |
| **常规刀数** | 1-2刀      | 最普遍的情况，切5-8刀是特殊情况    |
| **边丝宽度** | 8-15mm     | 两边各切8-15mm边丝                 |
| **废料处理** | 当废料处理 | 余料超过订单需求宽度时，作废料处理 |

#### 4.4.1 多订单搭配逻辑

当一卷料切给多个订单时，需要考虑米数搭配：

```
案例：
- 订单A：宽度500mm，米数要求12000m
- 订单B：宽度800mm，米数要求24000m

处理方式：
1. 按高要求（24000m）切割
2. 订单A的500mm宽度切出24000m
3. 再用薄剪仪将其切成两个12000m
```

### 4.5 车次号规则

分切后的子卷采用**车次号**标识，格式为：`母卷号/X-Y`

| 组成部分 | 示例       | 说明                       |
| -------- | ---------- | -------------------------- |
| 母卷号   | Y2412380   | 轧机成品的卷号             |
| /        | /          | 分隔符                     |
| X        | 1、2、3... | 第几车（分切时的小卷序号） |
| -Y       | -1         | ⏳待确认（Y的具体含义）    |

> **待确认问题**：车次号中Y的含义？是否会出现1-2、2-2等情况？

## 五、精切排产逻辑

### 5.1 设备概况

| 设备类型 | 数量 | 功能                       |
| -------- | ---- | -------------------------- |
| 精切机   | 49台 | 切宽度+切米数              |

> **说明**：精切机能力基本一致，无明显差异。

### 5.2 精切方案语法

#### 5.2.1 完整格式

精切计划包含两列核心信息：

| 列名     | 内容类型      | 示例                             |
| -------- | ------------- | -------------------------------- |
| 精切方案 | 切割方案      | ①精切:13*（654弘力*6500-7000m）  |
| 备注     | 质量评审+要求 | 底部5800米针孔切二级品，两遍电晕 |

#### 5.2.2 精切方案解析

以 `①精切:13*（654弘力*6500-7000m）` 为例：

| 组成部分 | 值         | 说明          |
| -------- | ---------- | ------------- |
| 序号     | ①          | 第1个精切方案 |
| 关键字   | 精切:      | 标识精切操作  |
| 厚度     | 13         | 13μm          |
| 宽度     | 654        | 654mm         |
| 客户     | 弘力       | 客户名称      |
| 米数范围 | 6500-7000m | 米数要求      |

#### 5.2.3 多方案顺序逻辑【重要】

当一个子卷需要切给多个订单时，①②的顺序**不是**简单的质量要求高低，而是基于**缺陷位置与订单长度的匹配关系**：

**案例说明**：

```
分切卷信息：
- 卷号：Y2412380/1-1
- 规格：13μm×1200mm×23000m

质量评审信息：
- "底部5800米针孔切二级品，Q620孔洞避留白，Q100针孔切除，C25针孔切除"
- 关键信息：底部5800米有针孔缺陷

待匹配订单：
- 弘力客户：要求长度6500-7000m
- 源元客户：要求长度14000-16000m

排序逻辑：
1. 缺陷在底部5800米，弘力要求的6500-7000m 刚好能覆盖这个缺陷区域
2. 源元要求的14000-16000m 远大于缺陷长度，若先切源元，缺陷会落在源元的料里
3. 所以先切弘力（①）把有缺陷的部分消化掉，再切源元（②）用剩余的好料

最终精切方案：
- ①精切:13*（654弘力*6500-7000m） ← 先切，消化缺陷
- ②精切:13*（860源元*14000-16000m） ← 后切，用好料
```

> **总结**：①②顺序的核心是**让缺陷区域落在能接受该缺陷的订单中**，同时保证其他订单拿到好料。

### 5.3 电晕处理

| 配置项         | 说明                                     |
| -------------- | ---------------------------------------- |
| **达因值要求** | ≥33（客户要求）                          |
| **电晕方式**   | 单遍或两遍（"两遍电晕"表示过两次电晕机） |
| **破卷含义**   | 破卷 = 电晕，可忽略独立考虑              |

## 七、智能排产可行性评估【核心章节】

> **本章目的**：客观评估"智能排产"的可实现程度，明确系统边界，避免对客户过度承诺。

### 7.1 目前无法实现"全自动智能排产"的核心原因

#### 7.1.1 障碍一：质量信息非结构化——程序无法理解"怎么切"

**问题本质**：

当前质量评审信息以**自由文本**形式记录在Excel中，格式不统一，程序无法自动解析。而智能排产的核心前提是：**程序能理解缺陷信息，计算切除方案，匹配合适订单**。

**具体表现**：

| 信息类型     | 实际记录示例                                       | 程序需要理解的内容                   | 能否自动解析 |
| ------------ | -------------------------------------------------- | ------------------------------------ | ------------ |
| **缺陷位置** | "Q860辊印印过，L985"                               | Q侧860mm处有辊印，周期985mm重复出现  | ❌ 无法解析  |
| **切除范围** | "Q840-990针孔切除"                                 | 从Q侧840mm到990mm需要切掉（共150mm） | ❌ 无法解析  |
| **处理方式** | "正常分切，精切辊印避留白"                         | 分切正常进行，精切时在辊印处留白     | ❌ 无法解析  |
| **降级判定** | "只能切二级品"                                     | 整卷降级，需匹配能接受二级品的客户   | ❌ 无法解析  |
| **混合情况** | "底部5800米针孔切二级品，Q620孔洞避留白，Q100切除" | 底部5800m降级 + Q620留白 + Q100切掉  | ❌ 无法解析  |

**为什么这是致命障碍？**

智能排产需要完成以下自动化链条，但每一步都被质量信息的非结构化阻断：

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                           智能排产的理想自动化链条                                        │
├─────────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                          │
│  1️⃣ 读取质量信息 → 2️⃣ 解析缺陷位置 → 3️⃣ 计算有效宽度 → 4️⃣ 匹配订单 → 5️⃣ 生成切刀方案    │
│         ↓                 ↓                 ↓               ↓               ↓           │
│   "Q840-990切除"    Q侧840-990mm      1350-150=1200mm   找≤1200mm订单   切除+边丝+订单   │
│                                                                                          │
│  ─────────────────────────────────────────────────────────────────────────────────────  │
│                                                                                          │
│  ⚠️ 当前状态：第1步就卡住了！                                                            │
│                                                                                          │
│  - 质量信息是自由文本，格式不统一                                                         │
│  - "Q840-990"和"Q侧840mm到990mm"程序无法识别为同一含义                                   │
│  - "避留白"、"辊印透"等术语没有标准化定义                                                │
│  - 多个缺陷混写在一起，无法拆分处理                                                       │
│                                                                                          │
└─────────────────────────────────────────────────────────────────────────────────────────┘
```

**计划员是怎么做的？**

计划员看到"Q840-990针孔切除"能立刻在脑中完成以下计算：

1. 这卷料原始宽度1350mm
2. Q侧840-990mm共150mm需要切掉
3. 有效宽度剩余0-840mm = 840mm
4. 脑中搜索订单池，找到宽度≤840mm的订单
5. 考虑边丝8mm×2=16mm，实际可切宽度824mm
6. 匹配到一个800mm宽度的订单，确定切刀方案

**程序无法做到的原因**：

- 无法从"Q840-990"提取出840和990这两个数值
- 无法理解"Q侧"是驱动侧，坐标从驱动侧边缘算起
- 无法将"切除"翻译为"这个范围的宽度不可用"
- 遇到"避留白"不知道应该怎么处理

---

#### 7.1.2 障碍二：决策逻辑在计划员脑中——难以完全程序化

**计划员原话**：

> "你问的问题对于不了解的人来说问很正常，但是让一个天天做计划的人就说不出来。这个料我做的时候就已经对应上订单了，他只要不出质量问题，那就是切这个订单了呀。"

**隐藏在"脑子里"的决策逻辑清单**：

| 决策环节             | 计划员脑中的隐性逻辑                                                    | 能否程序化    |
| -------------------- | ----------------------------------------------------------------------- | ------------- |
| **来料-订单匹配**    | 看到1350mm宽的料，立刻想到"手头有几个1300左右的订单可以做"              | ⚠️ 部分可以   |
| **多订单搭配**       | 1350mm的料可以切1个1300+1个边丝，或者2个650并排，哪种废料少？           | ⚠️ 部分可以   |
| **质量-订单匹配**    | 这卷有辊印，A客户要求高不能用，B客户可以接受轻微辊印                    | ❌ 困难       |


**关键洞察**：

这些决策逻辑有以下共同特点：

1. **非结构化**：没有写在任何文档或系统里
2. **经验积累**：靠长期工作积累的"感觉"
3. **情境依赖**：同样的情况在不同时间可能有不同处理
4. **人际信息**：来自电话、微信、口头沟通的信息

---

#### 7.1.3 障碍三：多目标动态权衡——权重无法固化

**排产决策需要平衡的多个目标**：

| 目标       | 含义               | 量化方式 | 目标间的冲突示例                     |
| ---------- | ------------------ | -------- | ------------------------------------ |
| **保交期** | 订单按时交付       | 交期天数 | 为赶A订单交期，B订单被延后           |
| **保客户** | 重要客户优先       | 客户等级 | 大客户订单插队，小客户被排后面       |
| **减废料** | 边丝和余料最小化   | 废料重量 | 为凑整刀减少废料，可能延迟某些订单   |
| **保质量** | 避免质量降级       | 一级品率 | 高质量要求的订单等好料，低要求的先做 |
| **保产能** | 设备利用率最大化   | 开机率   | 为凑满一炉退火，可能打乱分切顺序     |
| **减换刀** | 同规格订单集中切割 | 换刀次数 | 规格集中可能打乱交期顺序             |

**动态权衡的具体表现**：

```
场景1：月底冲产量
─────────────────────────────────────────────────────
权重调整：保产能 > 保交期 > 减废料
决策表现：能做的订单先做，不纠结最优匹配
计划员原话："月底就是赶量，先把能出的出了"

场景2：重要客户催单
─────────────────────────────────────────────────────
权重调整：保客户 > 保交期 > 其他
决策表现：重要客户订单插队，其他订单让路
计划员原话："河北兴恒的料不能拖，人家催了好几次了"

场景3：来料质量普遍不好
─────────────────────────────────────────────────────
权重调整：保质量 > 保交期 > 保客户
决策表现：高要求订单等好料，低要求订单先消化问题料
计划员原话："这批料都有点问题，先切要求低的，好料留给重要订单"

场景4：月初订单不紧张
─────────────────────────────────────────────────────
权重调整：减废料 > 减换刀 > 保交期
决策表现：精细优化切刀方案，多花时间凑最优搭配
计划员原话："今天不急，好好算算怎么切废料少"
```

**为什么程序无法处理？**

1. **权重没有固定公式**：不是"交期占60%+客户占30%+废料占10%"这种固定比例
2. **触发条件难以定义**：什么情况下启用"月底冲量模式"？靠人的判断
3. **信息来源分散**：月底压力来自销售、客户来自业务、质量来自检验——系统难以汇总
4. **反馈不闭环**：排产决策的好坏没有量化评估，全靠经验感觉

---

#### 7.1.4 障碍四：物料交接无追踪——不知道料在哪里

**现状描述**：

待确认问题中明确提到：

> "约1000个卷堆在现场，精切人员怎么知道哪些料可以切了？"

**影响分析**：

| 环节           | 现状                              | 对智能排产的影响                         |
| -------------- | --------------------------------- | ---------------------------------------- |
| **轧机→分切**  | 下料后放在指定位置，位置写在Excel | 系统不知道料的实时位置，无法判断是否可用 |
| **分切→精切**  | 分切完放在地上，精切人员自己找    | 1000卷堆在一起，系统不知道哪卷在哪里     |
| **质量评审后** | 评审完更新Excel，人工通知计划员   | 系统无法实时获取评审结果                 |

**底层问题**：

智能排产需要回答"现在有什么料可以排"，但系统不知道：

- 哪些料已经轧完在等分切？
- 哪些料已经分切完在等精切？
- 哪些料在评审中？评审结果是什么？
- 哪些料已经被排了计划正在执行？

---

### 7.3 系统定位建议

> **智能排产模块应定位为"辅助决策工具"，而非"自动驾驶系统"。**

| 定位         | ✅ 正确表述                          | ❌ 错误表述                  |
| ------------ | ------------------------------------ | ---------------------------- |
| **功能定位** | 帮助计划员更高效地完成排产           | 自动完成排产，不需要计划员   |
| **决策角色** | 系统提供信息和建议，计划员做最终决策 | 系统自动决策，计划员只需确认 |
| **价值主张** | 减少查找时间、减少计算错误、规范流程 | 智能优化、AI排产、无人化     |
| **对外沟通** | "智能辅助排产系统"                   | "全自动智能排产系统"         |

## 八、系统设计建议【方案B】

### 8.1 数据录入方案

采用**方案B：结构化字段 + 自由备注**，兼顾规范性和灵活性：

| 类别         | 结构化字段                                           | 自由备注字段 |
| ------------ | ---------------------------------------------------- | ------------ |
| **质量缺陷** | 缺陷类型（下拉）、缺陷位置（C/Q侧+起止mm）、处理方式 | 其他说明     |
| **订单信息** | 厚度、宽度、长度范围、客户名、产品类型               | 特殊备注     |
| **订单要求** | 管芯规格、壁厚、管芯长度                             | 其他要求     |
| **计划要求** | 电晕要求、废料控制                                   | 计划备注     |
| **质量评审** | 评审结论（下拉）、评审人、评审时间                   | 评审说明     |

### 8.2 建议的结构化字段

#### 8.2.1 缺陷类型（下拉选项）

```
- 针孔
- 辊印
- 孔洞
- 黑线
- 边部不良
- 表面异物
- 气道
- 其他
```

