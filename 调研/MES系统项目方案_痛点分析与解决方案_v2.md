## 痛点分析与解决方案

> **文档版本**：V2.0
> **编制日期**：2026年3月
> **文档目的**：基于两轮现场调研，梳理当前生产管理痛点，提出MES系统解决方案
> **更新说明**：整合排产模块、质量模块两轮深入调研成果，补充系统定位、分期策略、实操方案

# 第一章：项目背景

## 1.1 企业现状

贵公司是一家专业生产各类铝箔卷（电池箔、双零箔等）的制造企业，拥有：

- 轧机：14台（3台粗轧+3台中轧+1台中精轧+7台精轧）
- 分切机：14台（其中1#/2#/4#/5#共4台配备在线针检/表检设备，3#/6#无设备靠人工目检）
- 精切机：49台
- 合卷机：2台（双零箔专用）
- 退火炉：60台

目前生产管理主要依赖**Excel表格**和**人工协调**，随着产能扩大和客户质量要求提升，现有管理方式面临较大压力。

## 1.2 调研范围

本次调研重点覆盖以下环节：

```
胚料出库 → 轧机（多道次）→ 分切 → [退火] → 精切 → 包装
              ↓                ↓                ↓
           轧机质检        分切质检          精切质检
```

通过与计划员、质量工程师、质检员的**四轮深入沟通**（排产两轮+质量两轮），我们识别出当前生产管理的**4大核心痛点**。

## 1.3 系统定位

> **⚠️ 重要声明：本系统定位为"辅助决策工具"，而非"全自动系统"。**

| 定位         | ✅ 我们要做的                                  | ❌ 我们不做的                  |
| ------------ | ---------------------------------------------- | ------------------------------ |
| **功能定位** | 帮助计划员和质检员更高效地完成工作             | 自动排产、自动评审，不需要人   |
| **决策角色** | 系统提供信息、数据和建议，人做最终决策         | 系统自动决策，人只需点"确认"   |
| **核心价值** | 数据电子化、流程规范化、信息实时同步、辅助决策 | AI智能判定、全自动排产、无人化 |
| **对外定位** | "智能辅助生产管理系统"                         | "全自动智能排产/评审系统"      |

**为什么这样定位？**

经过深入调研，我们发现排产和质量评审的核心决策都依赖工人的**经验判断**，且规则随客户、订单、时间动态变化，短期内无法完全程序化。系统的最大价值在于**解决信息孤岛和流程断链**，而非替代人的判断。

# 第二章：现状痛点分析

## 2.1 痛点总览

| 痛点                            | 现状描述                                                               | 影响                                                   |
| ------------------------------- | ---------------------------------------------------------------------- | ------------------------------------------------------ |
| **1. 计划管理分散**             | 32个Excel Sheet分散管理轧机/分切/精切计划，无汇总视图                  | 计划员在Sheet间切换、信息不同步、难以把握整体进度      |
| **2. 质量评审不规范且传递断链** | 评审信息采用自由文本描述不统一，且需手动复制传递，包装靠共享文件夹获取 | 难以统计分析、评审信息滞后、漏看、计划可能先于评审执行 |
| **3. 在制品追踪困难**           | 约1000个分切子卷堆在现场，靠人工找料，位置信息靠记忆/手写              | 精切人员找料困难、在制品积压无法量化、呆滞料不可见     |
| **4. 订单进度不透明**           | 订单无状态跟踪，多卷满足一个订单时靠人工统计、逾期无预警               | 交期管理困难、逾期风险高、客户投诉                     |

---

## 2.2 痛点1：计划管理分散

### 现状描述

当前生产计划管理存在以下问题：

1. **计划表分散**：轧机/分切/精切计划分布在32个Excel Sheet中（每台设备一个Sheet）
2. **无汇总视图**：计划员需要在Sheet之间频繁切换，难以一眼把握整体进度
3. **信息不同步**：轧机下料后，分切计划需要手动更新可用料清单
4. **版本混乱**：计划修改后无历史记录，出问题时难以追溯
5. **日期靠颜色区分**：计划表中用**背景颜色**区分今天/已完成/未来计划（白色=今天、灰色上方=已完成、灰色下方=未来），无法按日期筛选历史

### 具体表现

- 计划员每天需要在32个Sheet间切换查看各机台进度
- 轧机完成后，需手动将信息复制到"轧制跨在制料表"供分切参考
- 计划调整后，其他人不知道改了什么、为什么改
- 计划通过**共享文件夹+纸质打印**下发到现场，中班/夜班靠白班提前排好

### 排产逻辑现状

计划员排产涉及**多目标动态权衡**，核心是"心算"：

| 目标       | 含义               | 冲突示例                             |
| ---------- | ------------------ | ------------------------------------ |
| **保交期** | 订单按时交付       | 为赶A订单交期，B订单被延后           |
| **保客户** | 重要客户优先       | 大客户订单插队，小客户被排后面       |
| **减废料** | 边丝和余料最小化   | 为凑整刀减少废料，可能延迟某些订单   |
| **保质量** | 避免质量降级       | 高质量要求的订单等好料，低要求的先做 |
| **保产能** | 设备利用率最大化   | 为凑满一炉退火，可能打乱分切顺序     |
| **减换刀** | 同规格订单集中切割 | 规格集中可能打乱交期顺序             |

> **关键发现**：计划员脑中的决策权重不是固定的，会随月底冲量、客户催单、来料质量等**场景动态调整**。系统应定位为**辅助计算和信息展示**，而非替代决策。

### 影响

- 计划员工作量大，容易遗漏
- 信息同步滞后，可能出现"轧机已下料但分切不知道"的情况
- 出问题时无法追溯计划调整记录

## 2.3 痛点2：质量评审信息不规范且传递断链

### 现状描述

质量评审信息存在以下问题：

1. **描述不规范**：评审信息采用自由文本填写，不同人员对同一类型缺陷的描述方式不一致，难以统计分析
2. **手动复制**：质量部门将评审结论写在质量报表Excel中，计划员需手动复制到分切计划
3. **可能滞后**：存在"料已到分切工位，但评审还没出"的情况
4. **传递链长**：评审信息从质量部门→计划员→分切→精切→包装，每个环节都靠人工传递
5. **容易漏看**：包装工序通过共享文件夹获取信息，容易遗漏

### 具体表现

**描述不规范的例子**：

| 同一类型缺陷 | 不同人员的描述方式                                                     |
| ------------ | ---------------------------------------------------------------------- |
| Q侧针孔问题  | "Q侧890-990针孔切除"、"Q890-990针孔密集需切"、"Q侧890mm处有针孔要切掉" |
| 边部辊印     | "C侧辊印避白"、"C边辊印要避开"、"C25辊印切除"                          |

**评审信息结构复杂**（基于2315条评审信息分析）：

一条评审信息可能包含多种组成部分：

```
示例1：让步放行，包装备注：Q0-80针孔切除，C0-40针孔切除
       ├─主结论─┤ ├─传递对象─┤ ├───处理指令1───┤ ├───处理指令2───┤

示例2：转精切，0-11000米切二级品，C0-30针孔切除
       ├主结论┤ ├─处理指令1（长度）─┤ ├─处理指令2（宽度）─┤
```

**传递断链的表现**：

- 质量评审结果写在Excel的"备注"列，计划员需手动复制到分切计划的"说明"列
- 包装需要到共享文件夹查看评审信息，确认是否需要"避留白"
- 评审完成后通过**共享文件夹、口头、微信**通知计划员，无统一渠道

### 评审组织结构

```
质量主管
    ↓
质量工程师（白班6人）—— 降级、让步放行、报废必须由工程师决定
    ↓
当班班长 —— 可评审当班所有料卷
    ↓
质检员（每工序每班约4人）—— 经验丰富者可评审本机台
```

- 第二天白班质量工程师会"全部重新评审一遍"（二次评审）
- 以前不合格的必须等白班工程师处理，现在核心质检人员和班长也可评审

### 检测设备数据采集可行性

| 设备       | 设备能输出的内容          | 实际录入Excel的内容       | 需人工判断？ | 自动采集价值 |
| ---------- | ------------------------- | ------------------------- | ------------ | ------------ |
| **针检仪** | A/B/C/D/E级针孔数量、密度 | A/B/C/D/E级针孔数量       | 部分是       | ⭐⭐⭐⭐ 高  |
| **表检仪** | 几十上百个缺陷详情        | 人工总结的关键缺陷描述    | **是**       | ⭐ 低        |
| **板型仪** | 下榻量数值、曲线          | 下榻量（B/C/D等级已取消） | 否           | ⭐⭐⭐ 中    |

> **结论**：针孔数量和下榻量可自动采集，表面检测信息必须人工总结（从几百个缺陷挑选关键问题），一期以手工录入为主，三期再考虑设备对接。

### 影响

- **难以统计分析**：同一类型缺陷描述不一致，无法统计"这个月针孔问题有多少"
- **理解歧义**：新员工或其他工序人员可能误解评审意图
- 评审信息可能滞后或漏传
- 存在"**未评审先生产**"的风险——客户明确要求系统卡住这一点
- 包装环节可能漏看重要的质量备注

## 2.4 痛点3：在制品追踪困难

### 现状描述

在制品管理存在以下问题：

1. **找料困难**：分切完成后约1000个子卷堆在现场，精切人员需要在大量料卷中找到目标料卷
2. **位置信息模糊**：有料框但**没有库位概念**，料卷放置比较随意
3. **管理差异大**：轧机→分切有"轧制跨在制料表"管理，分切→精切**完全没有**管理表
4. **状态不可见**：无法实时知道"现在有多少料等待精切""哪些料已经等了很久"
5. **呆滞料隐藏**：长期未处理的料卷可能被遗忘，无人管理

### 各环节在制品管理现状

| 环节      | 管理方式           | 位置记录                   | 状态可见性 |
| --------- | ------------------ | -------------------------- | ---------- |
| 轧机→分切 | 有"轧制跨在制料表" | 每台轧机有**固定备料区域** | **部分**   |
| 分切→精切 | **无**管理表       | 料框号，**无库位概念**     | **不可见** |
| 分切→退火 | 无管理表           | 无结构化记录               | **不可见** |

### 具体表现

- 精切人员拿到计划后，需要在现场凭经验寻找对应的料卷
- 有时"料明明在那里，但找了半天才找到"
- 部分料卷可能因为各种原因长期滞留现场，无人处理
- 分切下料后放入料框，料框号靠手写或口头传递

### 现场环境约束

- **噪音大**：车间噪音水平高
- **油污多**：铝箔轧制过程中使用轧制油
- **工人戴手套**：操作时无法频繁使用键盘
- **料卷放置随意**：没有固定库位制度

### 影响

- 找料时间长，影响生产效率
- 在制品积压情况不透明
- 呆滞料占用场地资源

---

## 2.5 痛点4：订单进度不透明

### 现状描述

订单管理存在以下问题：

1. **无状态跟踪**：订单在MES订单池中没有明确的状态标记（待排产/生产中/已完成）
2. **人工统计**：一个订单需要多卷料满足时，靠人工统计已完成多少
3. **无逾期预警**：交期临近时才发现可能赶不上，无提前预警机制（计划员认为提前3天预警**有帮助**）
4. **重排困难**：质量问题导致订单需要重新匹配料卷时，进度更难追踪
5. **实际遗漏**：确认存在"以为订单完成了，实际还差一卷"的情况

### 具体表现

- 订单优先级主要靠计划员记忆和经验判断
- 客户问"我的订单做到哪了"，需要翻多张Excel表统计
- 某个订单差一卷没做完，可能直到发货时才发现
- 交期优先级：日期越近优先级越高，逾期后销售不取消就接着干

### 影响

- 交期管理被动，逾期风险高
- 客户询问时无法快速响应
- 多卷订单容易遗漏

# 第三章：解决方案

## 3.1 解决方案概述

针对上述4大痛点，我们提出MES系统解决方案，包含以下核心模块：

| 痛点             | 对应模块         | 核心价值                             |
| ---------------- | ---------------- | ------------------------------------ |
| 计划管理分散     | **排产模块**     | 统一平台管理所有计划，提供生产看板   |
| 质量评审传递断链 | **质量模块**     | 评审信息结构化录入，自动同步到各环节 |
| 在制品追踪困难   | **物料管理模块** | 在制品实时可视，位置可查，呆滞预警   |
| 订单进度不透明   | **生产执行模块** | 订单状态实时跟踪，交期预警           |

### 设计原则

| 原则             | 说明                                                       |
| ---------------- | ---------------------------------------------------------- |
| **减少人工录入** | 能扫码的不手工输入，能联动的不重复录入                     |
| **现场环境适配** | 大字、大按钮、扫码为主，适应噪音大/油污多/戴手套的车间环境 |
| **渐进式实施**   | 分三期实施，先解决最痛的信息孤岛问题                       |
| **兼容纸质习惯** | 支持打印电子版计划，过渡期保留纸质流程                     |
| **辅助而非替代** | 系统提供数据和建议，最终决策由人做出                       |

## 3.2 排产模块：解决计划管理分散

### 功能设计

| 功能             | 说明                                                 | 价值                               | 实施阶段 |
| ---------------- | ---------------------------------------------------- | ---------------------------------- | -------- |
| **统一排产平台** | 轧机/分切/精切三层计划在同一系统中管理               | 取代32个Sheet，一个界面全览        | 一期     |
| **生产看板**     | 实时显示各机台的计划进度、在制数量                   | 一眼看清整体状况                   | 一期     |
| **计划联动**     | 轧机完成自动更新可分切料清单（取代"轧制跨在制料表"） | 减少手动复制，信息实时同步         | 一期     |
| **修改留痕**     | 所有计划调整保留修改记录（谁、什么时间、改了什么）   | 可追溯谁改了什么                   | 一期     |
| **计划打印**     | 支持将系统中的计划按机台导出/打印                    | 兼容现有纸质下发习惯               | 一期     |
| **日期化管理**   | 用日期字段替代颜色区分，支持按日期筛选历史计划       | 取代"白色=今天、灰色=已完成"       | 一期     |
| **订单匹配推荐** | 录入来料规格后，自动列出宽度/厚度/合金匹配的订单列表 | 辅助计划员查找，减少心算           | 二期     |
| **切刀方案计算** | 输入切刀位置后，自动计算边丝、余料、废料率           | 辅助计算，减少手工笔算错误         | 二期     |
| **质量匹配建议** | 根据结构化的缺陷位置计算有效宽度，筛选可匹配订单     | 辅助决策（需质量模块结构化先上线） | 三期     |

### 分切方案支持

系统需要理解分切方案的**切刀语法**：

```
语法格式：厚度×宽度产品类型×米数范围
示    例：13×1340电池箔×23000-24000
含    义：厚度13μm、宽度1340mm的电池箔、长度23000~24000米
```

系统通过结构化字段录入替代这种语法，降低理解难度。

### 切刀方案与废料计算说明

#### 什么是切刀方案？

分切工序的核心任务是把一卷宽幅母卷（如1350mm宽）按照订单需求切成若干窄幅子卷。**切刀方案**就是"怎么切、切几刀、每刀切多宽"的计划。

```
母卷横截面示意（从上方俯视）：

←────────────── 母卷宽度 1350mm ──────────────→

┌──┬────────────────┬────────────────┬──┐
│边│                │                │边│
│丝│   订单A: 650mm │   订单B: 650mm │丝│
│  │                │                │  │
│10│                │                │10│
│mm│                │                │mm│
└──┴────────────────┴────────────────┴──┘
 ↑        ↑                ↑          ↑
 切       切               切          切
 刀       刀               刀          刀
 1        2                3           4
```

#### 废料的组成

分切过程中产生的废料有以下几种：

| 废料类型   | 说明                                   | 示例                                     |
| ---------- | -------------------------------------- | ---------------------------------------- |
| **边丝**   | 母卷两侧必须切掉的边缘废料             | 每侧8~15mm，双边合计16~30mm              |
| **余料**   | 切完所有订单后，中间剩下的窄带         | 母卷1350mm切2个650mm+边丝20mm，余料=30mm |
| **切除料** | 因质量缺陷需要切掉的部分（如针孔区域） | "Q840-990针孔切除"→切掉150mm             |

#### 废料计算案例

**案例1：一刀切（常见，最简单）**

```
母卷宽度：1350mm
订单要求：1个1300mm宽的订单

切刀方案：
  边丝(10mm) + 订单(1300mm) + 边丝(10mm) = 1320mm
  余料 = 1350 - 1320 = 30mm → 当废料处理

废料率 = (10+10+30) ÷ 1350 = 3.7%
```

**案例2：两刀切（搭配两个订单）**

```
母卷宽度：1350mm
订单A：650mm宽
订单B：650mm宽

切刀方案：
  边丝(8mm) + 订单A(650mm) + 订单B(650mm) + 边丝(8mm) = 1316mm
  余料 = 1350 - 1316 = 34mm → 当废料处理

废料率 = (8+8+34) ÷ 1350 = 3.7%
```

**案例3：有质量缺陷时（更复杂）**

```
母卷宽度：1350mm
质量评审：Q侧840-990mm有针孔，需切除（共150mm不可用）
可用宽度 = 1350 - 150 = 1200mm

订单匹配：找一个宽度≤1200mm的订单
  → 找到一个1150mm宽的订单

切刀方案：
  边丝(10mm) + 订单(1150mm) + 缺陷切除(150mm) + 边丝(10mm) = 1320mm
  余料 = 1350 - 1320 = 30mm → 当废料处理

废料率 = (10+10+150+30) ÷ 1350 = 14.8%（因质量缺陷导致废料率升高）
```

#### 计划员目前怎么做？

计划员目前**用笔算/心算**完成上述计算：

1. 看母卷宽度（如1350mm）
2. 看质量评审有没有需要切除的区域
3. 计算有效宽度
4. 在脑中搜索订单池，找宽度匹配的订单
5. 考虑是否能搭配多个订单减少余料
6. 笔算边丝+订单+余料，验证废料率
7. 如果废料太多，换一种搭配方式重新算

> **这个过程对于有经验的计划员来说很快，但容易出错（特别是多订单搭配时），且新人难以入手。**

#### 系统能做什么？（二期功能）

系统不替代计划员做决策，但可以**辅助计算**：

| 功能             | 说明                                                 | 价值             |
| ---------------- | ---------------------------------------------------- | ---------------- |
| **废料自动计算** | 计划员输入切刀位置后，系统自动算出边丝、余料、废料率 | 减少笔算错误     |
| **搭配方案比较** | 显示不同搭配方案的废料率对比（如1刀切 vs 2刀切）     | 帮助选择最优方案 |
| **宽度校验**     | 自动校验：订单宽度总和+边丝 ≤ 母卷宽度，超宽时报错   | 防止切不下来     |
| **订单筛选联动** | 输入母卷宽度后，自动筛选出宽度匹配的订单列表         | 减少脑中搜索时间 |

### 位置管理方案：为什么料框号不够？

调研中确认客户**已经在用料框号**（如3195号料框），但依然存在"找不到料"的情况。原因是：

```
当前问题：
  知道料在3195号料框 → 但3195号框在车间哪个角落？→ 还是满场找

根本原因：
  料框号 ≠ 位置信息。料框会被搬动，框号只标识"哪个框"，不标识"框在哪里"。
```

因此需要引入**区域+料框**两级定位：

| 层级     | 说明                                | 示例              |
| -------- | ----------------------------------- | ----------------- |
| **区域** | 将现场划分为若干存放区域（如A~F区） | A区（1#分切机旁） |
| **料框** | 区域内的具体料框号                  | 3195号料框        |
| **组合** | 区域+料框 = 精确位置                | A区-3195号框      |

**实施建议**：

- 一期调研时与客户确认现场可划分几个区域，区域编号要简单（如A~F或1~6区）
- 料框移动后需更新区域信息（分切下料时顺便选择区域，增加1步操作）
- 若区域管理落地困难，可先退回到纯料框号管理，后续再升级

### 数据录入方案：解决"数据怎么进来"的问题

> **核心原则**：在车间环境中，不能让工人频繁操作系统。数据录入必须做到**最少操作步骤**。

#### 报工系统的选择（需与客户确认）

数据录入方案中涉及"报工联动"——即报工时自动触发料卷状态更新。但目前报工是在**用友MES**中完成的，这意味着"报工联动"有两种实现方式：

| 方案                        | 说明                                                             | 优势                           | 劣势                                     |
| --------------------------- | ---------------------------------------------------------------- | ------------------------------ | ---------------------------------------- |
| **方案A：对接用友MES**      | 工人仍在用友MES报工，新系统通过API监听报工事件，自动更新料卷状态 | 不改变工人现有操作习惯         | 需要用友MES提供API接口，技术依赖性高     |
| **方案B：报工迁移到新系统** | 将报工功能整合到新系统中，工人在新系统中完成报工+状态更新        | 数据闭环在新系统内，无外部依赖 | 工人需要适应新系统，过渡期可能需两边报工 |

> **建议**：一期优先采用**方案A**（对接用友MES），减少对工人操作习惯的影响。如果用友MES无法提供足够的API支持，再考虑方案B。此项需在详细设计阶段**与客户和用友MES供应商确认**。

#### 各节点录入方式

| 数据节点     | 录入方式                                                             | 操作步骤                      | 谁操作     |
| ------------ | -------------------------------------------------------------------- | ----------------------------- | ---------- |
| 轧机下料     | 报工时自动创建料卷记录，状态="已下料"（通过用友MES对接或新系统报工） | **0步**（报工联动）           | 轧机操作工 |
| 轧机质检完成 | 质检员录入评审信息后，状态自动变为"已质检"                           | **0步**（评审联动）           | 质检员     |
| 分切开始     | 分切操作工扫码领料，状态自动变为"分切中"                             | **1步**（扫码）               | 分切操作工 |
| 分切完成     | 报工时打印子卷标签，选择存放区域+料框号                              | **3步**（扫码+选区域+选料框） | 分切操作工 |
| 精切领料     | 精切操作工扫码领料，状态自动变为"精切中"                             | **1步**（扫码）               | 精切操作工 |
| 精切完成     | 报工确认                                                             | **1步**（扫码确认）           | 精切操作工 |

**关键设计**：

- 使用**区域+料框号**两级管理位置，区域告诉你去哪个方向，料框号告诉你找哪个框
- 区域和料框号均采用**下拉选择**而非手工输入
- 大部分状态变更通过**报工联动**或**扫码**实现，无需单独操作
- 报工联动的实现依赖于用友MES的API支持情况（需确认）

| 轧机     | 排计划时   | 计划员指定卷号→订单 | 卷号已确定                     |
| 分切     | 排计划时   | 计划员指定母卷→订单 | "这个料对应上订单了就不会变了" |
| 精切     | 排计划时   | 计划员指定子卷→订单 | 根据子卷规格匹配精切订单       |
| 质量重排 | 评审后触发 | 计划员手动重新关联  | 质量问题导致换料时手动调整     |

### 订单数据来源

```
用友MES订单池（现有系统）
        ↓ 定期同步（API / 导入）
新系统订单管理
        ↓
计划员排产时关联料卷 → 报工后更新进度 → 订单进度看板
```

> **与用友MES的关系**：新系统从用友MES**同步订单数据**，排产和质量管理在新系统中完成。用友MES的质量模块计划**逐步替代**（客户反馈用友质量模块不好用且多年未改进）。

# 第六章：系统价值总结

## 6.1 场景对比

| 场景               | 使用系统前                   | 使用系统后                     |
| ------------------ | ---------------------------- | ------------------------------ |
| **查看今天的计划** | 在32个Excel Sheet中切换      | 打开生产看板，一屏全览         |
| **了解订单进度**   | 人工统计各卷完成情况         | 系统自动汇总，实时显示         |
| **质量评审传递**   | 手动复制到分切计划，可能漏看 | 评审即时同步，漏不了           |
| **精切找料**       | 在1000个卷里凭经验找         | 扫码查料框号，快速定位         |
| **发现逾期风险**   | 临近交期才发现               | 提前3天预警，留出调整时间      |
| **追溯问题卷**     | 翻多张Excel表                | 输入卷号，一键查看全程质量信息 |
| **计划调整**       | 修改后无记录                 | 所有修改留痕可查               |
| **质量统计分析**   | 自由文本无法统计             | 结构化数据，一键生成月度报表   |
| **评审信息录入**   | 手抄纸质→录入Excel→录入MES   | 系统一次录入，同步所有环节     |

## 6.2 期望达成的效果

### 1. 提高效率

- 减少计划员在Excel间切换、手工复制、人工统计的时间
- 减少精切人员找料时间
- 减少质量评审信息传递和重复录入的时间（不再需要Excel+MES录两遍）

### 2. 减少错误

- 信息自动同步，避免漏看、漏填
- 流程卡点控制，避免"未评审先生产"
- 结构化录入，避免评审信息表述不一致导致的歧义
- 系统留痕，避免版本混乱

### 3. 增强可控性

- 实时掌握订单进度、在制品状态
- 异常情况（逾期、呆滞）及时预警
- 管理决策有数据支撑

### 4. 支撑追溯

- 完整的数据链条：胚料→轧机→分切→精切→包装
- 问题可追溯、责任可界定
- 满足客户质量追溯要求

### 5. 为未来智能化打基础

- 结构化的质量数据支持统计分析和趋势预测
- 结构化的缺陷位置信息支持自动计算有效宽度
- 设备数据接口为自动采集做准备

## A.4 质量评审规则

| 规则       | 说明                                     |
