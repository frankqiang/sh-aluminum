## 一、已确认的设计决策

### 1. 流程卡点设计

| 决策项       | 决策内容                                 |
| ------------ | ---------------------------------------- |
| **卡点模式** | 严格模式：评审完成后才能进入下道工序     |
| **卡点位置** | 质检完成后、下道工序开工前               |
| **实现方式** | 报工校验——未评审的料卷无法在下道工序报工 |

### 2. 评审生效规则

| 评审结论                                        | 生效规则                    |
| ----------------------------------------------- | --------------------------- |
| 让步放行、正常分切/轧制、沟通放行、转精切、改切 | ✅ 当班评审即生效           |
| 改切二级品、切等外、扒废/报废                   | ⚠️ 必须质量工程师确认后生效 |

**说明**：当班班长评审后，料卷可进入下道工序。质量工程师次日复核属于"事后确认"，不阻塞生产（严重问题除外）。

### 3. 合格品处理

| 决策项           | 决策内容                         |
| ---------------- | -------------------------------- |
| **是否需要录入** | 需要——质检员需点击"确认合格放行" |
| **记录内容**     | 谁在什么时间确认放行             |
| **目的**         | 确保追溯完整性                   |

### 4. 评审信息录入方式

| 决策项         | 决策内容                                               |
| -------------- | ------------------------------------------------------ |
| **录入模式**   | 混合模式：结构化为主 + 自由备注                        |
| **结构化字段** | 主评审结论（下拉）+ 多条处理指令（位置+缺陷+处理方式） |
| **自由备注**   | 保留自由文本字段，用于特殊情况说明                     |
| **自动计算**   | 系统根据处理指令自动计算有效宽度/长度                  |

### 5. 设备数据采集

| 决策项       | 决策内容                                       |
| ------------ | ---------------------------------------------- |
| **一期**     | 手工录入，专注于流程管控                       |
| **二期**     | 实现针检仪/板型仪数据自动采集                  |
| **优先顺序** | 板型仪Excel解析（简单）→ 针检仪PDF解析（复杂） |

### 6. 与现有系统关系

| 决策项              | 决策内容                    |
| ------------------- | --------------------------- |
| **用友MES质量模块** | 完全替代，不再使用          |
| **数据同步**        | 无需同步到用友              |
| **Excel报表**       | 系统上线后不再使用Excel录入 |

### 7. 历史数据迁移

| 决策项       | 决策内容                                          |
| ------------ | ------------------------------------------------- |
| **是否迁移** | 需要批量导入历史数据                              |
| **数据来源** | 现有Excel报表（轧机质量报表、分切不合格品报表等） |
| **迁移范围** | 待定（建议至少近6个月）                           |

### 8. 评审修改权限

| 决策项         | 决策内容                               |
| -------------- | -------------------------------------- |
| **是否可修改** | 可以修改                               |
| **修改记录**   | 保留完整修改历史（谁、何时、改了什么） |
| **修改权限**   | 质量工程师及以上可修改他人的评审       |

### 9. 录入终端

| 决策项       | 决策内容                    |
| ------------ | --------------------------- |
| **主要终端** | PC端（固定工位电脑）        |
| **辅助终端** | 移动端（手机/平板）作为补充 |
| **一期范围** | PC端优先，移动端视资源情况  |

## 二、业务规则确认

### 1. 避留白区域计算规则

| 规则项               | 规则内容                                           |
| -------------------- | -------------------------------------------------- |
| **是否算入有效宽度** | ✅ 算入有效宽度                                    |
| **是否需要标记**     | ✅ 需要标记位置，传递给下道工序                    |
| **与"切除"的区别**   | 切除=物理切掉不算入有效宽度；避留白=保留但标记位置 |

### 2. 评审信息传递方式

| 传递对象 | 当前方式               | 系统改进                  |
| -------- | ---------------------- | ------------------------- |
| 计划员   | 共享文件夹 + 口头/微信 | 系统实时可见              |
| 包装     | 共享文件夹             | 系统可查 + 可考虑打印标签 |
| 精切     | 共享文件夹             | 系统实时可见              |

## 四、一期功能范围

### 必做（P0）

1. **质量数据录入**
   - 替代Excel录入
   - 支持结构化录入 + 自由备注
   - 支持合格/不合格判定

2. **评审流程管理**
   - 待评审列表
   - 评审提交与确认
   - 权限控制（按角色区分）
   - 严重问题二次确认机制

3. **流程卡点**
   - 报工校验：未评审不能报工
   - 与排产模块联动

4. **信息实时同步**
   - 评审完成后计划员/下道工序可见

5. **追溯查询**
   - 按卷号/时间/缺陷类型查询

### 应做（P1）

1. 修改记录审计
2. 历史数据批量导入
3. 移动端录入（简化版）

### 可选（P2/二期）

1. 设备数据自动采集
2. 历史案例推荐
3. 客户阈值预警
