---

## 一、调研背景

本次调研针对铝箔生产企业的质量管理业务，目标是为MES系统的质量模块设计提供依据。调研覆盖了轧机、分切、精切三个工序的质量检测与评审流程。

**调研方式**：

- 与质量工程师沟通（第一次调研：业务流程）
- **与质检员沟通（第二次调研：设备数据录入流程）**
- 分析质量报表Excel文件
- 获取检测设备导出样例（针检PDF、表检PDF/图片、板型仪Excel）
- **分析2315条评审信息，完成结构化分类**
- 确认用友MES系统现状

---

## 二、核心结论

1. **设备有接口**：客户确认所有检测设备都有数据接口（2026-01-31）
2. **但智能数据采集价值有限**：设备输出的是原始数据，而录入Excel的是人工总结后的信息，自动采集无法替代人工判断
3. **用友MES不满足需求**：客户反馈用友未深入调研，系统不好用且多年未改进，有替换意愿
4. **系统核心价值重新定位**：从"智能自动化"调整为"数据电子化+流程规范化+辅助决策"

## 三、设备数据采集可行性【二次调研已确认】

> ✅ **重要说明**：以下结论已通过与质检员的二次调研确认。

### 3.1 核心结论：设备数据采集可行性矩阵

| 设备       | 设备能输出的内容          | 实际录入Excel的内容       | 需人工判断？ | 自动采集价值 |
| ---------- | ------------------------- | ------------------------- | ------------ | ------------ |
| **针检仪** | A/B/C/D/E级针孔数量、密度 | A/B/C/D/E级针孔数量       | 部分是       | ⭐⭐⭐⭐ 高  |
| **表检仪** | 几十上百个缺陷详情        | 人工总结的关键缺陷描述    | **是**       | ⭐ 低        |
| **板型仪** | 下榻量数值、曲线          | 下榻量（B/C/D等级已取消） | 否           | ⭐⭐⭐ 中    |

### 3.2 针检仪数据录入流程【已确认】

| 确认项                | 确认结果                                                                            |
| --------------------- | ----------------------------------------------------------------------------------- |
| PDF报告录入哪些信息？ | A、B、C、D、E级总针孔个数                                                           |
| 录入到Excel哪些列？   | "针孔检测信息(A/B级针孔)"列、"针孔检测信息(C/D级针孔)"列、"针孔检测信息(E级针孔)"列 |
| 是否直接抄PDF数字？   | **是**，直接抄写，不需要做处理                                                      |
| 密集型针孔如何判定？  | **需要人工判断**，设备暂无此功能                                                    |
| 录入耗时？            | 无具体数据，不是实时录入，有空就录                                                  |

**自动采集结论**：针孔数量（A/B/C/D/E级）**可以自动采集**，密集型针孔需保留人工判断。

### 3.3 表检仪数据录入流程【已确认】

| 确认项                          | 确认结果                                               |
| ------------------------------- | ------------------------------------------------------ |
| 一卷料产生多少缺陷？            | 几十上百个缺陷                                         |
| Excel中"表面检测信息"怎么来的？ | **人工总结**，从大量缺陷中挑选超出客户要求的关键缺陷   |
| 表检仪能否导出总结句子？        | **不能**，必须人工总结                                 |
| 挑选关键缺陷的依据？            | 超出客户要求的，或严重的。大多数表检缺陷都不严重可放行 |
| 录入耗时？                      | 无具体数据，不是实时录入，有空就录                     |

**自动采集结论**：表检仪数据**无法自动采集**，"表面检测信息"是人工从几百个缺陷中总结出的关键问题。

### 3.4 板型仪数据录入流程【已确认】

| 确认项                       | 确认结果                           |
| ---------------------------- | ---------------------------------- |
| 下榻量是否直接抄板型仪数据？ | **是**                             |
| B/C/D等级如何判定？          | **已取消**，该列现在可以不用填写了 |
| 录入耗时？                   | 无具体数据，不是实时录入，有空就录 |

## 四、质量评审业务现状

### 4.1 质量检测流程

```
轧机（末道次）→ 分切 → [退火] → 精切
     ↓           ↓              ↓
   轧机质检    分切质检       精切质检
   (厚度、板型)  (针孔、表面、板型)  (边部毛刺、铝粉、拉伸强度)
```

### 4.2 检测设备清单

| 设备类型   | 品牌                        | 数量        | 导出格式             |
| ---------- | --------------------------- | ----------- | -------------------- |
| 针检仪     | 意大利EDS、赤宵、双元、精质 | 8台         | PDF                  |
| 表检仪     | 赤宵、双元                  | 8台         | 图片/PDF             |
| 离线板型仪 | JB-BXY-1750                 | 5台         | Excel                |
| 轧机测厚   | 待确认                      | 每台轧机1台 | 无接口（用天平称重） |

### 4.3 数据流转现状

```
检测设备 → 设备电脑（厂商软件）→ 导出PDF/Excel → 人工看+判断+总结 → 手工录入Excel → 手工录入MES
```

**关键问题**：

- 用友MES质量模块也是**手工录入**，没有与检测设备对接
- 目前Excel和MES**两套并行**，质检员要录两遍

### 4.4 评审规则现状

| 问题             | 现状                            |
| ---------------- | ------------------------------- |
| 有无统一标准？   | ❌ 没有，因为"客户要求随时在变" |
| 评审依据是什么？ | 质检员的**经验判断**            |
| 规则有无沉淀？   | ❌ 没有，知识都在人脑子里       |

### 4.5 质量部门组织结构

#### 4.5.1 岗位层级

```
质量主管
    ↓
质量工程师（白班6人）
    ↓
当班班长
    ↓
质检员（每工序每班约4人，一人负责多条线）
```

#### 4.5.2 评审权限划分

| 岗位           | 职责                                     | 评审权限                                   |
| -------------- | ---------------------------------------- | ------------------------------------------ |
| **质检员**     | 检测设备数据采集、纸质记录、发现问题上报 | 经验丰富的可评审本机台，难以判定的上报班长 |
| **当班班长**   | 汇总当班质量情况、评审有争议的料卷       | 可评审当班所有料卷                         |
| **质量工程师** | 白班复核所有评审、处理疑难问题           | **降级、让步放行、报废**必须由工程师决定   |
| **质量主管**   | 审批重大质量决策                         | 需要更高层级审批的特殊情况                 |

#### 4.5.3 评审流程

```
发现质量问题
    ↓
质检员现场检测 → 记录纸质质量表
    ↓
当班班长汇总 → 评审（如有能力）
    ↓
第二天白班质量工程师 → 复核所有评审（二次评审）
    ↓
有争议的 → 上报质量主管
```

**关键点**：

- 以前不合格的必须等白班质量工程师处理，现在核心质检人员和班长也可评审
- 第二天质量工程师会"全部重新评审一遍"，主要是确认，有需要更改的就更改

### 4.6 质量术语词典

> **来源**：第一次调研中与质量工程师确认的专业术语解释

#### 4.6.1 位置描述术语

| 术语      | 含义                                      | 示例               |
| --------- | ----------------------------------------- | ------------------ |
| **C侧**   | 操作侧（Operator Side），从边缘开始量mm数 | C325 = 操作侧325mm |
| **Q侧**   | 驱动侧（Drive Side），从边缘开始量mm数    | Q280 = 驱动侧280mm |
| **L=xxx** | 周期长度，表示缺陷每隔xxx mm重复出现      | L985 = 每985mm一次 |

#### 4.6.2 缺陷类型术语

| 术语           | 含义                                                             | 产生原因           |
| -------------- | ---------------------------------------------------------------- | ------------------ |
| **辊印**       | 轧辊上的印记压到铝箔表面                                         | 轧辊表面有缺陷     |
| **辊印透**     | 辊印穿透到另一面，有洞（与辊印本质一样，只是记录方式不同）       | 轧辊缺陷严重       |
| **暗道**       | 铝箔表面出现黑色色差条纹                                         | 表面不均匀         |
| **白条**       | 白色条纹，宽度**1.5mm以下**                                      | 轧制问题           |
| **条纹**       | 白色条纹，宽度**1.5mm以上**（与白条的区别就是宽度）              | 轧制问题           |
| **气道**       | 比针孔大的孔洞                                                   | 铝液中的气泡       |
| **孔洞**       | 较大的洞（比针孔大，与气道类似）                                 | 多种原因           |
| **针孔**       | 铝箔上的微小孔洞，按直径分A/B/C/D/E级                            | 铝液杂质、轧制问题 |
| **密集型针孔** | 局部区域针孔密度比其他区域明显更高，**需人工判断，设备无法检测** | -                  |
| **直棱**       | 铝箔卷成卷后应平整但出现鼓起                                     | 卷取张力不均       |
| **棱印**       | 较轻微的直棱，有印子但没鼓起                                     | -                  |
| **下榻**       | 板型缺陷，铝箔中间部分下沉变形                                   | 板型问题           |
| **压坑**       | 表面压凹的坑                                                     | 异物压入           |
| **麻点**       | 异物导致的小压坑                                                 | 异物               |
| **油泥**       | 油和污泥的混合物附着在表面                                       | 轧制油污染         |
| **油斑**       | 油污斑点                                                         | 轧制油残留         |
| **带油**       | 表面带油                                                         | 轧制油未挥发完     |
| **划伤**       | 表面划痕                                                         | 搬运或加工过程碰撞 |
| **擦划**       | 擦伤/划伤                                                        | 同上               |
| **亮线**       | 黑色线条形成的色差，通常是辊子印出来的                           | 轧辊               |
| **黑线**       | 黑色线条                                                         | 多种原因           |
| **横折**       | 横向折痕                                                         | 卷取或搬运问题     |
| **起皱**       | 表面起皱                                                         | 张力问题           |
| **串层**       | 铝箔缠绕时层串出，导致端面不平整                                 | 卷取问题           |
| **斜纹**       | 辊子印的斜向纹路                                                 | 轧辊               |
| **闪筋**       | 铝箔表面突然出现的褶皱                                           | 张力突变           |

#### 4.6.3 处理方式术语

| 术语         | 含义                                                                       |
| ------------ | -------------------------------------------------------------------------- |
| **避留白**   | 在指定位置预留空白区域，告诉下道工序那个位置有缺陷，需要避开留在边上当废料 |
| **切除**     | 物理切掉指定区域，直接扔掉当废料                                           |
| **切二级品** | 该区域降级为二级品（也叫处理品），可以卖给能接受的客户                     |
| **切等外**   | 与切二级品含义一样                                                         |
| **让步放行** | 轻微缺陷、不影响使用的，可以放行但备注缺陷信息给客户                       |
| **改切**     | 需要切除指定位置后重新匹配订单                                             |
| **扒至**     | 扒除外层至指定米数，剩余放行                                               |
| **扒除**     | 扒掉外层（与扒至含义一样）                                                 |
| **扒废**     | 扒掉的部分作废                                                             |
| **反开**     | 反向开卷，使缺陷位置从底部变为顶部                                         |
| **反开卷**   | 与反开含义一样                                                             |
| **下料观察** | 下料后观察情况确认                                                         |
| **在线观察** | 生产过程中观察                                                             |
| **复查**     | 下料重新检查                                                               |
| **复测**     | 重新测量                                                                   |
| **复检**     | 重新检验                                                                   |
| **验证**     | 验证处理效果                                                               |
| **电晕**     | 电晕处理，提高达因值（表面张力）                                           |
| **降速**     | 降低生产速度（解决油污挥发问题）                                           |
| **导卷**     | 导卷处理                                                                   |
| **薄剪**     | 使用薄剪设备切割                                                           |
| **吸边**     | 表达切除的意思，Q侧多少mm吸边 = 切除这部分                                 |

#### 4.6.4 其他术语

| 术语             | 含义                                                                         |
| ---------------- | ---------------------------------------------------------------------------- |
| **合轧**         | 精轧时两张铝箔叠在一起轧，轧完后分开变成"上张"和"下张"两卷                   |
| **上张/下张**    | 合轧后分开的两卷，会分别记录厚度和针孔数                                     |
| **达因值**       | 衡量铝箔表面张力的指标（单位：dyne/cm），数值越高后续涂覆越容易附着          |
| **面密度**       | 单位面积重量（g/㎡），用天平称重计算                                         |
| **接头**         | 双零箔断了后用超声波焊接起来形成的接头数量                                   |
| **管芯**         | 卷芯（管子）的规格，如"63/60"表示两边突出的长度                              |
| **框号**         | 料存放在哪个料框里面                                                         |
| **打底**         | 卷内层预留不用的部分米数，因为料的底部不平整需要打底变平整（**现已不使用**） |
| **纵向针孔密度** | 长度方向上的针孔密度，单位可能是个/米                                        |

### 4.7 各工序检测项目详情

#### 4.7.1 轧机质检（末道次）

| 检测项目     | 检测方式               | 说明                                       |
| ------------ | ---------------------- | ------------------------------------------ |

## 五、智能评审可行性评估【核心章节】

> **本章目的**：客观评估"智能评审"的可实现程度，明确系统边界，避免对客户过度承诺。

### 5.1 目前无法实现"自动评审"的核心原因

#### 5.1.1 障碍一：评审规则不存在——没有标准可编程

**质量工程师原话**：

> "没有明确的指标，因为客户的要求随时都在变"
> "主要靠自检员各自的经验判断"
> "规则是缺失的，没有沉淀下来形成知识库"

**问题本质**：

智能评审系统的前提是有**明确、可量化、可编程的判定规则**。理想的规则应该是这样的：

```
IF 针孔A级 ≤ 100个 AND 针孔B级 ≤ 50个 THEN 合格
IF 板型曲线超出±3范围的点数 ≤ 5% THEN 等级=B
IF 辊印位置 IN (0-50mm) AND 严重程度 = "轻微" THEN 让步放行
```

但从调研中发现，**根本不存在这样的固定规则**：

| 检测项       | 期望的规则形式（可编程）               | 实际的判定方式（调研原话）                                                                     | 能否程序化      |
| ------------ | -------------------------------------- | ---------------------------------------------------------------------------------------------- | --------------- |
| **针孔**     | 针孔A级≤100个 且 B级≤50个 → 合格       | "看情况，A客户要求严，B客户要求松"                                                             | ❌ 无固定阈值   |
| **表面缺陷** | 缺陷数量≤X个 → 合格                    | "看缺陷严重程度，轻微的可以放行"                                                               | ❌ 靠经验判断   |
| **板型**     | 曲线超出±3范围点数≤5% → B级            | "看曲线形状，凭经验给B/C/D"                                                                    | ❌ 没有量化公式 |
| **综合判定** | IF 针孔不合格 AND 板型不合格 THEN 报废 | "多项指标同时有问题，我们会把所有指标都记录下来，例如针孔缺陷可以通过，那么就看版型是否能通过" | ❌ 无优先级规则 |

**深层原因分析：为什么规则无法固化？**

##### 原因1：客户差异化导致规则"一客一策"

**调研证据**：

- "不同客户有不同的要求"
- "没有明确的指标，因为客户的要求随时都在变"

**实际案例（来自Excel数据分析）**：

```
同一卷料（针孔A级=94个，B级=13个）的不同命运：

场景1：给河北兴恒（重要客户）
  → 质检员判定：针孔超标
  → 评审结论：改切二级品
  → 原因：该客户要求严格，A级针孔不超80个

场景2：给普通客户
  → 质检员判定：针孔偏多但可用
  → 评审结论：让步放行，包装备注
  → 原因：该客户要求宽松，接受A级针孔≤120个

场景3：如果是急单
  → 质检员判定：虽超标但不严重
  → 评审结论：与客户沟通后让步放行
  → 原因：交期紧张，客户同意轻微超标
```

**这意味着**：同样的检测数据（A=94，B=13），需要根据客户来查询不同的规则库。但调研发现：

- ❌ 没有"客户-质量要求"对照表
- ❌ 没有"产品-客户-阈值"配置表
- ❌ 要求不是固定的，会随订单、时间变化

##### 原因2：订单差异化导致规则"一单一策"

**调研证据**：

- "同一客户不同订单可能有不同要求（加急件可放宽、重要件更严格）"

**实际案例**：

```
同一客户（河北兴恒）的两个订单：

订单A（常规订单）：
  - 针孔要求：A级≤80个
  - 板型要求：B级或以上
  - 评审策略：严格把关

订单B（加急订单，客户催单）：
  - 针孔要求：协商后可放宽到≤100个
  - 板型要求：C级也可接受
  - 评审策略：优先保交期
```

**这意味着**：即使建立了"客户-规则库"，还需要按订单动态调整规则。

##### 原因3：缺陷的"严重程度"无法量化

**调研证据**：

- "看缺陷严重程度，轻微的可以放行"
- "真正影响判定结果的，例如有没有什么大的密集型的针孔，像这种是没法来量化的，人为主观性比较多"

**什么是"轻微"？什么是"严重"？**

| 缺陷类型   | 设备能检测到的 | 质检员实际判断的（无法量化）                                 |
| ---------- | -------------- | ------------------------------------------------------------ |
| 针孔密集型 | 针孔总数=150个 | "这150个分散在3万米料上，可以" vs "这150个集中在500米，不行" |
| 辊印       | 检测到辊印缺陷 | "轻微辊印，精切可以避开" vs "辊印严重，穿透了，必须切除"     |
| 板型       | 下榻量=2.8mm   | "虽然接近3mm但曲线平滑，B级" vs "虽然只有2.5mm但波动大，C级" |

**结论**：质检员在判断时考虑的因素（分布、程度、影响）无法从设备数据中提取。

---

#### 5.1.2 障碍二：设备数据≠评审结论——中间缺少"人的判断"

**设备输出 vs 评审结论的差距**：

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                           理想的自动评审链条                                              │
├─────────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                          │
│  1️⃣ 读取设备数据 → 2️⃣ 应用评审规则 → 3️⃣ 输出评审结论                                    │
│         ↓                 ↓                 ↓                                           │
│   "针孔A=94,B=13"    "A≤100且B≤50"      "合格"                                          │
│                                                                                          │
│  ─────────────────────────────────────────────────────────────────────────────────────  │
│                                                                                          │
│  ⚠️ 当前状态：第2步卡住了！                                                              │
│                                                                                          │
│  - 没有"A≤100且B≤50"这样的固定规则                                                       │
│  - 规则在质检员脑子里，随客户、订单、时间变化                                            │
│  - 即使有规则，也经常需要例外处理                                                        │
│                                                                                          │
└─────────────────────────────────────────────────────────────────────────────────────────┘
```

**表检仪的特殊困难**：

| 设备输出                                   | 需要人工做的判断               | 最终录入内容                    |
| ------------------------------------------ | ------------------------------ | ------------------------------- |
| 缺陷详情：压坑645个、横折2个、黑点210个... | 哪些是关键缺陷？哪些可以忽略？ | "20262米Q584油块"（总结性描述） |
| 缺陷位置图：二维地图                       | 缺陷分布是否影响客户使用？     | 只记录关键位置                  |

**人工判断无法省略的原因**：

1. 一卷料几万米，产生几百个缺陷，不可能全部录入
2. 质检员需要**筛选关键问题**，这是经验判断
3. 同样的缺陷数量，分布不同，结论可能不同

---

#### 5.1.3 障碍三：让步放行机制——规则之外的例外处理

**调研中发现的评审特殊情况**：

| 情况         | 描述                                     | 能否自动化        |
| ------------ | ---------------------------------------- | ----------------- |
| **让步放行** | 质量不达标但客户能接受，与客户沟通后放行 | ❌ 需要人工沟通   |
| **改切降级** | 原计划给A客户，发现质量不行，改给B客户   | ❌ 需要人工匹配   |
| **部分切除** | 某段有缺陷，切掉后剩余部分合格           | ⚠️ 需知道精确位置 |
| **复测确认** | 数据异常，需要复测确认                   | ❌ 需要人工判断   |

**让步放行的决策过程**：

```
发现质量问题 → 评估影响 → 与客户沟通 → 客户同意 → 让步放行
              ↓           ↓           ↓
           人工判断     人工沟通     人工确认
```

这个过程**完全无法自动化**，因为涉及：

- 与外部客户的沟通
- 紧急程度的权衡

---

#### 5.1.4 障碍四：评审必须灵活

无法建立统一的智能评审规则：

| 行业特性                | 调研中的证据                             | 对智能评审的影响                       |
| ----------------------- | ---------------------------------------- | -------------------------------------- |
| **订单驱动生产**        | 每个订单都可能有特殊的质量要求           | 规则需要按"客户+产品+订单"三级动态变化 |
| **客户差异化严重**      | "不同客户有不同的要求"、"标准不是统一的" | 无法建立一套通用的判定规则             |
| **产品类型差异大**      | 电池箔56列检测项 vs 双零箔29列           | 同一套规则无法覆盖所有产品类型         |
| **一级品/二级品价差大** | "不止20%"                                | 降级决策有重大经济影响，必须人工把关   |
| **让步放行常见**        | "轻微缺陷、不影响使用的可以让步"         | 涉及商务沟通，不是纯技术判定           |

**评审本质**：

```
不是"合格/不合格"的二分法判断
而是"这个质量问题，可以给哪个客户用"的匹配决策

例如：同一卷针孔超标的料
  → A客户：要求严格 → 不合格
  → B客户：要求宽松 → 可以用
  → C客户：价格敏感 → 降级后可接受

这种匹配决策需要计划员+质检员+销售多方沟通，无法自动化
```

---

#### 5.1.5 障碍五：现实执行困难——系统难以改变现有模式

调研中发现的现场执行问题，说明即使有智能系统也难以真正落地：

| 现实问题             | 调研证据                                    | 影响                           |
| -------------------- | ------------------------------------------- | ------------------------------ |
| **质检人员分布不均** | "一人可能负责多条线"、"每个工序一班4个左右" | 无法保证每卷都及时录入系统     |
| **信息同步滞后**     | "有时间就录入Excel"、"通常都会滞后"         | 评审信息出来时计划可能已经排了 |
| **评审权限分层**     | 质检员→当班班长→质量工程师→质量主管         | 评审流程长，不是一步到位       |
| **二次评审机制**     | "第二天质量工程师将所有料在全部评审一遍"    | 首次评审可能被推翻             |
| **计划先于评审**     | "评审前，这些料可能已经排计划了"            | 即使有即时提醒，也来不及阻止   |

**客户提出的关键诉求**（调研原话）：

> "希望系统能卡住这一点，他们那边不报工，评审结果不出来，他就不能切"

这条诉求揭示了核心矛盾：

- 理想状态：**评审→计划→生产**（串行）
- 实际状态：**生产 & 评审同时进行**（并行）

智能评审无法解决这个流程问题，需要通过**流程卡点**（如报工校验）来解决。

### 5.2 可实现功能与不可实现功能边界

#### 5.2.1 ✅ 可以实现的功能

### 5.3 系统定位建议

> **质量评审模块应定位为"辅助决策工具"，而非"自动判定系统"。**

| 定位         | ✅ 正确表述                              | ❌ 错误表述                  |
| ------------ | ---------------------------------------- | ---------------------------- |
| **功能定位** | 帮助质检员更高效地完成评审               | 自动完成评审，不需要质检员   |
| **决策角色** | 系统提供信息和历史参考，质检员做最终判定 | 系统自动判定，质检员只需确认 |
| **价值主张** | 减少录入时间、信息实时同步、流程可追溯   | 智能评审、AI判定、无人化     |
| **对外沟通** | "智能辅助质量管理系统"                   | "全自动智能评审系统"         |

### 6.5 评审信息结构模式总结

一条评审信息可能包含以下组成部分（按出现顺序）：

```plain
┌─────────────────────────────────────────────────────────────────────────────┐
│ 评审信息结构模式                                                              │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│ 【可选】主评审结论 + 【可选】传递对象备注 + 【多个】处理指令                    │
│                                                                              │
│ 示例1：让步放行，包装备注：Q0-80针孔切除，C0-40针孔切除                        │
│        ├─主结论─┤ ├─传递对象─┤ ├───处理指令1───┤ ├───处理指令2───┤          │
│                                                                              │
│ 示例2：Q0-80针孔切除，5800米气道切除，Q335辊印避留白                          │
│        ├─处理指令1──┤ ├─处理指令2────┤ ├──处理指令3──┤ （无主结论）           │
│                                                                              │
│ 示例3：转精切，0-11000米切二级品，C0-30针孔切除                               │
│        ├主结论┤ ├─处理指令1（长度）─┤ ├─处理指令2（宽度）─┤                    │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

**处理指令的结构模式**：

```plain
处理指令 = [位置范围] + [可选:缺陷类型] + [处理方式]

位置范围格式：
├─ 宽度方向：Q0-80（Q侧0到80mm）、C0-40（C侧0到40mm）、Q侧+C侧组合
├─ 长度方向：5800米、0-12000米、底部5800米、外圈2000米
├─ 边部位置：边部、两边部100mm
└─ 整体位置：整幅、整卷
```

### 6.6 排产模块需要的结构化信息

| 排产决策                         | 需要的质量信息            | 说明                                          |
| -------------------------------- | ------------------------- | --------------------------------------------- |
| **这卷料可用吗？**               | 主评审结论                | 合格放行 / 让步放行 / 改切 / 报废             |
| **能给哪个等级的客户？**         | 产品等级                  | 一级品 / 二级品                               |
| **有效宽度是多少？**             | 宽度方向需切除的范围      | Q0-80切除 → 有效宽度减少80mm                  |
| **有效长度是多少？**             | 长度方向需切除/降级的范围 | 底部5800米切二级品 → 一级品有效长度减少5800米 |
| **怎么安排切割顺序？**           | 缺陷位置 + 处理方式       | 底部有质量问题，先切低要求订单                |
| **需要传递什么指令给下道工序？** | 具体处理指令              | "C0-40针孔切除，Q335辊印避留白"               |

### 11.3 Excel质量报表字段说明

> **来源**：第一次调研中与质量工程师确认的字段含义

#### 11.3.1 轧机质量报表字段（1月份轧机质量报表.xlsx）

| 字段名            | 含义                                                       |
| ----------------- | ---------------------------------------------------------- |
| 序号              | 记录序号                                                   |
| 机台号            | 使用的轧机编号，如2#、4#、8#等                             |
| 道次              | 轧制道次，成品通常是第4-5道次                              |
| 卷号              | 料卷的唯一编号，如Y2512912                                 |
| 合金              | 铝合金牌号，如1060、1100                                   |
| 宽度（㎜）        | 铝箔宽度                                                   |
| 重量（㎏）        | 料卷重量                                                   |
| 上张厚度/下张厚度 | 合轧后分开的两卷各自的厚度                                 |
| 上张针孔/下张针孔 | 合轧后分开的两卷各自的针孔数量                             |
| 压延质量情况      | 轧制过程中发现的质量问题文字描述，如"Q860辊印印过，L985"   |
| 质量判定          | 合格/不合格                                                |
| 评审信息          | 质量工程师填写的处理意见                                   |
| 操作手            | 轧机操作工姓名                                             |
| 生产时间          | 生产日期                                                   |
| 评审人            | 质量工程师姓名                                             |
| 班组              | 甲/乙/丙班                                                 |
| 班次              | 白班/中班/夜班                                             |
| 达因值            | ~~以前检测，现已取消~~（由于配方升级，轧机工序已不再检测） |
| 备注              | 其他补充信息                                               |

#### 11.3.2 分切不合格品报表-电池箔Sheet字段

| 字段名                  | 含义                                               | 数据来源         |
| ----------------------- | -------------------------------------------------- | ---------------- |
| 序号                    | 记录序号                                           | 手工录入         |
| 机台号                  | 使用的分切机编号                                   | 手工录入         |
| 客户                    | 目标客户名称                                       | 手工录入         |
| 大卷号                  | 母卷（来料）的卷号                                 | 手工录入         |
| 合金                    | 铝合金牌号                                         | 手工录入         |
| 厚度（μm）              | 铝箔厚度                                           | 手工录入         |
| 宽度（㎜）              | 铝箔宽度                                           | 手工录入         |
| 车次                    | 分切后的子卷编号（一个母卷分成多车）               | 手工录入         |
| 米数（M）               | 米数/长度                                          | 手工录入         |
| 重量（㎏）              | 料卷重量                                           | 手工录入         |
| 接头                    | 双零箔断了后超声波焊接形成的接头数量               | 手工录入         |
| 面密度g/㎡              | 单位面积重量，用天平称上下张的左中右多点测量       | 天平称重计算     |
| 达因值（Dyne）          | 表面张力指标                                       | 达因笔检测       |
| 离线版型数据            | 用离线板型仪测量的数据                             | 离线板型仪导出   |
| 表面检测信息            | 表检仪检测结果的**人工总结**                       | 人工从表检仪总结 |
| 针孔检测信息(A/B级针孔) | 较大针孔数量，数值格式如"571/128"                  | 针检仪PDF抄录    |
| 针孔检测信息(C/D级针孔) | 较小针孔数量                                       | 针检仪PDF抄录    |
| E级针孔/密集型针孔      | E级针孔可抄录，密集型需人工判断                    | 针检仪+人工判断  |
| 离线针孔                | 用离线设备（透强光）检测的针孔数                   | 离线针检仪       |
| 纵向针孔密度            | 长度方向上的针孔密度（个/米）                      | 针检仪+人工判断  |
| 管芯                    | 卷芯（管子）的规格，如"63/60"表示两边突出的长度    | 手工录入         |
| 打底(M)                 | 卷内层预留不用的部分米数（**现已不使用**）         | ~~已取消~~       |
| 框号                    | 料存放在哪个料框里面                               | 手工录入         |
| 质量判定                | 不合格（此表只记录不合格的）                       | 人工判定         |
| 评审信息                | 质量工程师填写的处理意见                           | 人工填写         |
| 降级原因                | 导致降级的缺陷类型                                 | 人工填写         |
| 分切质量情况            | 分切过程中发现的质量问题描述（肉眼看到的其他缺陷） | 人工目检         |
| 主操手/质检员           | 操作工和质检员姓名                                 | 手工录入         |
| 日期/班组               | 生产日期和班组                                     | 手工录入         |
| 评审人                  | 质量工程师姓名                                     | 手工录入         |
| 再次评审/二次评审人     | 复审机制，存在争议时的二次评审                     | 手工录入         |
| 备注                    | 其他补充信息                                       | 手工录入         |

#### 11.3.3 分切不合格品报表-双零箔Sheet字段

> **说明**：双零箔Sheet字段比电池箔少（29列 vs 56列），因为用途和关注点不同

| 字段名     | 含义                              | 与电池箔的区别         |
| ---------- | --------------------------------- | ---------------------- |
| 针孔（个） | 只有总针孔数，无A/B/C/D/E级细分   | 电池箔有分级           |
| 板型       | B/C/D等级（**现已取消，可不填**） | 现用下榻量替代         |
| 下榻量     | 板型检测数据                      | 直接抄离线板型仪       |
| 分切缺陷   | 肉眼看到的缺陷描述                | 电池箔叫"分切质量情况" |

> 其他基础字段（序号、机台号、客户、卷号等）与电池箔Sheet相同

#### 11.3.4 精切车间Sheet字段

| 字段名               | 含义                                     | 数据来源     |
| -------------------- | ---------------------------------------- | ------------ |
| 实测宽度（㎜）       | 实际测量的宽度，与计划宽度可能有偏差     | 手工测量     |
| 接头（个）           | 接头数量                                 | 同分切       |
| 面密度g/m²           | 单位面积重量                             | 天平称重计算 |
| 端面毛刺量(μm)       | 精切后端面（切口处）的毛刺高度           | 显微镜测量   |
| 端面铝粉量(个/10㎝²) | 端面残留的铝粉颗粒数                     | 强光手电照射 |
| 表面质量             | 表面缺陷的文字描述                       | 人工目检     |
| 管芯（mm）           | 卷芯直径                                 | 手工测量     |
| 下榻量/mm            | 板型数据                                 | 离线板型仪   |
| 抗拉强度(Mpa)        | 材料抗拉能力（破坏性测试，每卷取样检测） | 拉力测试设备 |
| 延伸率(%)            | 材料延展性（破坏性测试，每卷取样检测）   | 拉力测试设备 |
| 标签粘贴处           | 原名"头尾备注"，用来备注哪些是头料尾料   | 手工录入     |
| 卷芯长度             | 管芯的轴向长度                           | 手工测量     |
| 打底米数（M）        | 预留的内层米数（**现已不使用**）         | ~~已取消~~   |

> 其他基础字段（序号、机台号、客户、卷号、厚度、宽度、米数、重量、质量判定、评审信息、降级原因等）与分切报表相同
