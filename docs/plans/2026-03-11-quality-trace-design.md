# 质量追溯查询设计方案

> **日期**：2026-03-11
> **模块**：质量管理 → 追溯查询
> **目标**：新增独立追溯查询页面，支持按卷号/时间/缺陷类型等多维度查询历史质量记录，结合可视化图表辅助质量统计分析
> **产出**：设计方案文档（经 brainstorming 流程讨论确认）

## 二、方案对比

### 方案 A：经典上下分区（✅ 推荐）

```
┌──────────────────────────────────────────────────────────┐
│ 标题"追溯查询" + 统计卡片（总数 / 降级率 / 本月评审数）  │
├──────────────┬──────────────┬──────────────┤
│ 缺陷类型分布  │  降级趋势     │  机台缺陷对比 │  ← 3个图表横排
│  (环形图)     │  (折线图)     │  (柱状图)     │
├──────────────┴──────────────┴──────────────┤
│ 筛选栏（卷号 / 时间范围 / 缺陷类型 / 结论 / 等级 / 机台）│
├──────────────────────────────────────────────────────────┤
│ 结果表格   ←── 点击行 → 右侧抽屉弹出完整质量档案 ──→    │
└──────────────────────────────────────────────────────────┘
```

- ✅ 信息层次清晰：统计概览 → 筛选 → 明细
- ✅ 符合 MES 系统常见布局习惯，操作人员容易上手
- ✅ 利用宽屏优势横排3个图表
- ⚠️ 图表区占一定高度，小屏幕上表格区域可能较矮

### 方案 B：可折叠统计区

与方案 A 相同布局，但图表区可以折叠/展开。折叠后只显示统计卡片，为表格腾出更多空间。

- ✅ 灵活，日常追溯查询时可折叠图表
- ⚠️ 额外的交互逻辑，增加少量复杂度

### 方案 C：左右分栏

```
┌────────────────┬────────────────────────────────┐
│ 统计图表（纵排）│  筛选栏                         │
│                │  结果表格                       │
└────────────────┴────────────────────────────────┘
```

- ❌ 横向空间被挤压，质量数据列较多（卷号+车次+机台+客户+规格+缺陷+结论+等级+评审人+时间），表格列数受限
- ❌ 不适合铝箔行业宽表格场景

**采用方案 A**：上下分区最符合 MES 系统的使用习惯，图表横排充分利用宽屏。

## 三、设计方案

### 3.1 统计区域 — 卡片 + 图表

#### 统计卡片（3个，图表上方一行）

| 卡片         | 数据源                               | 颜色 |
| ------------ | ------------------------------------ | ---- |
| 历史记录总数 | `completed` 记录数                   | 蓝色 |
| 降级率       | `productGrade === 'B'` / 总数 × 100% | 橙色 |
| 本月评审数   | 当月 `completed` 记录数              | 绿色 |

#### 图表（3个横排，使用 Chart.js / vue-chartjs）

| 图表         | 类型                   | 数据                                                | 交互                     |
| ------------ | ---------------------- | --------------------------------------------------- | ------------------------ |
| 缺陷类型分布 | **Doughnut（环形图）** | 按 `entryData.defectReasons` 统计各缺陷类型出现频次 | hover 显示具体数量和占比 |
| 降级趋势     | **Line（折线图）**     | 按日期统计每天的评审总数 + 降级数                   | 双线对比，支持看趋势     |
| 机台缺陷对比 | **Bar（柱状图）**      | 按 `machine` 分组统计降级卷数                       | 各机台横向对比           |

**图表区域高度**：固定 200px，不参与页面滚动。

---

### 3.2 筛选区域 — 多维度组合过滤

```
┌─────────────────────────────────────────────────────────────────┐
│ 🔍 卷号  │ 开始日期  │ 结束日期  │ 缺陷类型▼ │ 评审结论▼ │ 等级▼ │ 机台▼ │ [重置] │
└─────────────────────────────────────────────────────────────────┘
```

| 筛选项   | 控件类型                  | 选项来源                        | 匹配逻辑                                   |
| -------- | ------------------------- | ------------------------------- | ------------------------------------------ |
| 卷号     | `<input>` 文本搜索        | —                               | `coilNo` 模糊匹配（包含即命中）            |
| 时间范围 | `<input type="date">` × 2 | —                               | `reviewTime` 在 [开始, 结束] 区间内        |
| 缺陷类型 | `<select>` 单选           | `defectTypeOptions`（27种）     | `entryData.defectReasons` 数组中包含选中值 |
| 评审结论 | `<select>` 单选           | `mainConclusionOptions`（16种） | `reviewData.mainConclusion` 精确匹配       |
| 产品等级 | `<select>` 单选           | 全部/一级品A/二级品B            | `reviewData.productGrade` 匹配             |
| 机台     | `<select>` 单选           | 1#/2#/4#/5#                     | `machine` 精确匹配                         |
| 重置     | `<button>`                | —                               | 清空所有筛选条件                           |

筛选结果**同步影响图表数据**：图表仅展示筛选后的记录统计，实现"筛选即分析"。

---

### 3.3 结果表格

| 列       | 宽度  | 数据                                        |
| -------- | ----- | ------------------------------------------- |
| 大卷号   | 120px | `coilNo`，等宽字体                          |
| 车次     | 70px  | `tripNo`                                    |
| 机台     | 60px  | `machine`，badge样式                        |
| 客户     | 80px  | `customer`                                  |
| 规格     | 120px | `thickness`μm × `width`mm                   |
| 缺陷类型 | 140px | `entryData.defectReasons` 数组，tag标签展示 |
| 评审结论 | 100px | `reviewData.mainConclusion`，颜色编码 badge |
| 产品等级 | 70px  | A/B，badge样式                              |
| 评审人   | 80px  | `reviewer`                                  |
| 评审时间 | 100px | `reviewTime` MM-DD HH:mm                    |
| 操作     | 60px  | [查看] 按钮，点击打开抽屉                   |

**排序**：默认按评审时间倒序（最近的在最上面）。

**样式**：复用 `QualityView.vue` 的表格样式（`.data-table`、badge、状态文字等），与质量评审列表保持视觉一致。

## 四、数据变更

### Mock 数据扩充

当前 `quality-review-data.js` 仅有 3 条 `completed` 记录，无法支撑筛选和图表演示。新增约 12 条历史数据，覆盖：

| 维度     | 覆盖范围                           | 目的                     |
| -------- | ---------------------------------- | ------------------------ |
| 日期     | 3月1日 ~ 3月7日（7天）             | 支撑趋势折线图           |
| 机台     | 1# / 2# / 4# / 5#                  | 支撑机台对比柱状图       |
| 缺陷类型 | 针孔/气道/辊印/油斑/白条/下榻/压坑 | 支撑缺陷分布饼图         |
| 评审结论 | 让步放行/改切/正常处理/转精切/扒废 | 支撑结论筛选             |
| 产品等级 | A: 约60%, B: 约40%                 | 支撑等级筛选和降级率统计 |
| 客户     | 精切/宁德时代/河北兴恒/弘力        | 支撑客户维度             |
