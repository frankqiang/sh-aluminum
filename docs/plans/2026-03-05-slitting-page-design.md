# 分切排产页面设计文档

> **日期**：2026-03-05
> **目标**：实现分切排产页面的完整功能
> **技术栈**：Vite + Vue（与现有项目一致）
> **设计风格**：浅色主题，经典ERP表格风格（与轧机页面一致）

---

## 一、需求总结

基于 brainstorming 讨论确认的需求：

| 决策项       | 结论                                                                                           |
| ------------ | ---------------------------------------------------------------------------------------------- |
| 列表展示方式 | 统一表格 + 机台列 + 筛选（非按机台分组）                                                       |
| 列信息量     | 完整版：序号、机台、母卷号、合金、规格、执行单号、客户、产品类型、评审、切刀规格、废料率、状态 |
| 详情面板     | 有，右侧滑出面板，按原型设计文档 §3.3                                                          |
| 新增弹窗     | 有，包含完整切刀方案分段录入 + 自动废料率计算                                                  |
| 组件架构     | 方案A：新建独立组件（不复用轧机的 PlanTable）                                                  |

## 二、组件架构

### 2.1 文件结构

```
src/
├── views/
│   └── SlittingView.vue              # 主页面
├── components/
│   ├── SlittingPlanTable.vue          # 分切计划表格
│   ├── SlittingDetailPanel.vue        # 右侧详情面板
│   └── AddSlittingPlanModal.vue       # 新增分切计划弹窗
└── data/
    ├── mock.js                        # 已有数据（轧机/分切设备/计划等）
    ├── slitting-material-pool.js      # 待分切料池（母卷+评审信息）
    └── order-pool.js                  # 订单池（可匹配的订单列表）
```

### 2.2 组件职责

| 组件                       | 职责                                  | Props                 | Events                    |
| -------------------------- | ------------------------------------- | --------------------- | ------------------------- |
| `SlittingView.vue`         | 页面容器，管理筛选/分页/弹窗/面板状态 | —                     | —                         |
| `SlittingPlanTable.vue`    | 渲染表格，处理行样式（评审高亮）      | `data`                | `@view(row)`              |
| `SlittingDetailPanel.vue`  | 渲染详情：来料+评审+切刀+说明+记录    | `visible`, `row`      | `@close`                  |
| `AddSlittingPlanModal.vue` | 新增表单：基本信息+切刀分段+实时计算  | `visible`, `machines` | `@close`, `@submit(data)` |

### 2.3 数据流

```
mock.js (slittingPlans + slittingMachines)
    ↓
SlittingView.vue
    ├── 筛选/搜索/分页 → computed 过滤数据
    ├── → SlittingPlanTable（展示过滤后的数据）
    │       └── 点击「查看」→ emit('view', row)
    │                            ↓
    ├── → SlittingDetailPanel（selectedRow）
    │
    ├── 点击「新增」→ addModalVisible = true
    │                    ↓
    └── → AddSlittingPlanModal
            └── 提交 → emit('submit', data) → toast 提示
```

### 2.4 复用现有组件

- `ToastNotification.vue` — 提交成功提示
- `CustomSelect.vue` — 弹窗中的下拉选择
- 样式系统 — 使用 `style.css` 中已定义的 CSS 变量

| 列名     | 字段                  | 宽度  | 说明                           |

| 执行单号 | orderNo               | 130px | 如 "SZ260128..."               |
| 客户     | customer              | 90px  | 未匹配显示灰色 "(未匹配)"      |
| 产品     | productType           | 65px  | badge样式：电池箔/双零箔       |
| 评审     | reviewStatus          | 60px  | ✅已评审 / ⚠️待评审 / ❌未提交 |

| 废料率   | cuttingPlan.wasteRate | 70px  | 如 "11.1%"，高值标红           |

### 4.2 行状态样式

| 状态             | 行背景色                       | 说明                 |
| ---------------- | ------------------------------ | -------------------- |

## 六、新增弹窗：AddSlittingPlanModal.vue（重构版）

> **设计原则**：模拟计划员"看料→看评审→选订单→排刀"的实际决策流程。
> **调研依据**：排产模块调研总结 §4.2-4.4、质量模块调研总结 §4.5、MES项目方案 §3.2

### 6.1 弹窗配置

- 宽度：900px（需要容纳更多信息列）
- 居中弹出，有半透明遮罩
- 分为 5 个区块，对应计划员的实际思考顺序

### 6.2 整体信息架构

```
❶ 来料选择区（选母卷 + 选机台）
     ↓ 选择母卷后自动带出材料信息
❷ 质量评审区（只读参考 — 评审结论 + 处理指令 + 有效宽度）
     ↓ 计划员据此决定怎么切
❸ 切刀方案设计区（核心交互 — 添加段、绑定订单、废料率计算）
     ↓
❹ 计划说明区（选填文本）
     ↓
❺ 操作按钮（取消 / 确认添加）
```

### 6.3 ❶ 来料选择区

| 字段     | 类型         | 交互说明                                                                      |
| -------- | ------------ | ----------------------------------------------------------------------------- |
| 母卷号   | CustomSelect | 从「待分切料池」（`slitting-material-pool.js`）中选择，选中后自动填充下方信息 |
| 机台设备 | CustomSelect | 选择分切机台；若产品是电池箔，自动过滤掉无检测设备的机台                      |

选择母卷后，**以下字段自动带出（只读展示）**：

| 字段     | 说明            |
| -------- | --------------- |
| 合金     | 如 1060         |
| 产品类型 | 电池箔 / 双零箔 |
| 宽度     | 如 1350mm       |
| 厚度     | 如 13μm         |
| 长度     | 如 24000m       |
| 重量     | 如 850kg        |

### 6.4 ❷ 质量评审区（只读参考）

> **核心价值**：计划员必须先知道"这卷料哪里有缺陷、哪里要切掉"，才能决定切刀方案。
> 调研原文："看质量评审有没有需要切除的区域 → 计算有效宽度 → 在脑中搜索订单池"

选择母卷后，从 mock 数据中读取该母卷的评审信息，**只读展示**：

| 展示项       | 说明                                                          |
| ------------ | ------------------------------------------------------------- |
| 评审状态     | ✅已评审 / ⚠️待评审，带颜色标签                               |
| 评审结论     | 如"让步放行"、"正常分切"、"改切"                              |
| 处理指令列表 | 多条，每条含：侧(Q/C)、范围(start-end mm)、缺陷类型、处理方式 |
| **有效宽度** | = 原始宽度 - 需切除区域总宽度。**切刀方案的核心基准**         |
| 待评审提示   | 若未评审，显示黄色警告"该母卷尚未完成质量评审，请谨慎排产"    |

样式与 `SlittingDetailPanel.vue` 的评审区块保持一致，但更紧凑（水平布局）。

### 6.5 ❸ 切刀方案设计区（核心交互）

#### 段类型定义

| 类型   | 说明                   | 是否绑定订单        |
| ------ | ---------------------- | ------------------- |
| `边丝` | 两侧切掉的边料，8-15mm | ❌ 无需             |
| `订单` | 绑定一个具体订单       | ✅ **必须选择订单** |
| `切除` | 质量缺陷需要切掉的部分 | ❌ 填写缺陷说明     |
| `余料` | 切不完的剩余废料       | ❌ 无需             |

#### 段列表（动态表格）

| 列       | 边丝段 | 订单段               | 切除段   | 余料段 |
| -------- | ------ | -------------------- | -------- | ------ |
| 类型     | select | select               | select   | select |
| 宽度(mm) | number | **只读**（来自订单） | number   | number |
| 绑定订单 | —      | **订单选择器**       | —        | —      |
| 客户     | —      | 只读（自动带出）     | —        | —      |
| 米数范围 | —      | 只读（自动带出）     | —        | —      |
| 管芯规格 | —      | 只读（自动带出）     | —        | —      |
| 说明     | 自动   | —                    | 缺陷描述 | —      |
| 操作     | 删除   | 删除                 | 删除     | 删除   |

#### 订单选择器交互

当段类型为「订单」时，点击「选择订单」弹出订单选择下拉/弹窗：

- **数据来源**：`order-pool.js` 中的订单列表
- **自动筛选**：合金匹配当前母卷合金 + 宽度 ≤ 有效宽度
- **选中后自动带出**：执行单号、客户名、宽度、米数范围(min~max)、管芯规格
- **宽度字段变为只读**（由订单决定，不可手改）
- **同一订单不可重复选择**

#### 实时计算区

每次宽度字段变更时自动重算：

- **母卷总宽**：来自母卷信息
- **有效宽度**：来自评审信息（母卷宽度 - 需切除区域）
- **已分配宽度**：所有段宽度之和
- **剩余宽度**：母卷宽度 - 已分配宽度
- **废料宽度**：边丝 + 切除 + 余料的宽度之和
- **废料率**：废料宽度 ÷ 母卷宽度 × 100%
- **校验**：
  - 已分配 > 母卷宽度时 → ❌ 红色报错"超出母卷总宽"
  - 订单段宽度总和 > 有效宽度时 → ⚠️ 橙色警告"订单宽度超出有效宽度"

### 6.6 ❹ 计划说明

textarea 自由文本输入，如"合理计算米数尽量不废料，装框单备注管芯壁厚"。

### 6.7 按钮与校验

- [取消] — 关闭弹窗
- [确认添加] — 校验规则：
  - 必须选择母卷号 ✅
  - 必须选择机台 ✅
  - 至少有 1 个订单段 ✅
  - 已分配宽度 ≤ 母卷宽度 ✅
  - 校验通过后提交，弹 toast 提示"分切计划添加成功！"
