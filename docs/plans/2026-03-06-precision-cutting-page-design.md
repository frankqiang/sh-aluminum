# 精切排产页面设计文档

> **日期**：2026-03-06
> **目标**：实现精切排产页面的完整功能（原型演示）
> **技术栈**：Vite + Vue（与现有项目一致）
> **设计风格**：浅色主题，经典 ERP 表格风格（与分切页面一致）

---

## 一、需求确认（Brainstorming 结论）

| 决策项         | 结论                                                                      |
| -------------- | ------------------------------------------------------------------------- |
| 电晕与精切关系 | 同一台机器一步完成，不单独排电晕工序                                      |
| 计划创建方式   | 计划员手动新增（与分切排产对称）                                          |
| 列表展示方式   | 一行 = 一个子卷；多订单在"精切方案"列内多行展示（与分切"切刀规格"列一致） |
| 精切方案录入   | 从订单池选订单，系统自动带出宽度/米数/客户（与分切逻辑对称）              |
| 段设计         | 需要完整的段设计（边丝 + 订单 + 余料）+ 废料率实时计算                    |
| 质量评审展示   | 弹窗中展示完整的分切评审信息区（只读参考，与分切弹窗一致）                |
| 评审状态       | 列表需要评审列 + 评审卡点（待评审行黄色高亮，不可开始精切）               |
| 精切机台       | 49 台能力基本一致，选机台时无需按产品类型过滤                             |

## 二、组件架构

### 2.1 文件结构

```
src/
├── views/
│   └── PrecisionCuttingView.vue          # 主页面（筛选/分页/弹窗/面板状态管理）
├── components/
│   ├── PrecisionPlanTable.vue             # 精切计划列表表格
│   ├── PrecisionDetailPanel.vue           # 右侧详情面板
│   └── AddPrecisionPlanModal.vue          # 新增精切计划弹窗
└── data/
    ├── precision-material-pool.js         # 待精切料池（分切子卷列表）
    └── order-pool.js                      # 复用现有订单池（无需新建）
```

### 2.2 组件职责

| 组件                        | 职责                                                | Props            | Events                    |
| --------------------------- | --------------------------------------------------- | ---------------- | ------------------------- |
| `PrecisionCuttingView.vue`  | 页面容器，管理筛选/分页/弹窗/面板状态               | —                | —                         |
| `PrecisionPlanTable.vue`    | 渲染表格，处理行样式（评审高亮）                    | `data`           | `@view(row)`              |
| `PrecisionDetailPanel.vue`  | 渲染详情：来料 + 评审 + 精切方案 + 记录             | `visible`, `row` | `@close`                  |
| `AddPrecisionPlanModal.vue` | 新增表单：来料选择 + 评审参考 + 段设计 + 废料率计算 | `visible`        | `@close`, `@submit(data)` |

### 2.3 数据流

```
precision-material-pool.js  +  order-pool.js
         ↓
PrecisionCuttingView.vue（状态管理中心）
    ├── 筛选/分页 → computed 过滤数据
    │
    ├── → PrecisionPlanTable（展示计划列表）
    │       └── 点击「查看」→ emit('view', row)
    │                             ↓
    ├── → PrecisionDetailPanel（selectedRow）
    │       └── emit('close')
    │
    └── 点击「新增」→ showAddModal = true
                         ↓
         → AddPrecisionPlanModal
                 └── emit('submit', data) → toast 提示
```

### 2.4 复用现有组件

| 组件                    | 用途                           |
| ----------------------- | ------------------------------ |
| `ToastNotification.vue` | 提交成功提示                   |
| `CustomSelect.vue`      | 弹窗中的下拉选择               |
| `order-pool.js`         | 精切订单数据（复用，无需新建） |
| CSS 变量系统            | `style.css` 中已定义           |

```javascript
  frameNo: '3195',                 // 框号（分切下料时确定）
  alloy: '1100',
  productType: '电池箔',
  thickness: 13,                   // 厚度 μm

  length: 23000,                   // 子卷长度 m
  weight: 850,                     // 重量 kg
  // 分切质量评审（来自分切工序，决定精切方案和顺序）

    grade: '一级品',

      // 类型 width：宽度方向缺陷（影响有效宽度）
      { locationType: 'width', side: 'Q侧', position: 100, defectType: '针孔', action: '切除' },
      { locationType: 'width', side: 'C侧', position: 25,  defectType: '针孔', action: '切除' },
      { locationType: 'width', side: 'Q侧', position: 620, defectType: '孔洞', action: '避留白' },

    effectiveWidth: 1075,           // 有效宽度 = 1200 - 100 - 25
  }
}
```

### 3.3 废料率计算逻辑

精切废料率统一按**宽度方向**计算（与分切一致）：

```
宽度同时切（订单宽度之和 + 边丝 ≤ 子卷总宽）：
  废料宽度 = 左边丝 + 右边丝 + 余料
  余料 = 子卷宽度 - 左边丝 - 右边丝 - Σ订单宽度
  废料率 = 废料宽度 / 子卷宽度 × 100%

长度顺序切（订单宽度之和 > 子卷总宽）：
  每道废料宽度 = 子卷宽度 - 该订单宽度 - 左边丝 - 右边丝
  整体废料率 = 各道废料宽度均值 / 子卷宽度 × 100%
```

## 六、详情面板：PrecisionDetailPanel.vue

### 6.1 面板配置

- 宽度：420px，从右侧滑入
- 有半透明遮罩
- 关闭方式：✕ 按钮 / 点击遮罩

### 6.2 内容区域（从上到下）

#### ① 来料信息

- 子卷号、母卷号与框号（同行展示，方便找料）
- 合金、厚度、宽度、长度

#### ② 分切评审信息

- 评审状态标签（✅已评审 / ⚠️待评审 / ❌未提交）

- 处理指令列表，区分缺陷方向：

- 有效宽度（宽度方向缺陷切除后的余量）
- **待评审状态**：`⚠️ 该子卷尚未完成质量评审，请谨慎排产`

#### ③ 精切方案

- 电晕要求（1遍 / 2遍 + 达因值≥33）
- **多订单顺序说明**（仅当 plan.length > 1 时显示）
- 订单顺序表格（顺序 / 客户 / 宽度 / 米数范围 / 等级）
- 废料率

#### ④ 修改记录 & 操作按钮

- 时间 + 操作人 + 内容列表
- [编辑计划] + [删除]（原型演示，点击仅 toast 提示）

## 七、新增弹窗：AddPrecisionPlanModal.vue

> **设计原则**：模拟计划员"看来料 → 看评审 → 选机台 → 设计切法 → 填说明"的实际决策流程。

### 7.1 弹窗配置

- 宽度：900px，居中弹出，半透明遮罩
- 信息区块对应计划员实际思考顺序

### 7.2 整体信息架构

```
❶ 来料选择区（选子卷 + 选机台）
     ↓ 选定子卷后自动带出来料信息与评审信息
❷ 质量评审区（只读参考——评审结论 + 处理指令 + 有效宽度）
     ↓ 据此决定电晕遍数、切割顺序
❸ 精切方案设计区（核心：段设计 + 废料率实时计算）
❹ 顺序说明（多订单时显示）
❺ 计划备注（选填）
❻ 操作按钮（取消 / 确认添加）
```

### 7.3 ❶ 来料选择区

| 字段     | 类型         | 交互说明                                                     |
| -------- | ------------ | ------------------------------------------------------------ |
| 子卷号   | CustomSelect | 从 `precision-material-pool.js` 选择；选中后自动带出来料信息 |
| 机台设备 | CustomSelect | 1#~49#（精切机能力一致，无需按产品类型过滤）                 |

选择子卷后**自动带出（只读）**：合金、产品类型、厚度、宽度、长度、框号

### 7.4 ❷ 质量评审区（只读参考）

- 评审状态标签
- 主结论、产品等级
- 处理指令（📏 长度方向 / ↔️ 宽度方向 区分显示）
- 有效宽度（自动计算）
- **待评审时**显示黄色警告条

样式与 `PrecisionDetailPanel.vue` 的评审区块保持一致，但更紧凑。

### 7.5 ❸ 精切方案设计区（核心交互）

#### 电晕设置

```
电晕遍数: ◉ 1遍  ○ 2遍     达因值要求: ≥33（固定）
```

#### 段类型定义

| 类型   | 说明                       | 宽度来源                 |
| ------ | -------------------------- | ------------------------ |
| `边丝` | 两侧边料（各 8mm，可编辑） | 手填（默认8mm）          |
| `订单` | 绑定一个精切订单           | **只读**（来自订单宽度） |
| `余料` | 宽度方向剩余废料           | 自动计算                 |

#### 段列表（动态表格）

| 列       | 边丝             | 订单段                  | 余料       |
| -------- | ---------------- | ----------------------- | ---------- |
| 类型     | select           | select                  | select     |
| 宽度(mm) | [number, 可编辑] | [只读，来自订单]        | [自动计算] |
| 绑定订单 | —                | **[选择订单]** 弹出筛选 | —          |
| 客户     | —                | 只读（自动带出）        | —          |
| 米数范围 | —                | 只读（自动带出）        | —          |
| 等级     | —                | 只读（自动带出）        | —          |
| 操作     | —                | [删除]                  | —          |

`[+ 添加订单段]` 按钮，点击追加一个空白订单段。

#### 订单选择器交互

点击「选择订单」时弹出订单列表：

- **自动过滤**：合金匹配当前子卷 + 宽度 ≤ 有效宽度
- **选中后自动带出**：执行单号、客户、宽度（只读）、米数范围、产品等级
- **同一订单不可重复选择**

#### 实时计算区

```
子卷总宽:    1200mm
已分配宽度:   1530mm  ← 超出时数字变红
剩余宽度（余料）: -330mm

废料率: 22.5%（正常模式下显示）
```

#### 宽度校验逻辑

| 场景                    | 提示类型    | 说明                                       |
| ----------------------- | ----------- | ------------------------------------------ |
| 已分配 ≤ 子卷宽度       | ✅ 正常     | 宽度同时切，有余料                         |
| 已分配 > 子卷宽度       | ⚠️ 橙色警告 | `"总宽超出，将按长度顺序切割（高废料率）"` |
| 单个订单宽度 > 有效宽度 | ❌ 红色报错 | `"单个订单宽度超出有效宽度"`               |

### 7.6 ❹ 顺序说明（多订单时显示）

仅当 plan 中存在多个订单段时显示：

```
顺序说明（可选）：
[textarea] 如"底部有5800米缺陷，先切短单消化缺陷"
```

展示在详情面板的"⚠️ 多订单顺序说明"区块。

### 7.7 ❺ 计划备注（选填）

文本域，如"过电晕，要求达因值≥33"。

### 7.8 校验规则 & 按钮

| 项目   | 规则                                      |
| ------ | ----------------------------------------- |
| 子卷号 | 必选 ✅                                   |
| 机台   | 必选 ✅                                   |
| 订单段 | 至少 1 个 ✅                              |
| 宽度   | 单个订单宽度 ≤ 有效宽度（否则不可提交）✅ |

`[取消]` — 关闭弹窗  
`[确认添加]` — 校验通过后提交，toast 提示"精切计划添加成功！"

## 九、设计原则

| 原则         | 说明                                                |
| ------------ | --------------------------------------------------- |
| 浅色主题     | 白色背景 + 灰色边框 + 蓝色主色调                    |
| 表格为核心   | 接近 Excel 使用习惯                                 |
| 状态色彩统一 | ✅绿=完成 / 🔵蓝=进行中 / 🟡黄=待评审 / ⚪白=计划中 |
| 右侧详情面板 | 列表行点击后右侧滑出，无需跳转                      |
| 评审卡点     | 待评审行高亮，系统提示不可精切，与分切逻辑一致      |
| 修改留痕     | 每条计划都有修改记录区域                            |
| 多行精切方案 | 精切方案列内多行展示①②，与分切切刀规格列视觉一致    |
