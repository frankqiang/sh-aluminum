# 库位地图重构设计方案

> **设计日期**：2026-03-11
> **设计依据**：排产模块调研总结、质量模块调研总结、MES系统项目方案v2
> **目标页面**：`MaterialLocationView.vue`
> **目标用户**：管理者 / 计划员（办公室电脑端）

---

## 一、背景与问题

### 1.1 调研发现的核心痛点

- **找料困难**：约1000个分切子卷堆在现场，精切人员凭经验找料
- **位置信息模糊**：有料框但没有库位概念，料框会被搬动
- **状态不可见**：无法实时知道「有多少料等待精切」「哪些料已经等了很久」
- **呆滞料隐藏**：长期未处理的料卷可能被遗忘
- **未评审先生产**：客户明确要求系统卡住——评审不完成不能切

### 1.2 当前页面问题

| 问题               | 说明                                     |
| ------------------ | ---------------------------------------- |
| 虚拟网格与现场脱节 | 用「A排1号」虚拟坐标，现场不存在对应标记 |
| 缺少料框号         | 现场实际用料框号管理，页面完全没有体现   |
| 料卷状态未展示     | 格子只显示停放天数，看不出能不能切       |
| 未评审无标识       | 客户最需要的卡点功能缺失                 |
| 弹窗信息不足       | 缺少工序轨迹、关联订单、处理指令醒目展示 |
| 无呆滞料统计       | 管理者无法一眼了解全厂呆滞情况           |

## 二、设计决策记录

| 决策点   | 选项                                 | 最终决策                       | 理由                           |

| 料框结构 | -                                    | 每框2-3卷，可堆叠成垛（2-4层） | 基于现场照片确认               |

| 改进范围 | A.核心问题 B.较全面重构 C.全面       | **B**                          | 核心缺口 + 弹窗增强 + 呆滞统计 |
| 展示方案 | A.料框卡片网格 B.车间平面图 C.纯列表 | **A**                          | 兼具视觉直观性和业务真实性     |

## 三、整体页面结构

```
┌─────────────────────────────────────────────────────────────────┐
│  页面标题 + 搜索框                                               │
├─────────────────────────────────────────────────────────────────┤
│  顶部统计栏：各区域卷数/框数 + 呆滞预警（>7天、>14天数量）        │
├─────────────────────────────────────────────────────────────────┤
│  工具栏：区域Tab + 着色模式切换 + 图例                            │
├─────────────────────────────────────────────────────────────────┤
│  料框卡片网格（当前区域的所有料框）                                │
│    每张卡片 = 一个料框，内部按层展示料卷                          │
├─────────────────────────────────────────────────────────────────┤
│  点击料卷 → 增强版详情弹窗                                       │
└─────────────────────────────────────────────────────────────────┘
```

### 3.1 顶部统计栏

新增呆滞预警概览，管理者一眼掌握全厂状况：

```
分切区 25卷/18框    轧机区 12卷/8框    精切区 10卷/6框
                                        呆滞预警：>7天 8卷  >14天 3卷
```

### 3.2 工具栏

- **区域 Tab**：分切区 / 轧机区 / 精切区（保留当前设计）
- **着色模式切换**：
  - 默认：按停放时长（🟢<3天 🟡3-7天 🟠7-14天 🔴>14天）
  - 可切换：按料卷状态（各状态用不同颜色）
- **图例**：根据当前着色模式动态切换

### 3.3 料框卡片网格

每个料框一张卡片，以 CSS Grid/Flexbox 自适应排列。

### 4.2 料卷小格信息

| 信息      | 停放时长模式            | 状态模式               |
| --------- | ----------------------- | ---------------------- |
| 着色      | 绿/黄/橙/红             | 按状态着色             |

| 客户·宽度 | `弘力·654`              | 同左                   |

```

│           ∅ 当前无料卷                 │
└───────────────────────────────────────┘
```

虚线边框、灰色调，视觉降级。

### 4.4 卡片交互

- **点击料卷小格** → 弹出详情弹窗

- **呆滞料** → 🔴卷格带呼吸灯脉冲动画（保留当前设计）

## 五、增强版详情弹窗

分为 5 个区块：

### 5.1 状态与位置

```
🟡 分切已评审    🔴 已停放 16天
📦 料框 #3195 · 第1层（底层）
📍 分切区
```

- 新增：料框号、堆叠层数
- 堆叠层如果是底层，显示「底层」提示（意味着取料需搬开上面的框）

### 5.2 基础信息

保持当前设计：母卷号、规格、合金、客户、产品类型、重量、入库时间。

### 5.3 质量评审（醒目展示）

```
让步放行 · 一级品

处理指令：
  🔴 Q侧 0-80mm   针孔 → 切除
  🟠 C侧 0-40mm   针孔 → 切除
  🟡 Q侧 335mm    辊印 → 避留白
```

- 处理指令用**彩色条目**展示，红色=切除、橙色=改切/降级、黄色=避留白/观察
- 未评审时：标题显示 `⚠️ 待评审` 红色警示

### 5.4 关联订单（新增）

```
📋 SZ2601280123
客户：中航锂电
交期：2026-03-15  ⚠️ 还剩4天
```

- 无关联订单时显示灰色「未关联订单」

### 5.5 工序流转轨迹（新增）

```
● 2026-02-22  胚料出库     仓库
│
● 2026-02-22  开始轧制     8#精轧机
│
● 2026-02-23  轧机评审     李工
│
● 2026-02-23  开始分切     4#分切机
│
● 2026-02-23  分切评审     王工
│
◌ 等待精切...（已等16天）
```

- 纵向时间线，展示 movements 数据
- 最后一步如果是等待状态，用空心圆表示

## 六、数据模型变更

### 6.1 新增：料框数据

```javascript
{
  frameId: 'F3195',          // 料框唯一ID
  frameNo: '3195',           // 现场框号
  zone: 'slitting',          // 所属区域
  stackLevel: 1,             // 在垛中的层（1=底层）
  stackTotal: 3,             // 该垛总层数
  stackId: 'STK-A01',        // 所属垛编号
}
```

### 6.2 料卷数据变更

| 字段            | 变更     | 说明                 |
| --------------- | -------- | -------------------- |
| `row` / `col`   | **删除** | 不再使用虚拟网格坐标 |
| `frameId`       | **新增** | 关联到料框           |
| `orderId`       | **新增** | 关联订单号           |
| `orderDeadline` | **新增** | 订单交期             |
| `orderNo`       | **新增** | 订单执行单号         |

### 6.3 区域定义变更

```javascript
// 当前
{ id: 'slitting', rows: ['A','B','C','D'], cols: 10, ... }

// 改后：移除 rows/cols，料框独立管理
{ id: 'slitting', name: '分切区', description: '...' }
```
