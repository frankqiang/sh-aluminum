> 日期：2026-03-12
> 状态：待实施

> 关联：aluminum-expert 技能 + 行业最佳实践

---

## 1. 背景与目标

### 1.1 背景

客户提供了两份实际质量报表：

- **日报1**：主要电池箔母卷放行比例（包括备注缺陷放行），按日×按客户的折线图，涵盖莱尔、优箔、冠业、标兵等终端客户
- **日报2**：电池箔不合格统计日报，包含三层信息：
  1. 总览：母卷/精切 × 一期/二期的产量、不合格量(kg)、合格率
  2. 母卷不合格原因明细：按"坯料原因"和"铝箔原因"分类，含重量和占比
  3. 精切不合格原因明细：按具体原因列出重量和占比

### 1.2 决策

经 brainstorming 分析，决定：

1. **追溯查询页面（QualityTraceView）**：移除所有图表，仅保留统计卡片 + 筛选器 + 数据表格 + 右侧抽屉。聚焦"查找特定料卷记录"的操作型需求
2. **新建质量统计页面（QualityStatsView）**：承载所有图表和统计分析功能。聚焦"分析质量趋势和问题"的分析型需求

### 1.3 用户角色

主要使用者：**质量主管/质量经理**
核心诉求：

- 本周/本月质量怎么样？
- 哪台机效果最差？
- 哪个客户收到的问题料最多？
- 降级的主要原因是什么？跟上一期比有改善吗？

## 2. 页面结构

### 2.1 导航位置

侧边栏"质量管理"菜单下新增"质量统计"项：

```
质量管理
  ├─ 质量评审
  ├─ 追溯查询    （已有，移除图表）
  └─ 质量统计    （新增）
```

路由：`/quality/stats`

### 2.2 整体布局

```
┌──────────────────────────────────────────────────────────────┐
│  顶部：页面标题 + 筛选条                                       │

├──────────────────────────────────────────────────────────────┤
│  区块一：KPI 卡片（6个）                                       │
│  [总产量] [一次合格率↑] [让步放行率↓] [二级品率↓] [不合格量] [评审总数] │
├─────────────────────────┬────────────────────────────────────┤
│  区块二左：               │  区块二右：                         │
│  一次合格率趋势（按日）     │  客户放行率趋势（按日×按客户）       │
│  双线：合格率 + 让步放行率  │  每个客户一条折线，多色区分          │

│ (环形图) │ (柱状图) │                                         │
├────────┴────────┴───────────────────────────────────────────┤
│  区块四a：质量总览表格                                         │
│  种类 | 车间 | 总产量(kg) | 不合格量(kg) | 合格率               │
├──────────────────────────────────────────────────────────────┤
│  区块四b：降级原因分类明细表                                    │

│  每行: 原因名 | 重量(kg) | 占比(%)                              │
│  有小计行和总计行                                               │
└──────────────────────────────────────────────────────────────┘
```

## 3. 各区块详细规格

### 3.1 筛选条

| 筛选项   | 类型              | 默认值   | 说明                     |
| -------- | ----------------- | -------- | ------------------------ |
| 日期范围 | 日期选择器(起-止) | 最近30天 | 筛选评审时间             |
| 车间     | 下拉              | 全部     | 一期/二期/全部           |
| 机台     | 下拉              | 全部     | 1#/2#/4#/5#/全部         |
| 客户     | 下拉              | 全部     | 动态获取数据中的客户列表 |

所有图表和表格**响应式联动**：筛选变化后所有区块数据同步更新。

### 3.2 区块一：KPI 卡片

共6个卡片，横向排列：

| #   | 指标名     | 计算逻辑                                                  | 图标          | 颜色 | 环比               |
| --- | ---------- | --------------------------------------------------------- | ------------- | ---- | ------------------ |
| 1   | 总产量     | `sum(weight)` 所有已评审记录                              | Scale         | 蓝   | 显示环比变化量(kg) |
| 2   | 一次合格率 | `count(qualityJudgment=conform) / total × 100`            | ShieldCheck   | 绿   | 显示环比变化百分点 |
| 3   | 让步放行率 | `count(mainConclusion∈[让步放行,沟通放行]) / total × 100` | AlertTriangle | 橙   | 显示环比，↓为好    |
| 4   | 二级品率   | `count(productGrade=B) / total × 100`                     | TrendingDown  | 红   | 显示环比，↓为好    |
| 5   | 不合格量   | `sum(weight) where productGrade=B`                        | PackageX      | 红   | 显示环比变化量(kg) |
| 6   | 评审总数   | `count(status=completed)`                                 | Database      | 蓝   | 无环比             |

**环比逻辑**：将筛选范围内的数据按时间均分两段，后半段 vs 前半段。

### 3.3 区块二：趋势折线图

#### 3.3a 一次合格率趋势（左侧，50%宽度）

- **类型**：折线图（Line Chart）
- **X轴**：日期（按天）
- **Y轴**：0-100%
- **数据集**：
  - 绿色实线：一次合格率（按日计算 `conform/total × 100`）
  - 橙色虚线：让步放行率（按日计算 `concession/total × 100`）
- **交互**：hover显示当日具体数值
- **来源**：从现有追溯页迁移，逻辑不变

#### 3.3b 客户放行率趋势（右侧，50%宽度）

- **类型**：多线折线图（Multi-line Chart）
- **X轴**：日期（按天）
- **Y轴**：0-100%
- **数据集**：每个终端客户一条折线，不同颜色
  - 宁德时代（蓝色）
  - 河北火恒（绿色）
  - 弘力（橙色）
  - 精切（灰色虚线，作为内部工序参考）
- **放行率计算**：`count(该客户的productGrade=A 或 mainConclusion∈[让步放行,沟通放行,正常处理]) / count(该客户total) × 100`
- **交互**：hover显示某日/某客户放行率；点击图例可隐藏/显示某条线
- **来源**：对应客户日报1

### 3.4 区块三：分析图表

#### 3.4a 缺陷类型分布（左，1/3宽度）

- **类型**：环形图（Doughnut）
- **数据**：各缺陷类型出现的频次
- **颜色**：按频次排序，使用项目配色方案
- **图例**：底部排列
- **来源**：从现有追溯页迁移

#### 3.4b 机台合格率对比（中，1/3宽度）

- **类型**：柱状图（Bar）
- **X轴**：机台（1#, 2#, 4#, 5#）
- **Y轴**：0-100%
- **颜色**：合格率≥60%绿色，<60%红色
- **来源**：从现有追溯页迁移

#### 3.4c 降级原因帕累托图（右，1/3宽度）

- **类型**：组合图（柱状 + 折线）
- **X轴**：缺陷类型，按频次降序排列
- **左Y轴**：降级次数（柱状图）
- **右Y轴**：累计百分比（折线图，0-100%）
- **业务价值**：质量管理经典工具，快速识别"抓住前几个问题就能解决大部分降级"
- **来源**：新增，基于 mock 数据中的 `defectReasons` 字段

### 3.5 区块四：降级明细表格

#### 3.5a 质量总览表格

对应日报2第一层，结构化展示各工序×车间的质量概况：

| 列名         | 说明                                |
| ------------ | ----------------------------------- |
| 种类         | 电池箔母卷 / 电池箔精切             |
| 车间         | 一期 / 二期                         |
| 总产量(kg)   | 该分类下所有记录的总重量            |
| 不合格量(kg) | productGrade=B 的记录总重量         |
| 自行切除(kg) | 改切处理的重量                      |
| 合格率       | (总产量 - 不合格量) / 总产量 × 100% |

**样式**：表格带斑马纹，合格率列带颜色（≥80%绿，60-80%橙，<60%红），总计行加粗。

> 注：当前 prototype 的 mock 数据为分切（母卷）工序，精切数据用模拟数据占位。

#### 3.5b 降级原因分类明细表

对应日报2第二/三层，按"坯料原因 vs 铝箔原因"分类展示：

**布局**：左右两栏对比（一期 vs 二期）

**每栏结构**：

```
┌─────────────────────────────┐
│ 坯料原因                     │
│   针孔     28,739 kg  28.19% │
│   孔洞      3,270 kg   2.71% │
│   压坑      1,571 kg   1.30% │
│   气道      1,392 kg   1.15% │
│   小计              28.19%   │
├─────────────────────────────┤
│ 铝箔原因                     │
│   板形     14,789 kg  14.51% │
│   麻点      1,957 kg   1.92% │
│   米数不足  1,346 kg   1.32% │
│   小计              17.75%   │
├─────────────────────────────┤
│ 总计    46,833 kg    45.93%  │
└─────────────────────────────┘
```

**样式**：

- 分组标题行（坯料原因/铝箔原因）使用浅色背景
- 小计行使用半粗体
- 总计行使用深色背景 + 粗体
- 占比列按数值大小使用渐变色条（数据越大颜色越深）

- 统计卡片（4个 KPI）
