# 生产执行模块设计方案

> **模块定位**：解决订单进度不透明的痛点——提供订单状态跟踪、进度汇总、交期预警、多卷关联  
> **原型范围**：管理视角（订单管理看板），不含现场报工界面  
> **设计日期**：2026-03-09

---

## 一、页面结构

两个子页面，侧边栏「生产执行」分组下：

| 页面     | 路由                   | 侧边栏标签 | 图标            | 定位                    |
| -------- | ---------------------- | ---------- | --------------- | ----------------------- |
| 订单看板 | `/execution/dashboard` | 订单看板   | `BarChart3`     | 管理层快速掌握全局      |
| 订单跟踪 | `/execution/orders`    | 订单跟踪   | `ClipboardList` | 全量订单列表 + 详情面板 |

---

## 二、页面1：订单看板

### 2.1 页面头部

- 标题：「订单看板」
- 副标题：「2026年3月9日 · 全厂订单执行状态总览」
- 右侧：日期切换器（复用排产总览的样式）

### 2.2 统计卡片区（6张卡片，grid 3×2 或 6×1）

| 卡片         | 主数值       | 主题色 | 图标            | 说明                      |
| ------------ | ------------ | ------ | --------------- | ------------------------- |
| 今日活跃订单 | 活跃订单总数 | blue   | `Activity`      | 状态为「生产中」的订单    |
| 逾期订单     | 逾期数量     | red    | `AlertTriangle` | 已过交期但未完成          |
| 3天内到期    | 即将到期数量 | orange | `Clock`         | 交期 ≤ 3天内              |
| 订单准时率   | 百分比       | green  | `CheckCircle`   | 本月按时完成 / 本月已完成 |
| 本月已完成   | 完成数量     | teal   | `PackageCheck`  | 本月状态变为「已完成」的  |
| 生产中订单   | 生产中数量   | purple | `Loader`        | 有关联料卷且未全部完成    |

### 2.3 下方区域布局

采用 **2列 + 全宽** 混排布局：

```
┌──────────────────────────────────┬──────────────────────────────────┐
│  订单状态分布（饼图/环形图）     │  按工序维度分布（堆叠柱状图）    │
│  待排产 / 生产中 / 待包装 / 完成 │  轧机 / 分切 / 精切 各多少订单    │
└──────────────────────────────────┴──────────────────────────────────┘
┌─────────────────────────────────────────────────────────────────────┐
│  交期预警列表（全宽表格）                                            │
│  逾期 + 3天内到期订单，按紧急程度排序                                │
│  列：订单号 | 客户 | 规格 | 交期 | 距交期天数 | 进度 | 紧急等级       │
└─────────────────────────────────────────────────────────────────────┘
┌──────────────────────────────────┬──────────────────────────────────┐
│  按客户维度订单汇总              │  近期完成订单                     │
│  客户名 | 总订单 | 进行中 | 逾期  │  订单号 | 客户 | 完成时间 | 规格  │
└──────────────────────────────────┴──────────────────────────────────┘
```

### 2.4 图表实现方案

由于是原型阶段，**不引入图表库**，使用 CSS 实现：

- 状态分布：CSS 环形图（conic-gradient）+ 图例
- 工序分布：CSS 堆叠条形图（flex bar）

## 三、页面2：订单跟踪

### 3.1 整体布局

采用 **左表格 + 右详情面板** 模式（类似分切排产的 table + panel 模式）：

```
┌──────────────────────────────────────┬──────────────────────────────┐
│  筛选栏                              │                              │
│  ┌──────┬──────┬──────┬──────┬────┐  │                              │
│  │ 状态 │ 客户 │ 产品 │ 交期 │搜索│  │     订单详情面板              │
│  └──────┴──────┴──────┴──────┴────┘  │     （点击左侧某行展开）      │
├──────────────────────────────────────┤                              │
│  订单列表表格                        │  - 订单基本信息               │
│  ┌────────────────────────────────┐  │  - 进度条 + 交期倒计时       │
│  │ 订单号 | 客户 | 规格 | 交期    │  │  - 关联料卷列表（含状态）    │
│  │ 进度   | 状态 | 产品类型      │  │  - 质量评审摘要              │
│  │                                │  │  - 质量追溯链                │
│  │ （可排序、可筛选）             │  │  - 生产时间线                │
│  └────────────────────────────────┘  │                              │
└──────────────────────────────────────┴──────────────────────────────┘
```

### 3.2 筛选栏

| 筛选项     | 类型      | 选项                                     |
| ---------- | --------- | ---------------------------------------- |
| 订单状态   | 下拉单选  | 全部 / 待排产 / 生产中 / 待包装 / 已完成 |
| 客户名称   | 下拉单选  | 从数据中动态提取                         |
| 产品类型   | 下拉单选  | 全部 / 电池箔 / 双零箔                   |
| 交期范围   | 日期范围  | 起止日期（简化为下拉：本周/本月/已逾期） |
| 是否逾期   | 开关/标签 | 「仅看逾期」toggle                       |
| 关键词搜索 | 文本输入  | 订单号 / 合同号 / 卷号模糊搜索           |

### 3.3 订单列表表格

| 列       | 内容                 | 可排序 |
| -------- | -------------------- | ------ |
| 订单号   | NC执行单号           | ✅     |
| 合同号   | ERP合同编号          | ❌     |
| 客户     | 客户名称             | ✅     |
| 产品规格 | 厚度×宽度 + 产品类型 | ❌     |
| 合金     | 合金代号             | ✅     |
| 需求量   | 总卷数 / 总重量      | ✅     |
| 已完成   | 完成卷数 / 重量      | ✅     |
| 进度     | 进度条 + 百分比      | ✅     |
| 交期     | 日期 + 倒计时        | ✅     |
| 状态     | 状态标签             | ✅     |

行内交互：

- 点击某行 → 右侧展开详情面板
- 逾期行高亮红色左边框
- 3天内到期行高亮橙色左边框

### 3.4 详情面板（右侧）

详情面板分为5个区域，垂直堆叠，每个区域有小标题：

#### 区域1：订单基本信息

```
订单号：SZ2601280123           合同号：HT2026-0201
客户：河北兴恒                  产品：电池箔成品
规格：13μm × 872mm             合金：1100
米数要求：12000-13000m          管芯：152.4mm/壁厚4
交期：2026-03-12               状态标签：[生产中]
```

#### 区域2：生产进度

```
进度: ████████████░░░░░░░░ 60%
需求: 50吨 (10卷)  │  已完成: 30吨 (6卷)  │  剩余: 20吨 (4卷)
交期倒计时: 还剩 3 天  ⚠️
```

#### 区域3：关联料卷列表

表格形式：

| 卷号         | 工序阶段 | 状态      | 重量     | 质量评审 |
| ------------ | -------- | --------- | -------- | -------- |
| Y2601301/1-1 | 精切     | ✅ 已完成 | 10吨     | 合格     |
| Y2601301/2-1 | 精切     | 🔄 精切中 | 10吨     | 让步放行 |
| Y2601302/1-1 | 分切     | ⏳ 待分切 | 预计10吨 | 待评审   |
| 待分配       | —        | —         | 约10吨   | —        |

#### 区域4：质量追溯链

展示关联料卷的质量信息追溯链路：

```
母卷 Y2601301 (轧机8#, 2026-03-01)
  ├── 轧机质检: 合格，压延质量正常
  ├── 分切 → 子卷 Y2601301/1-1
  │     ├── 分切质检: 针孔A=52, B=8, 让步放行
  │     └── 精切质检: 合格
  └── 分切 → 子卷 Y2601301/2-1
        └── 分切质检: Q0-80针孔切除, 让步放行
```

#### 区域5：生产时间线

时间轴形式展示关键节点：

```

03-03 08:00  🔬 精切12# — 开始精切
03-03 12:00  ✅ 精切完成 — Y2601301/1-1
```

## 四、订单状态定义

| 状态   | 条件                             | 标签色           |
| ------ | -------------------------------- | ---------------- |
| 待排产 | 无任何关联料卷                   | 灰色             |
| 生产中 | 有关联料卷，且未全部完成         | 蓝色             |
| 待包装 | 所有料卷完成精切，等待包装       | 紫色             |
| 已完成 | 所有卷数/重量达标，已包装        | 绿色             |
| 已逾期 | 当前日期 > 交期 且 状态 ≠ 已完成 | 红色（叠加显示） |

## 七、与现有模块的关系

| 关联模块 | 关联方式                                                                |
| -------- | ----------------------------------------------------------------------- |
| 排产模块 | 订单通过卷号关联到排产计划                                              |
| 质量模块 | 详情面板展示关联料卷的质量评审信息                                      |
| 物料模块 | 关联料卷的状态与物料模块的在制品状态一致                                |
| 订单池   | 生产执行模块独立 mock 数据，结构参考 `order-pool.js` 但增加执行跟踪字段 |

## 八、设计风格

- 与现有模块保持一致的设计系统（CSS变量、StatCard、panel样式）
- 卡片使用渐变主题色（参考排产总览的StatCard）
- 表格/面板使用已有的 `data-table`、`panel` 样式类
- 详情面板参考分切排产的侧边面板交互模式
- 图表使用纯CSS实现（环形图、条形图）
