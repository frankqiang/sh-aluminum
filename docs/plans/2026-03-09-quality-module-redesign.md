# 质量模块重构设计文档

> **日期**：2026-03-09
> **场景**：电池箔分切工序质检（56列字段）
> **依据**：质量模块调研总结.md、MES项目方案v2.md、1月份分切不合格品报表（电池箔Sheet）

---

## 一、当前问题

基于调研文档与真实 Excel 数据的交叉比对，识别出以下问题：

| #   | 问题                                                              | 严重度 |
| --- | ----------------------------------------------------------------- | ------ |
| 1   | "新建记录"弹窗多余 — 数据应由上游系统自动生成，质检员无需手动创建 | 🔴     |
| 2   | 达因值用了 `type="number"` — 实际为 `30/30缩` 格式的自由文本      | 🔴     |
| 3   | 缺少操作员元数据 — 主操手/质检员/班组/日期                        | 🔴     |
| 4   | 处理指令"长度位置"只有单值 — 无法支持排产计算有效长度             | 🟡     |
| 5   | 评审页左栏缺少米数/重量/接头等基本物理参数                        | 🟡     |

## 二、设计决策记录

| 决策项           | 结论                      | 理由                         |
| ---------------- | ------------------------- | ---------------------------- |
| 产品范围         | 仅聚焦电池箔分切          | 原型阶段先做复杂度最高的场景 |
| 面密度标签       | 保持"上/中/下 × 左/右"    | 是取样物理位置，非上下两张箔 |
| 达因值           | 自由文本框                | 支持 `30/30缩` 格式          |
| 针孔 A/B/C/D/E   | 保持独立数字框            | 含义明确，便于统计           |
| 二次评审         | 不区分初次/二次           | 原型阶段简化                 |
| 客户红线         | 保持硬编码                | 后续改为数据驱动             |
| 字段命名统一     | 原型阶段不统一            | 正式开发时再统一             |
| "新建记录"按钮   | 删除                      | 数据来自上游轧机工序自动流转 |
| 处理指令长度字段 | 改为起始/终止两个数字字段 | 排产模块需计算有效长度       |

## 三、状态流转

```
轧机下线 → 系统自动生成记录（待录入）
                ↓ 质检员点击"录入数据"
           待评审（pending_review）
                ↓ 工程师评审
           已完成（completed）
```

状态保持 3 个：`pending_entry` / `pending_review` / `completed`

## 四、改动清单

### 4.1 QualityView.vue（列表页）

- [ ] 删除"+ 新建记录"按钮
- [ ] 删除新建弹窗（dialog）及相关逻辑（newForm、createRecord）
- [ ] 保留 3 个统计卡片（待录入/待评审/已完成）
- [ ] 操作列：`pending_entry` → "录入数据"按钮直接跳转录入页（无弹窗）

### 4.2 QualityEntryView.vue（录入页）

- [ ] 左栏改为只读展示料卷基本信息（卷号/车次/机台/客户/合金/规格），数据从上游带入
- [ ] 达因值：`type="number"` → `type="text"`，placeholder 改为 `如 30/30缩`
- [ ] 新增"操作员信息"区块（区块七）：
  - 主操手（text）
  - 质检员（text）
  - 班组（select：甲/乙/丙）
  - 日期（date，默认当天）
- [ ] 数据模型 `createEmptyEntryData()` 新增对应字段

### 4.3 QualityReviewView.vue（评审页）

- [ ] 左栏"料卷信息"区块补充：米数、重量、接头数
- [ ] 处理指令表增加两列：`长度起始(m)` 和 `长度终止(m)`
  - 原 `lengthPosM`（单值）→ `lengthStartM` + `lengthEndM`（两个数字字段）
- [ ] 数据模型 `createEmptyReviewData()` 中 instructions 默认字段更新

### 4.4 quality-review-data.js（数据层）

- [ ] `createEmptyEntryData()` 新增：`operator`、`inspector`、`shift`、`date`
- [ ] `createEmptyEntryData()` 中 `dyneValue` 类型从 number 改为 string
- [ ] `createEmptyReviewData()` 中 instructions 模板：`lengthPosM` → `lengthStartM` + `lengthEndM`
- [ ] Mock 数据中原 `pending_entry` 状态的记录需包含来自上游的基本信息（卷号等预填，entryData 为 null）
- [ ] 已完成记录的 mock 数据同步更新新字段

## 五、不改的项目

- 面密度 6 点布局（已确认正确）
- 客户红线面板（原型硬编码）
- 跨模块字段命名统一（正式开发再处理）
- 电池箔/双零箔模板切换（仅做电池箔）

## 六、模块联动提醒

> 质量模块的处理指令数据是排产模块计算**有效宽度**和**有效长度**的数据源。
> 修改指令字段结构时，需确认 `slitting-material-pool.js` 和 `precision-material-pool.js` 的数据格式可对应。
> 原型阶段字段名可不一致，但语义和数值内容必须一致。
