# 质量模块原型设计文档

> **模块名称**：3.3 质量模块——解决评审信息不规范和传递断链
> **设计日期**：2026-03-07
> **文档版本**：v1.0
> **设计依据**：质量模块调研总结 v2.0 + MES系统项目方案 v2.0（2315条评审信息分析）

---

## 一、设计背景与目标

### 1.1 解决的核心痛点

| 痛点           | 现状                                       | 系统解法                                         |
| -------------- | ------------------------------------------ | ------------------------------------------------ |
| 评审信息不规范 | 自由文本描述，同一缺陷有多种写法，无法统计 | 结构化表单：缺陷类型/位置/处理方式均使用下拉选择 |
| 传递断链       | 靠共享文件夹/口头/微信通知，容易漏看       | 评审提交后系统即时同步，各环节在各自页面可见     |
| 未评审先生产   | 没有流程卡点，料已到工位但评审未出         | 待评审列表让质检员清楚知道哪些料还未评审         |

### 1.2 原型边界（不做的内容）

- ❌ 多级权限流（质检员→班长→工程师→主管），仅做录入本身
- ❌ 双零箔字段（仅做电池箔，字段最复杂的场景）
- ❌ 与检测设备数据对接（一期手工录入）
- ❌ 上道工序质检数据引入（字段过多，质检员已通过纸质表了解）
- ❌ 历史案例推荐（二期功能）

## 三、页面一：待评审列表页（QualityView.vue）

### 3.1 布局结构

```
┌─────────────────────────────────────────────────────────────┐
│  页面标题：质量评审         统计：待评审 12 | 今日已完成 8    │
├─────────────────────────────────────────────────────────────┤
│  筛选栏：[工序▼] [状态▼] [搜索卷号...]                        │
├──────┬──────────┬──────────┬──────┬──────────┬──────┬───────┤
│  序号 │  卷号    │  来料规格  │ 工序 │  下料时间  │ 状态 │ 操作  │
├──────┼──────────┼──────────┼──────┼──────────┼──────┼───────┤
│  1   │Y260138…  │13×1350mm │ 分切 │ 3小时前   │⏳待评│[进入] │
│  2   │Y260139…  │15×1200mm │ 分切 │ 1小时前   │⏳待评│[进入] │
│  3   │Y260135…  │13×1350mm │ 轧机 │ 昨天      │✅已评│[查看] │
└──────┴──────────┴──────────┴──────┴──────────┴──────┴───────┘
```

### 3.2 关键字段说明

| 字段         | 说明                                                                                |
| ------------ | ----------------------------------------------------------------------------------- |
| **状态**     | `⏳ 待评审`（橙色）/ `✅ 已评审`（绿色）/ `🔄 修改中`（蓝色，用于二次评审覆盖场景） |
| **下料时间** | 相对时间显示（"3小时前"），超过24小时变红色 ⚠️                                      |
| **默认排序** | 待评审靠前，按下料时间升序（最久未评审的排最前），有优先级意识                      |

### 3.3 Mock 数据设计（quality-review-data.js）

需包含至少 4 条不同状态的料卷记录：

- 2 条「待评审」状态（不同工序：分切、轧机）
- 1 条「已评审」（含完整评审数据，用于测试"查看"功能）
- 1 条「修改中」（模拟二次评审场景）

## 四、页面二：评审录入详情页（QualityReviewView.vue）

### 4.1 整体布局

左右两栏，左 35% 只读信息栏 + 右 65% 评审录入区：

```
┌────────────────────────┬─────────────────────────────────────────────┐
│   左栏：料卷基本信息    │           右栏：评审录入区                    │
│   （只读，35%）         │           （可填写，65%）                     │
├────────────────────────┴─────────────────────────────────────────────┤
│                 顶部：← 返回列表   卷号标题        [保存草稿] [提交评审] │
└─────────────────────────────────────────────────────────────────────┘
```

### 4.2 左栏：料卷基本信息（只读）

仅展示本卷自身信息，不引入上道工序质检数据（上道工序字段过多且质检员已通过纸质表掌握）：

```
卷号:     Y2601380
来料卷号:  Y2601301（母卷）
合金:     1060
厚度:     13μm
宽度:     1350mm
米数:     ~24000m
重量:     856kg
机台:     2# 分切机
下料时间:  2026-03-07 09:30
下料已过:  ⚠️ 3小时25分
```

### 4.3 右栏：评审录入区（四个区块）

#### 区块一：主评审结论

| 字段         | 录入方式         | 选项来源                                                                                       |
| ------------ | ---------------- | ---------------------------------------------------------------------------------------------- |
| 主评审结论   | 下拉选择（必填） | 16种（基于2315条分析）：让步放行/正常处理/改切/转精切/沟通放行/改切二级品/扒废/退火后处理/其他 |
| 传递对象备注 | 下拉选择（可选） | 包装备注 / 分切备注 / 精切备注                                                                 |
| 产品等级     | 单选按钮（必填） | 一级品 / 二级品                                                                                |

#### 区块二：检测数据（电池箔，5个子分组）

**【针孔检测】**

| 字段         | 录入方式                  | 说明                     |
| ------------ | ------------------------- | ------------------------ |
| A/B级针孔    | 两个数字输入框（个）      | 来自针检仪PDF抄录        |
| C/D级针孔    | 两个数字输入框（个）      | 来自针检仪PDF抄录        |
| E级针孔      | 数字输入框（个）          | 来自针检仪PDF抄录        |
| 密集型针孔   | 单选（无/有）+ 备注文本框 | 设备无法检测，需人工判断 |
| 离线针孔     | 数字输入框（个）          | 离线针检仪               |
| 纵向针孔密度 | 数字输入框（个/m）        | 针检设备+人工判断        |

**【表面检测】**

| 字段         | 录入方式     | 说明                                       |
| ------------ | ------------ | ------------------------------------------ |
| 表面检测信息 | 多行自由文本 | 人工从几百个缺陷中总结关键问题，无法结构化 |
| 分切质量情况 | 多行自由文本 | 目检发现的其他缺陷描述                     |

**【板型检测】**

| 字段   | 录入方式         | 说明                                |
| ------ | ---------------- | ----------------------------------- |
| 下榻量 | 数字输入框（mm） | 离线板型仪，B/C/D等级已取消只填数值 |

**【其他检测】**

| 字段   | 录入方式           | 说明                   |
| ------ | ------------------ | ---------------------- |
| 面密度 | 数字输入框（g/㎡） | 天平称重计算           |
| 达因值 | 数字输入框（Dyne） | 达因笔检测，电池箔必填 |

**【物料信息】**

| 字段     | 录入方式   | 说明                         |
| -------- | ---------- | ---------------------------- |
| 管芯规格 | 文本输入框 | 如 "63/60"，表示两边突出长度 |
| 框号     | 文本输入框 | 料存放的料框号               |

#### 区块三：处理指令（动态添加行）

列表表格，每行字段横排：

| 列          | 类型               | 选项/说明                                                                                                                                      |
| ----------- | ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| #           | 自动序号           | 1, 2, 3...                                                                                                                                     |
| 缺陷类型    | 下拉选择（必填）   | 27种：针孔/气道/孔洞/辊印/白条/下榻/条纹/压坑/油泥/油斑/板形/闪筋/亮线/直棱/横折/划伤/厚度/擦划/带油/棱印/起皱/黑线/串层/米数/油线/麻点/斜纹   |
| 位置侧      | 下拉选择（必填）   | Q侧 / C侧 / 整幅                                                                                                                               |
| 起始(mm)    | 数字输入框         | 宽度方向起始位置                                                                                                                               |
| 终止(mm)    | 数字输入框         | 宽度方向终止位置                                                                                                                               |
| 长度位置(m) | 数字输入框（可选） | 如：5800（从底部5800米处）                                                                                                                     |
| 处理方式    | 下拉选择（必填）   | 23种：切除/避留白/改切/反开/切二级品/电晕/扒至/下料观察/反开卷/吸边/扒除/复查/降速/切等外/验证/扒废/复测/导卷/薄剪/改制/在线观察/复检/工艺导卷 |
| 操作        | ✕ 删除按钮         | 删除该行                                                                                                                                       |

底部有 `[+ 添加处理指令]` 按钮。

> **结构依据**：`处理指令 = [位置范围] + [可选:缺陷类型] + [处理方式]`，基于2315条评审信息分析。

#### 区块四：评审备注

| 字段         | 类型                           |
| ------------ | ------------------------------ |
| 备注（可选） | 多行自由文本                   |
| 评审人       | 自动填充当前用户（Mock：张三） |
| 评审时间     | 自动填充当前时间               |

### 4.4 顶部操作按钮

| 按钮         | 行为                                           |
| ------------ | ---------------------------------------------- |
| `← 返回列表` | 路由跳转回 `/quality`                          |
| `保存草稿`   | 保存当前录入内容但不提交，状态留为"待评审"     |
| `提交评审`   | 校验必填项后提交，状态改为"已评审"，跳回列表页 |
