# 物料管理模块设计文档

> **日期**：2026-03-09
> **模块类型**：前端原型/Demo
> **目标**：展示物料管理模块完整最终效果，解决"在制品追踪困难"痛点
> **依据**：MES系统项目方案\_痛点分析与解决方案\_v2.md、排产模块调研总结.md、质量模块调研总结.md

## 一、模块定位

解决**痛点3：在制品追踪困难**：

| 现状痛点                         | 系统解决方案             |
| -------------------------------- | ------------------------ |
| 约1000个子卷堆在现场，靠人工找料 | 库位地图可视化，搜索定位 |
| 只有料框号，框在哪里不知道       | 引入区域+库位两级定位    |
| 分切→精切无管理表                | 在制品状态看板，实时可见 |
| 状态不可见，不知道料在哪个环节   | 18种细粒度状态流转追踪   |
| 呆滞料长期积压无人管理           | 超期自动预警，分级标红   |

## 三、页面设计

### 3.1 在制品看板（MaterialDashboardView）

**布局结构**：上方统计卡片行 + 下方左右分栏

**统计卡片行（6个）**：

| 卡片       | 数值 | 图标色 |
| ---------- | ---- | ------ |
| 在制品总数 | 156  | 青色   |
| 今日流转   | 42   | 绿色   |
| 待分切     | 23   | 蓝色   |
| 待精切     | 128  | 紫色   |
| 超7天未动  | 12   | 橙色   |
| 呆滞预警   | 5    | 红色   |

**左侧区域（约60%）**：

- **工序流转可视化**：横向流程图展示 胚料→轧机→分切→精切→包装 每个节点标注在制品数量
- **各区域在制品分布条形图**：轧机区/分切区/精切区/退火区的在制品数量对比

**右侧区域（约40%）**：

- **呆滞预警列表**：超期料卷排序，显示卷号、停放天数、位置、状态，红/橙分级
- **最近流转记录**：时间线列表，显示最近的入库/出库/移库操作

### 3.2 库位地图（MaterialLocationView）

**布局结构**：搜索栏 + 工序区域Tab + 网格地图 + 点击弹窗

**搜索栏**：输入卷号搜索，定位到地图上对应库位高亮闪烁

**工序区域Tab**：

- 分切区（在制品最多的区域，默认显示）
- 轧机区
- 精切区
- 退火区

**每个区域的网格地图**：

- 按排(A/B/C/D...)×列(01-10)共约30-50个库位
- 每个库位格显示：缩写卷号 + 停放天数
- 颜色编码：
  - 虚线框 = 空位
  - 绿色 = 正常（<3天）
  - 黄色 = 注意（3-7天）
  - 橙色 = 超期（7-14天），脉冲动画
  - 红色 = 呆滞（>14天），脉冲动画

**库位详情弹窗**：点击库位格弹出浮窗

- 卷号、规格、母卷号、客户、入库时间、停放天数
- 质量状态（已评审/质检中）
- 操作按钮：查看详情（跳转料卷查询）、发起流转

### 3.3 料卷查询（MaterialCoilsView）

**布局结构**：搜索筛选栏 + 左右分栏（列表+详情面板）

**搜索筛选栏**：

- 卷号搜索框
- 状态筛选下拉（支持多选）
- 工序筛选下拉（轧机/分切/精切/退火）
- 区域筛选下拉（分切区/轧机区/精切区/退火区）

**左侧列表（约40%）**：

- 表格形式：卷号、规格（精简）、当前状态（带色标签）、位置
- 选中行高亮，触发右侧详情面板更新

**右侧详情面板（约60%）**：

- **基本信息区**：卷号、母卷号、车次、规格（厚度×宽度×米数）、客户、合金、重量、当前位置
- **质量评审区**：主结论 + 处理指令列表（与质量模块数据联动）
- **流转历史时间线**：从胚料出库到当前状态的完整履历，每个节点标注：
  - 时间（日期+时间）
  - 状态变更
  - 操作人
  - 位置变更

## 五、Mock数据设计

### 5.1 库位区域

| 区域   | 排数       | 每排库位数 | 总库位 | 在制品填充率     |
| ------ | ---------- | ---------- | ------ | ---------------- |
| 分切区 | 4排（A-D） | 10列       | 40     | ~70%（28个有料） |
| 轧机区 | 3排（A-C） | 8列        | 24     | ~60%（14个有料） |
| 精切区 | 3排（A-C） | 10列       | 30     | ~50%（15个有料） |
| 退火区 | 2排（A-B） | 6列        | 12     | ~50%（6个有料）  |

### 5.2 料卷数据量

- 总计约63个在制品料卷（分布在各区域）
- 各状态分布合理：大部分在"待精切"阶段（符合实际约1000卷堆在分切→精切之间的痛点）
- 呆滞料5-8个（>14天），超期料10-15个（7-14天）

### 5.3 流转记录

约20条最近流转记录，覆盖入库、出库、移库、质检完成等各种操作类型

## 六、与已有模块的数据关联

| 关联模块 | 关联方式             | 说明                                    |
| -------- | -------------------- | --------------------------------------- |
| 排产模块 | 排产计划中的来料信息 | 料卷的排产状态对应排产模块中的计划记录  |
| 质量模块 | 质量评审结果         | 料卷详情中展示该卷的评审主结论+处理指令 |
| 订单池   | 料卷关联的订单信息   | 料卷详情中展示目标客户/订单号           |

> **原型阶段**：数据通过独立的 `material-data.js` mock数据文件提供，不与其他模块数据文件直接引用，但字段语义保持一致。

## 七、不做的项目

- 现场终端页（扫码/大按钮界面）— 属于移动端/工控屏场景
- 入库/出库/移库操作弹窗 — 原型只展示结果，不模拟操作流程
- 与用友MES的报工对接逻辑
- 电池箔/双零箔的差异化展示（原型统一展示）
