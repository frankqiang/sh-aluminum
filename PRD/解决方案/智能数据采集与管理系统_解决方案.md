# 智能数据采集与管理系统 —— 解决方案

## 第一部分：方案概述

### 1.1 项目背景

根据对分切工序现状的调研分析，当前生产流程存在以下核心问题：

| 序号 | 问题类别     | 具体描述                                       | 影响程度   |
| ---- | ------------ | ---------------------------------------------- | ---------- |
| 1    | 数据采集滞后 | 依赖手抄报表，次日下午才能录入系统             | ⭐⭐⭐⭐⭐ |
| 2    | 评审流程滞后 | 权限范围外需待次日质量工程师评审，料卷暂放现场 | ⭐⭐⭐⭐⭐ |
| 3    | 计划匹配困难 | 计划人员需人工匹配订单与评审信息               | ⭐⭐⭐⭐   |
| 4    | 找料挑料困难 | 现场约1000个料卷，人工挑选费时费力             | ⭐⭐⭐⭐⭐ |
| 5    | 数据不集中   | 针检、表检、离线板型、下料数据分散             | ⭐⭐⭐⭐   |
| 6    | 追溯困难     | 包装工序匹配木箱依赖人工手抄，错误率高         | ⭐⭐⭐⭐   |
| 7    | 安全隐患     | 物料堆积高、人员爬上爬下挂吊带存在风险         | ⭐⭐⭐     |

### 1.2 系统定位

**智能数据采集与管理系统** 作为分切工序智能化改造的 **第一阶段核心基础系统**，定位如下：

- **近期目标：** 实现数据实时采集、智能评审、自动排产，消除信息滞后
- **中期目标：** 为高架库、AGV等硬件智能化提供数据基础和系统对接能力
- **远期目标：** 构建分切工序完整的数字化运营体系

### 1.3 系统架构总览

系统采用 **"4+4"架构**，包含4个核心业务模块和4个基础支撑模块：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                               应用层                                         │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐          │
│  │ 生产看板 │ │ 质量报表 │ │ 排产管理 │ │ 追溯查询 │ │ 系统管理 │          │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘ └──────────┘          │
├─────────────────────────────────────────────────────────────────────────────┤
│                               业务层                                         │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐                       │
│  │ 数据采集 │ │ 智能评审 │ │ 智能排产 │ │ 质量追溯 │   ← 核心业务模块      │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘                       │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐                       │
│  │ 基础数据 │ │料卷生命周期│ │ 用户权限 │ │ 消息通知 │   ← 基础支撑模块      │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                               对接层                                         │
│  ┌──────────────────┐    ┌──────────────────┐    ┌──────────────────┐      │
│  │  现有ERP系统对接  │    │ 后续WMS/高架库对接│    │  消息推送对接    │      │
│  └──────────────────┘    └──────────────────┘    └──────────────────┘      │
├─────────────────────────────────────────────────────────────────────────────┤
│                               采集层                                         │
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐        │
│  │ 针检仪 │ │ 表检仪 │ │ 板型仪 │ │ 分切机 │ │ 电晕机 │ │ 精切机 │        │
│  └────────┘ └────────┘ └────────┘ └────────┘ └────────┘ └────────┘        │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 第二部分：核心业务模块设计

### 2.1 数据自动采集模块

#### 2.1.1 模块目标

替代现有的手抄报表，消除"次日下午才能录入"的滞后问题，实现生产数据实时采集与集中管理。

#### 2.1.2 数据采集清单

| 数据类型     | 数据来源    | 采集方式        | 采集时机 | 采集内容                                          |
| ------------ | ----------- | --------------- | -------- | ------------------------------------------------- |
| 料卷基础信息 | 分切机/人工 | 扫码 + 人工录入 | 下料时   | 卷号、规格、重量、母卷号、产品类型（母卷/精切料） |
| 针检数据     | 针检仪      | 设备接口/人工   | 实时     | 针孔数量、密度、位置分布、采集时间                |
| 表检数据     | 表检仪      | 设备接口/人工   | 实时     | 缺陷类型、数量、位置、缺陷图像                    |
| 板型数据     | 离线板型仪  | 设备接口/人工   | 检测后   | 板型偏差值、曲线图、检测时间                      |
| 电晕数据     | 电晕机      | 设备接口/人工   | 生产时   | 电晕功率、速度、宽度、处理时间                    |
| 精切数据     | 精切机/人工 | 设备接口/人工   | 下料时   | 精切规格、米数、宽度、厚度、操作人员              |
| 位置信息     | 人工扫码    | 扫码定位        | 转运时   | 当前存放区域、货位号（为高架库预留）              |

### 2.2 智能评审模块

#### 2.2.1 模块目标

根据采集数据自动进行质量判定，权限范围内自动评审，消除"需待次日质量工程师评审"的滞后问题。

#### 2.2.2 产品类型与评审流程差异

根据调研，分切后物料分为两类，评审流程有所不同：

| 产品类型       | 合格后流向            | 不合格后流向                       | 说明               |
| -------------- | --------------------- | ---------------------------------- | ------------------ |
| **电池箔母卷** | 直接转包装 → 入库发货 | 二次评审 → 合格转包装/不合格暂放   | 主产品，流转快     |
| **精切料**     | 暂放现场等待排产      | 二次评审 → 合格等待排产/不合格暂放 | 需与订单匹配后精切 |

#### 2.2.3 自动评审规则框架

| 检测项   | 自动判定合格 | 自动判定不合格 | 需人工评审    |
| -------- | ------------ | -------------- | ------------- |
| 针孔数量 | ≤ A个/㎡     | > B个/㎡       | A < 数量 ≤ B  |
| 表面缺陷 | 无A类缺陷    | 存在A类缺陷    | 仅存在B类缺陷 |
| 板型偏差 | < X mm       | > Y mm         | X ≤ 偏差 ≤ Y  |

> **说明：** A、B、X、Y等具体阈值需与客户质量部门确认

#### 2.2.4 评审结果与产品等级

**评审结果类型：**
| 结果类型 | 说明 | 后续处理 |
|----------|------|----------|
| 合格-一等品 | 完全符合标准，无任何缺陷 | 可用于最高要求客户订单 |
| 合格-二等品 | 存在轻微瑕疵，但在可接受范围 | 可用于一般要求客户订单 |
| 合格-三等品 | 降级使用 | 可用于低要求客户订单 |
| 不合格-返工 | 可修复缺陷 | 返回上游工序修复 |
| 不合格-报废 | 无法修复 | 报废处理 |

**降级使用规则：**
| 降级原因 | 原等级 | 降至等级 | 审批权限 |
|----------|--------|----------|----------|
| 针孔略超标（≤10%） | 一等品 | 二等品 | 当班质检 |
| B类表面缺陷 | 一等品 | 二等品 | 当班质检 |
| 板型偏差略超标（≤15%） | 一等品 | 二等品 | 当班质检 |
| 针孔超标（10%-30%） | 一等品 | 三等品 | 质量工程师 |
| 多项指标略超标 | 一等品 | 三等品 | 质量工程师 |

#### 2.2.5 评审流程图

```
                                    电池箔母卷流程
                                         │
质检完成 → 系统自动判定 → ┬→ 合格 ────────┴──────────→ 直接转包装 → 入库发货
                         │
                         ├→ 待评审 → 通知质量工程师 → 人工评审 → ┬→ 合格 → 转包装
                         │                                      ├→ 降级 → 转包装（标注等级）
                         │                                      └→ 不合格 → 隔离处理
                         │
                         └→ 不合格 → 自动隔离 → ┬→ 返工 → 待检测
                                               └→ 报废

                                    精切料流程
                                         │
质检完成 → 系统自动判定 → ┬→ 合格 ────────┴──────────→ 暂放现场 → 等待排产
                         │
                         ├→ 待评审 → 通知质量工程师 → 人工评审 → ┬→ 合格 → 等待排产
                         │                                      ├→ 降级 → 等待排产（标注等级）
                         │                                      └→ 不合格 → 隔离处理
                         │
                         └→ 不合格 → 自动隔离 → ┬→ 返工 → 待检测
                                               └→ 报废
```

### 2.3 智能排产模块

#### 2.3.1 模块目标

根据质检数据自动选料，与客户订单自动匹配，生成精切来料计划，解决"计划滞后、订单匹配困难"问题。

#### 2.3.2 核心功能

| 功能         | 说明                                     | 解决的问题       |
| ------------ | ---------------------------------------- | ---------------- |
| 订单管理     | 从ERP同步客户订单，管理订单状态          | 订单信息集中管理 |
| 自动选料     | 根据订单规格要求，自动从合格料卷池中匹配 | 替代人工匹配     |
| 排产计划     | 生成精切来料计划，指导生产               | 消除计划滞后     |
| 订单跟踪     | 跟踪订单执行进度，支持订单变更           | 提高交付可控性   |
| 降级产品匹配 | 自动匹配可接受降级产品的客户订单         | 降级产品及时消化 |

#### 2.3.3 物料匹配规则

| 匹配参数  | 匹配规则                 | 优先级 | 说明                         |
| --------- | ------------------------ | ------ | ---------------------------- |
| 厚度      | 必须完全匹配             | 1      | 硬性条件                     |
| 宽度      | 料卷宽度 ≥ 订单要求宽度  | 2      | 允许切边                     |
| 长度/米数 | 料卷米数 ≥ 订单要求米数  | 3      | 允许切割                     |
| 质量等级  | 料卷等级 ≥ 订单要求等级  | 4      | 一等品可用于二等品订单       |
| 库存时间  | 优先选择库存时间长的料卷 | 5      | 先进先出原则                 |
| 存放位置  | 优先选择易取的料卷       | 6      | 减少找料时间（为高架库预留） |

**降级产品匹配逻辑：**

```
降级料卷 → 查找可接受该等级的客户订单 → 自动匹配 → 纳入排产计划
                    │
                    └→ 若无匹配订单 → 标记"待匹配" → 等待新订单或人工处理
```

#### 2.3.4 排产优先级规则

系统按照以下规则确定订单的排产顺序（规则1最重要，依次递减）：

| 优先级 | 规则名称         | 规则说明                                       | 举例说明                                     |
| ------ | ---------------- | ---------------------------------------------- | -------------------------------------------- |
| **1**  | **交期优先**     | 交货日期越近的订单，越优先安排生产             | 1月25日交货的订单，比2月10日交货的订单优先   |
| **2**  | **重要客户优先** | VIP客户的订单优先于普通客户                    | 同样是1月25日交货，VIP客户的订单先安排       |
| **3**  | **相同规格集中** | 尽量把相同规格的订单连续排产，减少设备换型次数 | 先集中做0.012mm厚度的订单，做完再做0.015mm的 |
| **4**  | **先进先出**     | 库存时间长的料卷优先使用，避免料卷长期积压     | 1月10日入库的料卷，比1月20日入库的优先使用   |

**规则权重（可配置调整）：**

- 交期优先占 **40%** 权重
- 重要客户优先占 **30%** 权重
- 相同规格集中占 **20%** 权重
- 先进先出占 **10%** 权重

> **通俗理解：** 系统会综合考虑以上4个因素，自动计算每个订单的"紧急程度分数"，分数越高的订单越先安排生产。如果两个订单交期相同，就看谁是VIP客户；如果都是VIP客户，就看能否和其他订单凑成同规格一起生产。

#### 2.3.5 找料过渡方案（高架库前）

针对"现场约1000个料卷，找料困难"的问题，在高架库建成前，系统提供以下过渡方案：

**地堆区位置管理：**
| 功能 | 说明 |
|------|------|
| 区域划分 | 将地堆区划分为A/B/C/D等区域 |
| 位置登记 | 料卷下料时登记存放区域 |
| 扫码更新 | 料卷移动后扫码更新位置 |
| 找料指引 | 排产后系统显示料卷所在区域 |
| 位置优化建议 | 系统建议将常用规格放置于易取区域 |

**位置信息字段设计（预留高架库扩展）：**

```
位置信息 = {
    位置类型: "地堆区" / "高架库" / "在途",
    区域编号: "A" / "B" / "C" / "D" / "货架号",
    层号: "1" / "2" / "3"（地堆层数）/ "货位层",
    更新时间: "2026-01-21 10:30:00",
    更新人员: "张三"
}
```

### 2.4 追溯查询模块

#### 2.4.1 模块目标

实现全流程数据追溯（从轧机到包装），便于质量问题追溯和分析，为包装工序提供准确的匹配信息，解决"手抄信息错误率高"问题。

#### 2.4.2 追溯方向

| 追溯类型     | 追溯路径                                  | 应用场景               |
| ------------ | ----------------------------------------- | ---------------------- |
| **正向追溯** | 轧机母卷 → 分切料卷 → 精切料卷 → 包装成品 | 了解产品来源和加工过程 |
| **反向追溯** | 包装成品 → 精切料卷 → 分切料卷 → 轧机母卷 | 客户投诉时追溯问题源头 |
| **横向追溯** | 同一母卷的所有分切料卷、同批次料卷        | 批次性质量问题排查     |

#### 2.4.3 追溯信息内容

| 信息类别     | 具体内容                                                 |
| ------------ | -------------------------------------------------------- |
| **生产信息** | 各工序的生产时间、使用设备、操作人员、班次               |
| **质量信息** | 质检数据（针检/表检/板型）、评审结果、评审人员、评审时间 |
| **工艺信息** | 分切参数、电晕参数、精切参数                             |
| **状态信息** | 料卷状态流转历史（何时从A状态变为B状态）                 |
| **位置信息** | 料卷位置变更历史（何时从A区域移至B区域）                 |
| **缺陷信息** | 缺陷记录、缺陷来源工序、处理结果                         |

#### 2.4.4 包装工序信息支持

系统为包装工序提供以下准确信息，解决"人工手抄错误率高"问题：

| 提供信息                       | 用途                   |
| ------------------------------ | ---------------------- |
| 精切料卷规格（厚度×宽度×米数） | 匹配合适尺寸的木箱     |
| 料卷重量                       | 匹配木箱承重能力       |
| 客户订单信息                   | 确保包装与发货对应     |
| 质量等级                       | 包装标识和质量证书打印 |

### 3.2 料卷全生命周期管理模块

#### 3.2.1 模块目标

管理料卷从产生到发货的全状态流转，实现料卷全程可追踪。

#### 3.2.2 料卷核心属性

| 属性分类     | 属性字段                                          | 说明               |
| ------------ | ------------------------------------------------- | ------------------ |
| **基础属性** | 卷号、母卷号、规格（厚×宽×米数）、重量            | 身份标识和物理属性 |
| **分类属性** | 产品类型（母卷/精切料）、质量等级（一/二/三等品） | 分类管理           |
| **状态属性** | 当前状态、状态更新时间                            | 流程跟踪           |
| **位置属性** | 位置类型、区域编号、层号                          | 找料定位           |
| **订单属性** | 关联订单号、客户编码                              | 排产匹配           |
| **质量属性** | 针检结果、表检结果、板型结果、评审结果            | 质量追溯           |

#### 3.2.3 状态定义

| 状态   | 说明                   | 可执行操作                     | 适用产品类型 |
| ------ | ---------------------- | ------------------------------ | ------------ |
| 待检测 | 分切下料后，等待质检   | 开始质检                       | 母卷、精切料 |
| 检测中 | 正在进行质检           | 完成质检                       | 母卷、精切料 |
| 待评审 | 质检完成，需要人工评审 | 提交评审                       | 母卷、精切料 |
| 合格   | 质检/评审通过          | 转包装（母卷）、排产（精切料） | 母卷、精切料 |
| 待精切 | 已分配订单，等待精切   | 上料精切                       | 精切料       |
| 精切中 | 正在精切生产           | 完成精切                       | 精切料       |
| 待包装 | 等待包装               | 开始包装                       | 母卷、精切料 |
| 已包装 | 已装箱，等待发货       | 安排发货                       | 母卷、精切料 |
| 已发货 | 已出库发货             | 归档                           | 母卷、精切料 |
| 不合格 | 质检/评审不通过        | 返工、报废                     | 母卷、精切料 |

#### 3.2.4 状态流转图

```
【电池箔母卷流程】
分切下料 → 待检测 → 检测中 → ┬→ 合格 → 待包装 → 已包装 → 已发货
                             ├→ 待评审 → 评审 → ┬→ 合格/降级 → 待包装
                             │                  └→ 不合格 → 隔离
                             └→ 不合格 → 隔离 → ┬→ 返工 → 待检测
                                               └→ 报废

【精切料流程】
分切下料 → 待检测 → 检测中 → ┬→ 合格 → 待精切 → 精切中 → 待包装 → 已包装 → 已发货
                             ├→ 待评审 → 评审 → ┬→ 合格/降级 → 待精切
                             │                  └→ 不合格 → 隔离
                             └→ 不合格 → 隔离 → ┬→ 返工 → 待检测
                                               └→ 报废
```

## 第六部分：异常处理与离线机制

### 6.1 异常处理机制

| 异常场景         | 处理方式                   | 恢复措施           |
| ---------------- | -------------------------- | ------------------ |
| 设备数据采集失败 | 记录异常日志，支持人工补录 | 恢复后继续自动采集 |
| 网络中断         | 本地缓存数据，提示离线状态 | 网络恢复后自动同步 |
| 系统故障         | 自动保存操作数据           | 支持数据恢复       |
| 数据校验失败     | 提示错误信息，要求重新输入 | 日志记录便于排查   |
| ERP对接异常      | 记录异常，人工通知IT处理   | 支持手工数据补录   |

### 6.2 离线模式设计

- 车间终端支持离线操作（网络中断时可继续录入）
- 数据本地缓存，网络恢复后自动上传
- 离线数据标识，便于后续核对
- 离线期间关键报警通过短信备份通知
