## 1. 产品概述

### 1.1 产品背景

某铝箔公司分切车间目前存在以下核心痛点：

| 痛点         | 现状描述                                   | 业务影响                     |
| ------------ | ------------------------------------------ | ---------------------------- |
| **评审滞后** | 质检数据手抄记录，次日下午才能录入系统     | 料卷积压、计划滞后、占用场地 |
| **数据分散** | 针检、表检、板型数据分散在不同设备和报表中 | 无法快速获取完整质量信息     |
| **找料困难** | 约1000个料卷地堆存放，人工查找             | 平均找料耗时20分钟，效率低下 |
| **计划滞后** | 订单与料卷匹配依赖人工操作                 | 响应慢、易出错、效率低       |
| **追溯困难** | 数据分散在纸质报表和Excel中                | 质量追溯需要半天以上时间     |

### 1.2 产品目标

建立集中式智能管理系统，实现以下目标：

| 目标             | 衡量指标                       | 目标值             |
| ---------------- | ------------------------------ | ------------------ |
| **消除录入滞后** | 数据录入到系统可查询的时间     | ≤5分钟             |
| **加速评审流程** | 从质检完成到评审结论的时间     | ≤2小时（当前次日） |
| **提高排产效率** | 订单匹配与料卷分配时间         | 系统自动完成       |
| **快速质量追溯** | 从查询到获取完整追溯信息的时间 | ≤5分钟（当前半天） |

## 3. 核心业务功能

### 3.1 数据采集功能

#### 3.1.1 分切下料数据采集

**业务场景：**
分切操作工完成分切下料后，需要快速录入料卷信息，替代现有的手抄报表方式。

**操作流程：**

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  分切完成   │────▶│  扫描母卷   │────▶│  录入信息   │────▶│  打印标签   │
│  准备下料   │     │  条码/二维码│     │  确认提交   │     │  粘贴料卷   │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
                          │                   │                   │
                          ▼                   ▼                   ▼
                    系统自动带出         系统自动生成         料卷状态变为
                    母卷规格信息         料卷编号             "待检测"
```

**功能逻辑说明：**

| 步骤        | 触发条件                     | 处理逻辑                                                       | 输出结果         |
| ----------- | ---------------------------- | -------------------------------------------------------------- | ---------------- |
| 1. 扫描母卷 | 操作工扫描母卷条码           | 系统查询母卷信息，自动填充厚度、批次等继承信息                 | 界面显示母卷信息 |
| 2. 录入规格 | 操作工输入宽度、长度、重量   | 系统校验数据有效性（参见BR-101~BR-103）                        | 数据暂存         |
| 3. 确认提交 | 操作工点击"确认"按钮         | 系统生成料卷编号（参见BR-104），创建料卷记录，状态设为"待检测" | 返回成功提示     |
| 4. 打印标签 | 系统自动触发或操作工手动触发 | 生成标签内容，发送打印指令                                     | 打印料卷标签     |

**业务规则：**

| 规则编号 | 规则名称     | 规则描述                                                                           |
| -------- | ------------ | ---------------------------------------------------------------------------------- |
| BR-101   | 宽度有效性   | 宽度必须在50mm~2000mm范围内，精度0.1mm                                             |
| BR-102   | 长度有效性   | 长度必须在100m~10000m范围内，精度1m                                                |
| BR-103   | 重量有效性   | 重量必须在10kg~5000kg范围内，精度0.1kg                                             |
| BR-104   | 料卷编号规则 | 格式：FC + 日期(8位) + 班次(1位) + 设备号(2位) + 流水号(3位)，如：FC20260120A01001 |
| BR-105   | 母卷关联     | 一个母卷可分切多个料卷，系统自动记录关联关系                                       |
| BR-106   | 数据修改限制 | 下料数据当班可修改，需班组长确认；跨班后只读，特殊情况需生产主管审批               |

**异常处理：**

| 异常场景         | 处理逻辑                                                |
| ---------------- | ------------------------------------------------------- |
| 母卷条码无法识别 | 提示"条码无法识别"，支持手工输入母卷号                  |
| 母卷号不存在     | 提示"母卷号不存在，请核对"，不允许继续                  |
| 必填项未填写     | 高亮显示未填写字段，提示"请填写必填项"                  |
| 数据超出有效范围 | 提示具体字段超出范围，如"宽度超出有效范围（50-2000mm）" |
| 标签打印失败     | 提示"打印失败"，支持重新打印，不影响数据保存            |

#### 3.1.2 质检数据采集

**业务场景：**
质检员对分切后的料卷进行针检、表检、板型三项检测，并录入检测数据。

**操作流程：**

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  选择料卷   │────▶│  录入针检   │────▶│  录入表检   │────▶│  录入板型   │
│  扫码确认   │     │  数据       │     │  数据       │     │  数据       │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
                                                                   │
                    ┌─────────────┐     ┌─────────────┐           │
                    │  通知相关   │◀────│  自动判定   │◀──────────┘
                    │  人员       │     │  质量等级   │
                    └─────────────┘     └─────────────┘
```

**功能逻辑说明：**

| 步骤        | 触发条件             | 处理逻辑                                                    | 输出结果                       |
| ----------- | -------------------- | ----------------------------------------------------------- | ------------------------------ |
| 1. 选择料卷 | 质检员扫描料卷条码   | 校验料卷状态为"待检测"，显示料卷基本信息                    | 进入质检录入界面               |
| 2. 录入针检 | 输入针孔数量/密度    | 系统保存针检数据，根据产品类型判定单项结果                  | 针检结果（合格/待评审/不合格） |
| 3. 录入表检 | 选择缺陷类型和数量   | 系统保存表检数据，根据缺陷等级判定单项结果                  | 表检结果（合格/待评审/不合格） |
| 4. 录入板型 | 输入板型偏差值       | 系统保存板型数据，根据偏差判定单项结果                      | 板型结果（合格/待评审/不合格） |
| 5. 综合判定 | 所有检测项录入完成   | 系统根据复合判定规则（BR-202）计算综合结果                  | 料卷质量等级                   |
| 6. 状态更新 | 综合判定完成         | 更新料卷状态：合格→"合格"，待评审→"待评审"，不合格→"不合格" | 状态变更                       |
| 7. 消息通知 | 状态为"待评审"时触发 | 向质量工程师发送评审通知                                    | 系统消息/微信通知              |

**业务规则：**

| 规则编号 | 规则名称         | 规则描述                                                                                    |
| -------- | ---------------- | ------------------------------------------------------------------------------------------- |
| BR-201   | 质检权限边界     | 当班质检可判定：针孔≤10个/㎡、无A类缺陷、板型≤5mm的料卷为"合格"；超出此范围需质量工程师评审 |
| BR-202   | 复合判定规则     | 任一检测项为"不合格"→综合结果"不合格"；任一检测项为"待评审"→综合结果"待评审"；其他→"合格"   |
| BR-203   | 产品差异化标准   | 电池箔：针孔≤5个/㎡为A级；包装箔：针孔≤15个/㎡为A级（具体标准待客户确认）                   |
| BR-204   | 缺陷等级定义     | A类缺陷（严重）：有1个即不合格；B类缺陷（一般）：≤3个合格；C类缺陷（轻微）：≤10个合格       |
| BR-205   | 质检数据不可删除 | 质检数据一旦保存不可删除，只能由质量工程师修改并记录修改原因                                |

### 3.2 智能评审功能

#### 3.2.1 自动判定逻辑

**业务场景：**
系统根据质检数据和预设规则，自动判定料卷质量等级，减少人工判定工作量。

**判定规则引擎逻辑：**

```
质检完成
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│                     自动判定规则引擎                          │
│                                                              │
│  输入：针检数据、表检数据、板型数据、产品类型                  │
│                                                              │
│  Step 1: 查询产品类型对应的判定标准                          │
│  Step 2: 逐项对比检测数据与标准阈值                          │
│  Step 3: 根据BR-202复合判定规则计算综合结果                  │
│  Step 4: 判断是否在当班质检权限范围内（BR-201）              │
│                                                              │
│  输出：判定结果（合格/待评审/不合格）、质量等级（A/B/C/D）    │
└─────────────────────────────────────────────────────────────┘
    │
    ├──▶ 合格 ──▶ 状态更新为"合格"，可参与排产
    │
    ├──▶ 待评审 ──▶ 状态更新为"待评审"，通知质量工程师
    │
    └──▶ 不合格 ──▶ 状态更新为"不合格"，自动隔离
```

**判定标准配置（示例，待客户确认）：**

| 检测项   | 产品类型 | A级（优） | B级（良）  | C级（待评审） | D级（不合格）  |
| -------- | -------- | --------- | ---------- | ------------- | -------------- |
| 针孔密度 | 电池箔   | ≤5个/㎡   | 6-8个/㎡   | 9-12个/㎡     | >12个/㎡       |
| 针孔密度 | 包装箔   | ≤10个/㎡  | 11-15个/㎡ | 16-20个/㎡    | >20个/㎡       |
| 表面缺陷 | 全部     | 无缺陷    | B类≤2个    | B类3-5个      | 有A类或B类>5个 |
| 板型偏差 | 电池箔   | ≤2mm      | 2-4mm      | 4-6mm         | >6mm           |
| 板型偏差 | 包装箔   | ≤5mm      | 5-8mm      | 8-10mm        | >10mm          |

#### 3.2.2 人工评审工作流

**业务场景：**
对于自动判定为"待评审"的料卷，质量工程师进行人工评审并做出最终决定。

**评审工作流：**

```
┌───────────────────────────────────────────────────────────────────────────┐
│                            人工评审工作流                                   │
└───────────────────────────────────────────────────────────────────────────┘

┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ 待评审队列  │────▶│  查看详情   │────▶│  做出决定   │────▶│  填写意见   │
│ （按优先级）│     │  质检数据   │     │  评审结论   │     │  提交评审   │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
      │                   │                   │                   │
      ▼                   ▼                   ▼                   ▼
 系统自动排序        可查看图像、        选择：合格/降级/      系统更新状态
 紧急优先           曲线图等            返工/报废             并通知相关人员
```

**评审决定类型：**

| 评审结论     | 说明                   | 后续处理                                       | 状态变更            |
| ------------ | ---------------------- | ---------------------------------------------- | ------------------- |
| **合格**     | 虽在临界值但可正常使用 | 进入可排产料卷池                               | 待评审 → 合格       |
| **降级使用** | 降低质量等级后可使用   | 标记降级原因，进入可排产池，参见BR-301降级规则 | 待评审 → 合格(降级) |
| **返工**     | 返回上游工序重新加工   | 生成返工单，通知生产，参见BR-302返工规则       | 待评审 → 待返工     |
| **报废**     | 无法使用，作报废处理   | 生成报废单，移入隔离区                         | 待评审 → 报废       |

**业务规则：**

| 规则编号 | 规则名称   | 规则描述                                                                           |
| -------- | ---------- | ---------------------------------------------------------------------------------- |
| BR-301   | 降级规则   | A级降B级：针孔10-12个/㎡或有1-2个B类缺陷；B级降C级：针孔15-18个/㎡或有3-4个B类缺陷 |
| BR-302   | 返工条件   | 仅板型不合格可返工（重新精切）；表面缺陷和针孔超标不可返工                         |
| BR-303   | 评审时限   | 待评审料卷应在2小时内完成评审；超时自动升级通知质量主管                            |
| BR-304   | 评审优先级 | 紧急订单关联料卷 > 库存时间超7天 > 临界值料卷 > 其他                               |

#### 3.3.2 自动选料匹配逻辑

**业务场景：**
根据订单要求，从合格料卷池中自动匹配合适的料卷，减少人工匹配工作量。

**匹配算法逻辑：**

```
订单需求输入
      │
      ▼
┌─────────────────────────────────────────────────────────────┐
│                     自动选料匹配引擎                          │
│                                                              │
│  Step 1: 筛选条件（必须满足）                                │
│    • 料卷状态 = 合格                                        │
│    • 料卷厚度 = 订单厚度（精确匹配）                         │
│    • 料卷宽度 ≥ 订单宽度（可精切调整）                       │
│    • 料卷长度 ≥ 订单长度（不能不足）                         │
│    • 料卷等级 ≥ 订单要求等级（可高配低用）                   │
│                                                              │
│  Step 2: 排序优先级                                          │
│    优先级1: 宽度最接近（减少损耗）                           │
│    优先级2: 等级刚好满足（避免高配低用浪费）                 │
│    优先级3: 库存时间最长（先进先出）                         │
│                                                              │
│  Step 3: 分配逻辑                                            │
│    • 累计分配直到满足订单数量                               │
│    • 支持一个订单对应多个料卷                               │
│    • 记录料卷与订单的绑定关系                               │
└─────────────────────────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────────────────────────┐
│  匹配结果：                                                  │
│  • 完全满足：推荐料卷列表，等待计划员确认                   │
│  • 部分满足：推荐可用料卷，提示缺口数量                     │
│  • 无法满足：提示"无合适料卷"，建议等待或调整订单要求      │
└─────────────────────────────────────────────────────────────┘
```

**匹配规则（待客户确认优先级）：**

| 规则编号 | 匹配参数 | 匹配方式                         | 优先级    |
| -------- | -------- | -------------------------------- | --------- |
| BR-401   | 厚度     | 精确匹配，必须一致               | 1（最高） |
| BR-402   | 宽度     | 料卷宽度 ≥ 订单宽度              | 2         |
| BR-403   | 长度     | 料卷长度 ≥ 订单长度              | 3         |
| BR-404   | 质量等级 | 料卷等级 ≥ 订单要求等级          | 4         |
| BR-405   | 库存时间 | 优先选择库存时间长的料卷（FIFO） | 5         |

| BR-102   | 长度有效性   | 长度必须在100m~10000m范围内，精度1m                  |
| BR-103   | 重量有效性   | 重量必须在10kg~5000kg范围内，精度0.1kg               |
| BR-104   | 料卷编号规则 | 格式：FC+日期(8位)+班次(1位)+设备号(2位)+流水号(3位) |
| BR-105   | 母卷关联     | 一个母卷可分切多个料卷，系统自动记录关联关系         |
| BR-106   | 数据修改限制 | 当班可修改需班组长确认，跨班只读需生产主管审批       |

### 6.2 质量判定规则（BR-2XX）

| 规则编号 | 规则名称       | 规则描述                                                |
| -------- | -------------- | ------------------------------------------------------- |
| BR-201   | 质检权限边界   | 当班质检可判定：针孔≤10个/㎡、无A类缺陷、板型≤5mm       |
| BR-202   | 复合判定规则   | 任一不合格→综合不合格；任一待评审→综合待评审；其他→合格 |
| BR-203   | 产品差异化标准 | 不同产品类型适用不同判定标准                            |
| BR-204   | 缺陷等级定义   | A类（严重）1个即不合格；B类≤3个合格；C类≤10个合格       |
| BR-205   | 质检数据保护   | 质检数据不可删除，只能由质量工程师修改并记录原因        |

### 6.3 评审规则（BR-3XX）

| 规则编号 | 规则名称   | 规则描述                                       |
| -------- | ---------- | ---------------------------------------------- |
| BR-301   | 降级规则   | A降B、B降C的具体条件（待客户确认详细阈值）     |
| BR-302   | 返工条件   | 仅板型不合格可返工，表面缺陷和针孔超标不可返工 |
| BR-303   | 评审时限   | 待评审料卷应在2小时内完成评审                  |
| BR-304   | 评审优先级 | 紧急订单>库存超7天>临界值>其他                 |

### 6.4 排产规则（BR-4XX）

| 规则编号 | 规则名称 | 规则描述                         |
| -------- | -------- | -------------------------------- |
| BR-401   | 厚度匹配 | 精确匹配，必须一致               |
| BR-402   | 宽度匹配 | 料卷宽度≥订单宽度                |
| BR-403   | 长度匹配 | 料卷长度≥订单长度                |
| BR-404   | 等级匹配 | 料卷等级≥订单要求等级            |
| BR-405   | 库存优先 | 优先选择库存时间长的料卷（FIFO） |
